    Beispiel für Brightcove Spieler: Zufällige Stoßstangen anzeigen

    In diesem Thema erfahren Sie, wie Sie einen Brightcove-Player verwenden, der zufällig ausgewählte Stoßstangen vor jedem Video in einer Wiedergabeliste anzeigt.

    Der Inhalt dieses Dokuments gilt nur für Video Cloud.

    Player-Beispiel

    Dieses Beispiel und der zugehörige Code dienen als Leitfaden für Ihre Produktionsentwicklung. Der Code ist weder für den produktiven Einsatz zertifiziert noch offiziell unterstützt.

    In diesem Beispiel werden zufällig ausgewählte Stoßstangen vor jedem Video aus einer Wiedergabeliste abgespielt. In diesem Fall handelt es sich bei den Stoßfängern um Anzeigen, die jedoch nicht von einem Anzeigenserver stammen. Stattdessen handelt es sich um Videos, die in der Video Cloud-Bibliothek definiert sind.

    Siehe den Stift Zufällige Stoßstangen anzeigen von Brightcove Learning Services ( rcrooks1969 ) auf CodePen.

    Quellcode

    Schau den Komplettlösung auf GitHub.

    Verwenden des CodePen

    Hier sind einige Tipps, um das obige CodePen effektiv zu nutzen:

    • Schalten Sie die tatsächliche Anzeige des Players ein, indem Sie auf die Result Schaltfläche klicken.
    • Klicken Sie auf die HTML/CSS/JS Schaltflächen, um EINEN der Codetypen anzuzeigen.
    • Später in diesem Dokument werden die in der Anwendung verwendete Logik, der Ablauf und das Styling in den Abschnitten Player/HTML-Konfiguration, Anwendungsfluss und Anwendungsstyling besprochen. Der beste Weg, um den Informationen in diesen Abschnitten zu folgen, ist:
      1. Klicken Sie auf die EDIT ON CODEPEN Schaltfläche im CodePen und halten Sie den Code in einem Browser/Browser-Tab zur Verfügung.
      2. Passen Sie in CodePen an, welcher Code angezeigt werden soll. Sie können die Breite verschiedener Codeabschnitte in CodePen ändern.
      3. Zeigen Sie die Abschnitte Player/HTML-Konfiguration, Anwendungsfluss und/oder Anwendungsstyling in einer anderen Browser-/Browserregisterkarte an. Sie können nun den Codeerklärungen folgen und gleichzeitig den Code anzeigen.

    Entwicklungs-Sequenz

    Hier ist die empfohlene Entwicklungssequenz:

    1. Verwenden Sie die In-Page Embed Player Implementierung, um die Funktionalität Ihres Players, Plugins und CSS zu testen (falls CSS benötigt wird)
    2. Platzieren Sie das JavaScript und das CSS des Plugins in separate Dateien für lokale Tests
    3. Stellen Sie den Plugin-Code und CSS auf Ihrem Server bereit, sobald Sie Fehler behoben haben
    4. Verwenden Sie Studio, um das Plugin und das CSS zu Ihrem Player hinzuzufügen
    5. Ersetzen Sie die In-Page-Embite-Player-Implementierung, wenn Sie feststellen, dass die iframe-Implementierung besser geeignet ist (im nächsten Abschnitt beschrieben)

    Einzelheiten zu diesen Schritten finden Sie im Schritt-für-Schritt: Leitfaden zur Plugin-Entwicklung

    iframe oder In-Page einbetten

    Bei der Entwicklung von Verbesserungen für den Brightcove Player müssen Sie entscheiden, ob der Code für die Implementierung von iframe oder In-Page Embed am besten geeignet ist. Die Best-Practice-Empfehlung besteht darin, ein Plugin für die Verwendung mit einer iframe-Implementierung zu erstellen. Die Vorteile der Verwendung des iFrame-Players sind:

    • Keine Kollisionen mit bestehendem JavaScript und/oder CSS
    • Automatisch ansprechbar
    • Der iframe erleichtert die Verwendung in Social-Media-Apps (oder wann immer das Video in andere Apps „reisen“ muss)

    Obwohl die Integration des In-Page-Embed Players komplexer sein kann, gibt es Zeiten, in denen Sie Ihren Code für diese Implementierung planen. Um zu verallgemeinern, ist dieser Ansatz am besten, wenn die enthaltene Seite mit dem Spieler kommunizieren muss. Konkret sind hier einige Beispiele:

    • Der Code auf der enthaltenen Seite muss Spielereignisse abhören und darauf reagieren
    • Der Spieler verwendet Stile von der enthaltenden Seite
    • Der iframe führt dazu, dass die App-Logik fehlschlägt, wie eine Weiterleitung von der enthaltenden Seite

    Selbst wenn Ihre endgültige Implementierung den Iframe-Code nicht verwendet, können Sie den In-Page-Einbettungscode immer noch mit einem Plugin für Ihr JavaScript und einer separaten Datei für Ihr CSS verwenden. Dies kapselt Ihre Logik, sodass Sie sie problemlos in mehreren Spielern verwenden können.

    Spieler-/HTML-Konfiguration

    Dieser Abschnitt beschreibt alle speziellen Konfigurationen, die während der Spielererstellung erforderlich sind. Darüber hinaus werden andere HTML-Elemente beschrieben, die der Seite über den In-Page-Einbettungscode des Einbettungsspielers hinaus hinzugefügt werden müssen.

    Spieler-Konfiguration

    Für den Brightcove Player, den Sie für dieses Beispiel erstellen, ist keine spezielle Konfiguration erforderlich.

    Anderes HTML

    Der Seite werden keine weiteren HTML-Elemente hinzugefügt.

    Ablauf der Anwendung

    Die grundlegende Logik hinter dieser Anwendung lautet:

    • Eine Reihe von Video Cloud-Videos wird als Stoßfänger eingerichtet, indem sie in einem Array definiert werden. In diesem Beispiel sind die Stoßstangen Anzeigen.
    • Ein Stoßfänger wird zufällig abgespielt, gefolgt von einem Video aus einer Wiedergabeliste. Die Benutzeroberfläche der Wiedergabeliste wird NICHT angezeigt. Da die Stoßstangen zufällig verwendet werden, kann ein Stoßfänger möglicherweise mehrmals verwendet werden.
    • Dieser Vorgang wechselt dann zwischen zufällig aus dem Array ausgewählten Stoßfängern und dem nächsten Video aus der Wiedergabeliste.

    Hören Sie zu ended Veranstaltungen für Stoßstangen und Videos

    Finde den Code, der beschriftet ist:

    // ### Listen for the ended event and play the next video or bumper ###

    Dieser Ereignishandler verfolgt, welches Video gerade abgespielt wurde, entweder einen Stoßfänger oder ein Video aus der Wiedergabeliste, und ruft dann die entsprechende Funktion auf, um einen weiteren Stoßfänger / ein anderes Video zu starten.

    Spielen Sie ein zufälliges Stoßfängervideo ab

    Finde den Code, der beschriftet ist:

    // ### Play random bumper video ###

    Diese Funktion generiert eine Zufallszahl im Bereich der verfügbaren Stoßstangen, ruft das Video ab, lädt es in den Player und spielt es ab. Diese Funktion wird auf Spieler bereit aufgerufen.

    Spielen Sie das nächste Video von der Wiedergabeliste ab, nachdem ein Stoßfänger fertig ist

    Finde den Code, der beschriftet ist:

    // ### Play next video from playlist ###

    Diese Funktion verfolgt die Videos in der Wiedergabeliste und spielt das nächste Video in der Wiedergabeliste ab.

    Holen Sie sich die Wiedergabeliste beim ersten Laden des Players

    Finde den Code, der beschriftet ist:

    // ### Get playlist ###

    Diese Funktion verwendet den Video Cloud-Katalog, um eine Wiedergabeliste abzurufen. Es wird nur einmal aufgerufen.

    Anwendungs-Styling

    Für dieses Beispiel ist kein CSS erforderlich.

    Plugin-Code

    Normalerweise sind beim Umwandeln des JavaScript in ein Brightcove Player-Plugin nominale Änderungen erforderlich. Eine erforderliche Änderung besteht darin, die Standardverwendung der ready() Methode durch den Code zu ersetzen, der ein Plugin definiert.

    Hier ist der sehr häufig verwendete Start-JavaScript-Code, der mit dem Player funktioniert:

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

    Sie ändern die erste Zeile, um die Standardsyntax zum Starten eines Brightcove Player-Plugins zu verwenden:

    videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

    Wie bereits erwähnt, können Sie den JavaScript-Code des Plugins im entsprechenden GitHub-Repo dieses Dokuments sehen: display-Stoßstangen.js.

    Verwenden des Plugins mit einem Spieler

    Sobald Sie die CSS- und JavaScript-Dateien des Plugins an einem für das Internet zugänglichen Ort gespeichert haben, können Sie das Plugin mit einem Player verwenden. In Studios SPIELER Modul können Sie einen Spieler auswählen, dann in der PLUGINS Fügen Sie die URLs zu den CSS- und JavaScript-Dateien hinzu und fügen Sie die hinzu Name und Optionen , wenn Optionen benötigt werden.

    Weitere Informationen zur Verwendung von Studio zum Zuweisen von Plugins zu Spielern finden Sie im Dokument Configuring Player Plugins .
