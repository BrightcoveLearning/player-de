Entwicklungs-Sequenz

Hier ist die empfohlene Entwicklungssequenz:

  1. Verwenden Sie die In-Page-Embed-Player-Implementierung, um die Funktionalität Ihres Players, Plugins und CSS zu testen (falls CSS benötigt wird)
  2. Legen Sie JavaScript und CSS des Plugins in separate Dateien zum lokalen Testen
  3. Stellen Sie den Plugin-Code und das CSS auf Ihrem Server bereit, sobald Sie Fehler herausgefunden haben
  4. Benutze Studio, um das Plugin und das CSS deinem Player hinzuzufügen
  5. Ersetzen Sie die In-Page-Embed-Player-Implementierung, wenn Sie feststellen, dass die Iframe-Implementierung besser passt (im nächsten Abschnitt beschrieben)

Weitere Informationen zu diesen Schritten finden Sie in der Schritt-für-Schritt-Anleitung: Leitfaden zur Entwicklung von Plugins

iframe oder In-Page einbetten

Bei der Entwicklung von Verbesserungen für den Brightcove Player müssen Sie entscheiden, ob der Code am besten für die Iframe- oder In-Page-Einbettungsimplementierung geeignet ist. Die Best Practice-Empfehlung besteht darin, ein Plugin zur Verwendung mit einer Iframe-Implementierung zu erstellen. Die Vorteile der Verwendung des iFrame-Players sind:

  • Keine Kollisionen mit vorhandenem JavaScript und/oder CSS
  • Reaktionsschnell
  • Der iframe erleichtert die Verwendung in Social-Media-Apps (oder wann immer das Video in andere Apps „reisen“ muss)

Obwohl die Integration des In-Page-Einbettungsplayers komplexer sein kann, planen Sie Ihren Code in dieser Implementierung manchmal. Um zu verallgemeinern, ist dieser Ansatz am besten, wenn die enthaltende Seite mit dem Spieler kommunizieren muss. Im Einzelnen sind hier einige Beispiele:

  • Der Code auf der enthaltenden Seite muss Spielerereignisse abhören und darauf reagieren
  • Der Player verwendet Stile von der enthaltenden Seite
  • Der Iframe führt dazu, dass die App-Logik fehlschlägt, wie eine Weiterleitung von der enthaltenen Seite

Selbst wenn Ihre endgültige Implementierung den Iframe-Code nicht verwendet, können Sie den In-Page-Einbettungscode trotzdem mit einem Plugin für Ihr JavaScript und einer separaten Datei für Ihr CSS verwenden. Dies kapselt Ihre Logik, sodass Sie sie problemlos in mehreren Spielern verwenden können.

Wählen Sie die Implementierung

Obwohl der hier beschriebene Spieler in einer Iframe-Einbettung funktioniert, wird auf den Player einige CSS angewendet, um die Ecken zu runden und die Steuerelemente sichtbar zu halten. Dieses Styling hängt davon ab, dass der Spieler eine ID hat und funktioniert nur mit der In-Page-Einbettung. Die In-Page-Einbettung wird hier verwendet.

Erstelle den Spieler

Gehen Sie folgendermaßen vor, um einen Player zu erstellen und ihm ein Video zuzuweisen:

  1. Erstellen Sie im Player-Modul einen neuen Player oder verwenden Sie einen vorhandenen Spieler.
  2. Bearbeiten Sie die Spielereinstellungen, um die Größe zu ändern:
    • Wählen Sie eine feste Größe
    • Wählen Sie ein benutzerdefiniertes Seitenverhältnis
    • Stellen Sie die Breite auf was immer Sie möchten ein, aber stellen Sie die Höhe auf 80 Pixel
  3. Wählen Sie im Medienmodul ein Nur-Audio-Element aus und veröffentlichen Sie es mit diesem Player.Weisen Sie dem Player Nur-Audioinhalte zu und veröffentlichen Sie ihn.
  4. Kopieren Sie den In-Page-Einbettungscode und fügen Sie ihn in eine neue HTML-Datei ein.
  5. Füge den id AudioOnly dem Player hinzu.
Weitere Informationen darüber, wie Sie diesen Einbettungscode erhalten, finden Sie im Schritt für Schritt: Brightcove Player-Dokument.

Der Einbettungscode sollte folgendermaßen aussehen:

<video-js id="audioOnly" data-video-id="4841158458001" data-account="1752604059001" data-player="rkgpS9K" data-embed="default" class="video-js" controls style="width: 480px; height: 80px;"></video-js>
<script src="https://players.brightcove.net/1752604059001/rkgpS9K_default/index.min.js"\></script>

Benutzerdefiniertes CSS hinzufügen

Fügen Sie nun ein benutzerdefiniertes CSS in einem Stilblock im Head-Bereich der Seite hinzu. Die Stile werden drei Dinge tun:

  • Entferne den Hintergrund auf dem großen Play-Button (weil er zu groß für die Höhe des Spielers ist)
  • Halten Sie die Steuerleiste während der Wiedergabe sichtbar, auch wenn der Benutzer die Maus ausschaltet
  • Runden Sie die Ecken des Players um, damit er ein wenig audioplayer-ähnlicher aussieht

Der CSS-Code, den Sie hinzufügen müssen, ist unten dargestellt:

/* keep the controls visible during playback */
#audioOnly .vjs-control-bar {
    transform: none;
    background-color: transparent;
}
/* round the corners of the player */
#audioOnly {
    border-radius: 1em;
    width: 480px;
    height: 80px;
}
/* remove the background from the big play button */
.video-js.vjs-mouse .vjs-big-play-button {
    background: none;
}
/* round the corners of the dock */
#audioOnly .vjs-dock-text {
    border-radius: 1em 1em 0 0;
}
