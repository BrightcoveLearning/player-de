    Beispiel für Brightcove Spieler: Audioplayer simuliert

    In diesem Thema lernen Sie, einen minimalistischen Brightcove-Player zu erstellen, der für die reine Audiowiedergabe ausgelegt ist. Ziel ist es, einen Player zu erstellen, der wie ein Audio-Player aussieht.

    Entwicklungs-Sequenz

    Hier ist die empfohlene Entwicklungssequenz:

    1. Verwenden Sie die In-Page Embed Player Implementierung, um die Funktionalität Ihres Players, Plugins und CSS zu testen (falls CSS benötigt wird)
    2. Platzieren Sie das JavaScript und das CSS des Plugins in separate Dateien für lokale Tests
    3. Stellen Sie den Plugin-Code und CSS auf Ihrem Server bereit, sobald Sie Fehler behoben haben
    4. Verwenden Sie Studio, um das Plugin und das CSS zu Ihrem Player hinzuzufügen
    5. Ersetzen Sie die In-Page-Embite-Player-Implementierung, wenn Sie feststellen, dass die iframe-Implementierung besser geeignet ist (im nächsten Abschnitt beschrieben)

    Einzelheiten zu diesen Schritten finden Sie im Schritt-für-Schritt: Leitfaden zur Plugin-Entwicklung

    iframe oder In-Page einbetten

    Bei der Entwicklung von Verbesserungen für den Brightcove Player müssen Sie entscheiden, ob der Code für die Implementierung von iframe oder In-Page Embed am besten geeignet ist. Die Best-Practice-Empfehlung besteht darin, ein Plugin für die Verwendung mit einer iframe-Implementierung zu erstellen. Die Vorteile der Verwendung des iFrame-Players sind:

    • Keine Kollisionen mit bestehendem JavaScript und/oder CSS
    • Automatisch ansprechbar
    • Der iframe erleichtert die Verwendung in Social-Media-Apps (oder wann immer das Video in andere Apps „reisen“ muss)

    Obwohl die Integration des In-Page-Embed Players komplexer sein kann, gibt es Zeiten, in denen Sie Ihren Code für diese Implementierung planen. Um zu verallgemeinern, ist dieser Ansatz am besten, wenn die enthaltene Seite mit dem Spieler kommunizieren muss. Konkret sind hier einige Beispiele:

    • Der Code auf der enthaltenen Seite muss Spielereignisse abhören und darauf reagieren
    • Der Spieler verwendet Stile von der enthaltenden Seite
    • Der iframe führt dazu, dass die App-Logik fehlschlägt, wie eine Weiterleitung von der enthaltenden Seite

    Selbst wenn Ihre endgültige Implementierung den Iframe-Code nicht verwendet, können Sie den In-Page-Einbettungscode immer noch mit einem Plugin für Ihr JavaScript und einer separaten Datei für Ihr CSS verwenden. Dies kapselt Ihre Logik, sodass Sie sie problemlos in mehreren Spielern verwenden können.

    Wählen Sie eine Implementierung

    Obwohl der hier beschriebene Player in einer Iframe-Einbettung funktioniert, wird auf den Player CSS angewendet, um die Ecken abzurunden und die Steuerelemente sichtbar zu halten. Dieses Styling hängt davon ab, ob der Player eine ID hat, und funktioniert nur mit der Einbettung in die Seite. Hier wird die In-Page-Einbettung verwendet.

    Der Beispiel-Player befindet sich in der JSFiddle unten. Klicken Sie auf die Registerkarte CSS, um das auf den Player angewendete CSS anzuzeigen.

    So funktioniert es

    In den folgenden Abschnitten werden der im Beispiel verwendete Player und der CSS-Code erläutert.

    Dieser Player benötigt auch Nur-Audio-Inhalte. Sehen Audio-Inhalt Erfahren Sie, wie Sie Audio-Inhalte zur Video Cloud hinzufügen.

    Dieser Player benötigt außerdem Nur-Audio-Inhalte in Form einer MP4-Datei ohne Videospur. Ein Poster für das Audio ist optional, kann jedoch zum Anzeigen des Audiotitels verwendet werden, wie im Beispiel-Player hier gezeigt.

    Erstelle den Spieler

    Gehen Sie folgendermaßen vor, um einen Player zu erstellen und ihm ein Video zuzuweisen:

    1. Erstellen Sie im Spieler-Modul einen neuen Spieler oder verwenden Sie einen vorhandenen.
    2. Bearbeiten Sie die Player-Einstellungen, um die Größe zu ändern:
      • Wählen Sie eine feste Größe
      • Wählen Sie ein benutzerdefiniertes Seitenverhältnis
      • Stellen Sie die Breite nach Belieben ein, aber stellen Sie die ein Höhe zu 80 Pixel
    3. In dem Medien Modul, wählen Sie eine Nur-Audio-Artikel und veröffentlichen Sie es mit diesem Player.Weisen Sie dem Player Nur-Audio-Inhalte zu und veröffentlichen Sie sie.
    4. Kopieren Sie den In-Page-Einbettungscode und fügen Sie ihn in eine neue HTML-Datei ein.
    5. Ergänzen Sie die id Nur Audio an den Spieler.
    Einzelheiten dazu, wie Sie diesen Einbettungscode erhalten, finden Sie im Schritt für Schritt: Brightcove Player-Dokument .

    Der Einbettungscode sollte ähnlich wie folgt aussehen:

    <video-js id="audioOnly" data-video-id="4841158458001" data-account="1752604059001" data-player="rkgpS9K" data-embed="default" class="video-js" controls style="width: 480px; height: 80px;"></video-js>
<script src="https://players.brightcove.net/1752604059001/rkgpS9K_default/index.min.js"\></script>

    <video-js id="audioOnly" data-account="3676484086001" data-player="B1doF3j" data-embed="default" class="video-js" controls style="width: 480px; height: 80px;"></video-js>
<script src="https://players.brightcove.net/3676484086001/B1doF3j_default/index.min.js"></script>

    Fügen Sie benutzerdefiniertes CSS hinzu

    Fügen Sie nun in einem Stilblock im Kopfbereich der Seite benutzerdefiniertes CSS hinzu. Die Stile werden drei Dinge tun:

    • Entfernen Sie den Hintergrund auf der großen Wiedergabetaste (weil er für die Größe des Players zu groß ist)
    • Lassen Sie die Steuerleiste während der Wiedergabe sichtbar, auch wenn der Benutzer die Maus ausschaltet
    • Runden Sie die Ecken des Players ab, damit er etwas mehr wie ein Audio-Player aussieht

    Der CSS-Code, den Sie hinzufügen müssen, wird unten angezeigt:

    /* keep the controls visible during playback */
#audioOnly .vjs-control-bar {
    transform: none;
    background-color: transparent;
}
/* round the corners of the player */
#audioOnly {
    border-radius: 1em;
    width: 480px;
    height: 80px;
}
/* remove the background from the big play button */
.video-js.vjs-mouse .vjs-big-play-button {
    background: none;
}
/* round the corners of the dock */
#audioOnly .vjs-dock-text {
    border-radius: 1em 1em 0 0;
}
