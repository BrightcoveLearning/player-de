    Beispiel für Brightcove Spieler: Wiedergaberate Einsteller

    In diesem Thema erfahren Sie, wie Sie der Brightcove Player-Steuerleiste eine Schaltfläche für die Wiedergaberate hinzufügen und mithilfe der Middleware die aktuelle Zeit und Videodauer richtig anzeigen, wenn Sie zwischen den Wiedergabegeschwindigkeiten wechseln.

    Player-Beispiel

    Dieses Beispiel und der zugehörige Code dienen als Leitfaden für Ihre Produktionsentwicklung. Der Code ist weder für den produktiven Einsatz zertifiziert noch offiziell unterstützt.

    Starten Sie die Videowiedergabe und wählen Sie die Wiedergaberate in der Steuerungsleiste aus (Standard ist 1x). Wähle eine andere Rate, um zu sehen, wie sich die Geschwindigkeit der Videowiedergabe ändert Beachten Sie, dass beim Ändern der Wiedergaberate die angezeigte Dauer basierend auf der ausgewählten Rate neu berechnet wird.

    Siehe den Stift Einsteller für die Wiedergaberate von Brightcove Learning Services ( bcls1969 ) auf CodePen.

    Quellcode

    Schau den Komplettlösung auf GitHub.

    Verwenden des CodePen

    Hier sind einige Tipps, um das obige CodePen effektiv zu nutzen:

    • Schalten Sie die tatsächliche Anzeige des Players ein, indem Sie auf die Result Schaltfläche klicken.
    • Klicken Sie auf die HTML/CSS/JS Schaltflächen, um EINEN der Codetypen anzuzeigen.
    • Später in diesem Dokument werden die in der Anwendung verwendete Logik, der Ablauf und das Styling in den Abschnitten Player/HTML-Konfiguration, Anwendungsfluss und Anwendungsstyling besprochen. Der beste Weg, um den Informationen in diesen Abschnitten zu folgen, ist:
      1. Klicken Sie auf die EDIT ON CODEPEN Schaltfläche im CodePen und halten Sie den Code in einem Browser/Browser-Tab zur Verfügung.
      2. Passen Sie in CodePen an, welcher Code angezeigt werden soll. Sie können die Breite verschiedener Codeabschnitte in CodePen ändern.
      3. Zeigen Sie die Abschnitte Player/HTML-Konfiguration, Anwendungsfluss und/oder Anwendungsstyling in einer anderen Browser-/Browserregisterkarte an. Sie können nun den Codeerklärungen folgen und gleichzeitig den Code anzeigen.

    Entwicklungs-Sequenz

    Hier ist die empfohlene Entwicklungssequenz:

    1. Verwenden Sie die In-Page Embed Player Implementierung, um die Funktionalität Ihres Players, Plugins und CSS zu testen (falls CSS benötigt wird)
    2. Platzieren Sie das JavaScript und das CSS des Plugins in separate Dateien für lokale Tests
    3. Stellen Sie den Plugin-Code und CSS auf Ihrem Server bereit, sobald Sie Fehler behoben haben
    4. Verwenden Sie Studio, um das Plugin und das CSS zu Ihrem Player hinzuzufügen
    5. Ersetzen Sie die In-Page-Embite-Player-Implementierung, wenn Sie feststellen, dass die iframe-Implementierung besser geeignet ist (im nächsten Abschnitt beschrieben)

    Einzelheiten zu diesen Schritten finden Sie im Schritt-für-Schritt: Leitfaden zur Plugin-Entwicklung

    iframe oder In-Page einbetten

    Bei der Entwicklung von Verbesserungen für den Brightcove Player müssen Sie entscheiden, ob der Code für die Implementierung von iframe oder In-Page Embed am besten geeignet ist. Die Best-Practice-Empfehlung besteht darin, ein Plugin für die Verwendung mit einer iframe-Implementierung zu erstellen. Die Vorteile der Verwendung des iFrame-Players sind:

    • Keine Kollisionen mit bestehendem JavaScript und/oder CSS
    • Automatisch ansprechbar
    • Der iframe erleichtert die Verwendung in Social-Media-Apps (oder wann immer das Video in andere Apps „reisen“ muss)

    Obwohl die Integration des In-Page-Embed Players komplexer sein kann, gibt es Zeiten, in denen Sie Ihren Code für diese Implementierung planen. Um zu verallgemeinern, ist dieser Ansatz am besten, wenn die enthaltene Seite mit dem Spieler kommunizieren muss. Konkret sind hier einige Beispiele:

    • Der Code auf der enthaltenen Seite muss Spielereignisse abhören und darauf reagieren
    • Der Spieler verwendet Stile von der enthaltenden Seite
    • Der iframe führt dazu, dass die App-Logik fehlschlägt, wie eine Weiterleitung von der enthaltenden Seite

    Selbst wenn Ihre endgültige Implementierung den Iframe-Code nicht verwendet, können Sie den In-Page-Einbettungscode immer noch mit einem Plugin für Ihr JavaScript und einer separaten Datei für Ihr CSS verwenden. Dies kapselt Ihre Logik, sodass Sie sie problemlos in mehreren Spielern verwenden können.

    Middleware-Funktionen

    Dieses Beispiel verwendet Video.js Middleware, um die Codelösung zu implementieren. Auf einem sehr hohen Niveau sind die Anwendungsfälle, in denen Middleware am meisten von Wert ist, wenn Sie das Kernverhalten des Spielers abfangen und verändern möchten, z. B. das Einstellen der Quelle oder der aktuellen Uhrzeit. Video.js Middleware im Allgemeinen wird in diesem Dokument nicht ausführlich behandelt, aber der tatsächlich für die Middleware-Funktion verwendete Code wird erläutert. Hier sind Dokumente aus den Quellen von Video.js, die Details von Middleware ausführen:

    • Middleware (aus dem Dokumentationssatz Video.js)
    • Funktion im Rampenlicht: Middleware (aus dem Video.js Blog)
    • videojs-playbackrate-einstellgerät (eine Beispiel-App)

    Implementierung von Middleware-Funktionen mit Brightcove Player

    Die einfachste Möglichkeit, eine Middleware-Funktion zu verwenden, besteht darin:

    1. Schreiben Sie die Middleware-Funktion.
    2. Platzieren Sie die Middleware-Funktion entweder auf der Seite mit dem Player in einem script Tag oder schließen Sie die Middleware-Funktion ein, indem Sie das src Attribut mit einem script Tag verwenden. Beide Implementierungen werden in diesem Dokument gezeigt.
    3. Registrieren Sie die Middleware-Funktion mit dem Player mit der use() Methode, wie hier gezeigt: 
      videojs.use('mimeType', functionName);

      Für den Mimetyp können Sie einen Standard-Video-Mimetyp wie Video/MP4 verwenden oder ein * verwenden, wenn Sie möchten, dass die Middleware-Funktion auf alle Mimetypen reagiert.

      Plugins und Middleware können so eingerichtet werden, dass sie sich automatisch registrieren, wenn ihre Skripte enthalten sind. In diesem Fall müssten Sie das nicht verwenden videojs.use() Methode.

    Ablauf der Anwendung

    Die grundlegende Logik hinter dieser Anwendung lautet:

    • Definieren Sie die Middleware-Funktionen
    • Registriere die Middleware beim Player
    • Definieren Sie die Wiedergaberate
    • Initialisieren Sie die Wiedergaberate Taste

    Definieren Sie die Middleware-Funktionen

    Finde den Code, der beschriftet ist:

    // +++ Define the middleware functions +++

    Erstellen Sie die folgenden Middleware-Funktionen:

    1. Erstellen Sie eine unterstützende Middleware-Funktion, die die Start- und Endzeiten der Wiedergabe basierend auf der ausgewählten Wiedergaberate neu berechnet.

    2. Erstellen Sie eine Middleware-Funktion, die die Startzeit, die Dauer und sogar die aktuelle Zeit bei der Suche aktualisiert.

      Die grundlegende Syntax zum Erstellen einer Middleware-Funktion lautet wie folgt:

      const playbackrateAdjuster = function(player) {
   return {
   ...
 };

      Beachten Sie, dass das player Objekt an die Funktion übergeben wird.

      Diese Middleware-Funktion führt Folgendes aus:

      1. Hören Sie auf das Ratechange-Ereignis:: Wenn die Wiedergaberate geändert wird, lösen Sie die aus durationchange und timeupdate Veranstaltungen.

      2. Implementieren Sie die Methode setSource ():: Dies ist eine erforderliche Methode für jede Middleware-Funktion. Durch die Verwendung der angezeigten Syntax wird die Middleware unabhängig vom Videotyp immer ausgewählt: 
        setSource(srcObj, next) {
  next(null, srcObj);
},

      3. Überschreiben Sie Methoden mit der aktuellen Wiedergaberate:: Überschreiben Sie die Methoden für duration , currentTime , buffered , seekable und played mit Werten, die unter Verwendung der aktuellen Wiedergaberate berechnet wurden.

    Registriere die Middleware beim Player

    Finde den Code, der beschriftet ist:

    // +++ Register the middleware with the player +++

    Registrieren Sie die Middleware-Funktion beim Player über das use() Methode.

    Definieren Sie die Wiedergaberate

    Finde den Code, der beschriftet ist:

    // +++ Define the playback rate options +++

    Definieren Sie die Wiedergaberaten, die in der Menüschaltfläche Wiedergaberate angezeigt werden sollen.

    Initialisieren Sie die Wiedergaberate Taste

    Finde den Code, der beschriftet ist:

    // +++ Initialize the playback rate button +++

    Fügen Sie die Menüschaltfläche für die Wiedergaberate zur Steuerleiste des Players hinzu und initialisieren Sie sie mit den im Optionsobjekt angegebenen Werten.

    Anwendungs-Styling

    Es gibt keine zusätzlichen Stile für dieses Beispiel.

    Verwenden der Middleware-Funktion

    Sobald die Middleware-Funktion geschrieben wurde, möchten Sie sie verwenden. Zwei Wege wurden bereits erwähnt und werden jetzt detailliert beschrieben.

    Code auf derselben Seite wie Spieler

    Hier definieren Sie die Middleware-Funktion auf derselben Seite wie der Player und registrieren sie dann. Das Spielerbeispiel im obigen Codepen verwendet diese Methode.

    Code in einer separaten JavaScript-Datei

    Im folgenden Code funktioniert das JavaScript für die Middleware und das use() Die Methode zur Registrierung ist einfach auf der Seite enthalten.

    <video-js id="myPlayerID"
	  data-video-id="5604123528001"
	  data-account="1752604059001"
	  data-player="default"
	  data-embed="default"
	  data-application-id
	  class="video-js"
	  width="640" height="360"
	  controls muted></video>
	<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

  <!-- Include the rate adjuster middleware functions -->
  <script src="rate-adjuster.js"></script>

  <script type="text/javascript">
  	videojs.getPlayer("myPlayerID").ready(function(){
  		// When the player is ready, get a reference to it
  		var myPlayer = this;
  		console.log("myPlayer",myPlayer)
  		// Define the playback rate options
  		options = {"playbackRates":[0.5, 1, 1.5, 2, 4]};
  		// Initialize the playback rate button
  		myPlayer.controlBar.playbackRateMenuButton = myPlayer.controlBar.addChild('PlaybackRateMenuButton', {
      	playbackRates: options.playbackRates
  	  });
    });
  </script>

    In diesem Beispiel wird davon ausgegangen, dass Sie die JavaScript-Datei auf Ihrem eigenen Server hosten. Den vollständigen Code finden Sie in der Plugin-Bereich im GitHub Repo.

    Das videojs-playbackrate-adjuster Middleware wird veröffentlicht npm und verfügbar, um in Ihr Projekt aufgenommen zu werden: //unpkg.com/videojs-playbackrate-adjuster@1.0.0/dist/browser/videojs-playbackrate-adjuster.min.js

    Notiere dass der unpkg.com Website ist kostenlos und Open Source. Es garantiert keine Verfügbarkeit und Infrastrukturunterstützung.
