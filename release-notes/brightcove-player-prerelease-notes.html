Beachten Sie, dass alle 1-2 Wochen vorveröffentlichte Versionen zur aktuellen Version werden. Zu diesem Zeitpunkt werden alle nicht gesperrten Spieler auf die aktuelle Version aktualisiert. Wenn Sie Ihren Player aus irgendeinem Grund für eine bestimmte Version sperren möchten, lesen Sie das Abmelden von automatischen Player-Updates (externe Website).
 
 

Probiere eine neue Version

Wenn eine Vorabversion verfügbar ist, ist es am einfachsten, eine Vorversions-Version auszuprobieren, zum PLAYER-Modul von Studio zu wechseln, auf den Player zu klicken, den Sie ändern möchten, und dann die Player-Informationen zu bearbeiten . Wählen Sie mit der Versionsauswahl die neueste Vorversion aus:

Sie können einen Player auf die Vorabversion aktualisieren, indem Sie eine curl-Anweisung verwenden. Eine kurze Übersicht über Curl finden Sie oben im Schritt-für-Schritt: Spielerverwaltung (externe Website). Im Dokument gibt es einen bestimmten Abschnitt, wie Sie Ihren Player aktualisieren können.

Hier ist der curl-Befehl zum Aktualisieren Ihres Players, bei dem Sie entsprechende Werte für YOUR_EMAIL, YOUR_ACCOUNT_ID und YOUR_PLAYER_ID ersetzen . 

curl \
  --header "Content-Type: application/json" \
  --user YOUR_EMAIL \
  --request PATCH \
  --data '{
    "player": {
    "template": {
      "version": "6.57.0"
      }
    }
  }' \
  https://players.api.brightcove.com/v2/accounts/YOUR_ACCOUNT_ID/players/YOUR_PLAYER_ID/configuration

In den meisten Fällen möchten Sie den Player veröffentlichen, was mit der folgenden curl-Anweisung geschehen kann. Auch hier müssen Sie die richtigen Werte ersetzen, wo angegeben. 

curl \
  --header "Content-Type: application/json" \
  --user YOUR_EMAIL  \
  --request POST \
  https://players.api.brightcove.com/v2/accounts/YOUR_ACCOUNT/players/YOUR_PLAYER_ID/publish
