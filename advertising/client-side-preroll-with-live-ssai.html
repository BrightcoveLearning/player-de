Überblick

Wenn Sie Livestreams mit serverseitiger Anzeigeneinfügung (SSAI) mit Brightcove Player liefern, können Sie eine clientseitige Preroll-Anzeige einfügen. Für clientseitige Anzeigen unterstützt diese Funktion IMA-Anzeigen.

Für diese Funktion ist Folgendes erforderlich:

  • IMA3-Plugin v3.5.0+
  • SSAI-Plugin v1.11.0+
  • Brightcove Player Version 6.44.0+

Vorbehalte

  • Kunden, die diese Funktion verwenden, müssen Dynamic Delivery verwenden.
  • Es werden nur IMA- und SSAI-Plugins unterstützt (kein FreeWheel).
  • Derzeit unterstützt diese Funktion nur einen einzelnen Player auf der Seite.

Erste Schritte

Gehen Sie folgendermaßen vor, um eine clientseitige IMA-Peroll-Anzeige mit einem Live-SSAI-Stream abzuspielen:

  1. Erstellen Sie einen SSAI-fähigen Livestream
  2. Erstelle einen Brightcove-Spieler
  3. Implementieren Sie clientseitigen Preroll mit Studio

Erstellen Sie einen SSAI-fähigen Livestream

Das Live-Modul unterstützt serverseitige Anzeigeneinfügung (SSAI), sodass serverseitige Anzeigen während eines Live-Streams angefordert und angezeigt werden können. Um Ihren Livestream zu erstellen, lesen Sie Folgendes:

  • Implementierung serverseitiger Anzeigen im Live-Modul

Erstelle einen Brightcove-Spieler

Erstellen Sie mit Video Cloud Studio einen neuen Brightcove-Player. Weitere Informationen finden Sie im Schnellstart: Erstellen und Stylen eines Player-Dokuments.

Sie müssen einen Brightcove Player Version 6.44.0+ verwenden.

Implementierung von clientseitigem Preroll mit Studio

Der einfachste Weg, Ihren Player für Auto-Failover-Anzeigen zu konfigurieren, ist Video Cloud Studio. Sobald Sie eine Anzeigenkonfiguration und einen Player erstellt haben, können Sie den Player wie folgt für das automatische Failover konfigurieren:

  1. Öffnen Sie das PLAYERS-Modul und suchen Sie den Player, dem Sie Werbefunktionen hinzufügen möchten.
  2. Wähle den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
  3. Wählen Sie im linken Navigationsmenü Werbung aus.
  4. Aktivieren Sie das Kontrollkästchen Clientseitig (IMA) aktivieren.
  5. Fügen Sie die URL für Ihr IMA-Anzeigen-Tag ein. In diesem Beispiel verwenden wir die Beispiel-Ad-Tag-URL.

    Weitere Informationen zu den Eigenschaften der Player-Werbung finden Sie im Dokument Player-Werbung mit dem Player-Modul konfigurieren.

  6. Aktivieren Sie das Kontrollkästchen Serverseitig (SSAI) aktivieren.

    Die serverseitige Option wird nur angezeigt, wenn Ihr Konto für SSAI aktiviert wurde.
  7. Wählen Sie im Dropdown-Menü Konfiguration auswählen die Anzeigenkonfiguration aus, die Sie diesem Player zuordnen möchten.
  8. Wenn Sie möchten, dass Overlays über Ihre Anzeigen angezeigt werden, aktivieren Sie das Kontrollkästchen Overlays für Anzeigeninformationen aktivieren. Dies beinhaltet „Mehr erfahren“ und Overlays zur Anzeigenzählung.
    Overlays werden erst angezeigt, wenn Sie sie hier aktiviert und in Ihrem VAST Ad-Tag konfiguriert haben.
  9. Drücke den Speichern Taste.
  10. Wählen Sie im linken Navigationsmenü den JSON-Editor aus.

  11. Scrollen Sie im JSON-Editor nach unten, bis Sie die ad_failover: true Eigenschaft sehen.

  12. Ersetzen Sie die ad_failover: true Eigenschaft durch Folgendes: 
    "ima_preroll_with_ssai": true
  13. Ihr JSON-Editor sollte folgendermaßen aussehen:
  14. Klicken Sie auf die Schaltfläche Speichern.
  15. Um den Player zu veröffentlichen, wählen Sie Veröffentlichen & Einbetten > Änderungen veröffentlichen aus.
  16. Jetzt können Sie Ihr Live-Event veröffentlichen. Weitere Informationen finden Sie im Dokument Serverseitige Anzeigen implementieren im Live-Modul.
