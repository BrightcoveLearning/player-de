    Brightcove Spielerlader

    In diesem Thema erfahren Sie, wie Sie mit dem Brightcove Player Loader einen Brightcove Player mit modernen Build-Tools laden. Diese Bibliothek wird als Werkzeug für technischere Kunden angesehen.

    Player Loader Übersicht

    Der Brightcove Player Loader ist ein Open Source-Tool und wird von Brightcove NICHT offiziell unterstützt. Außerdem unterstützt diese Bibliothek nur Brightcove Players v6.11.0 und höher.

    Für Benutzer von Brightcove Player, die weder die In-Page-Einbettungs- noch die Iframe-Player-Implementierungen kopieren und einfügen möchten, ist dieses Modul in ihrer Webanwendung enthalten und bietet eine Möglichkeit, Brightcove Player zu verwenden, ohne Integrationscode zum Herunterladen schreiben zu müssen Spieler und binden sie ein. Dieses Tool löst das Problem mit einer Brightcove-Bibliothek, die jeden veröffentlichten Brightcove-Player herunterladen und in das DOM einbetten kann.

    Diese Bibliothek unterstützt gängige immergrüne Browser, Chrome, Firefox, Edge, Safari und IE11. Frühere Versionen von IE werden NICHT unterstützt.

    Dieses Dokument enthält im Wesentlichen Beispielanwendungen für den Brightcove Player Loader. Ausführliche Informationen finden Sie in der Bibliothek Brightcove / Player-Loader GitHub Repo.

    Es gibt ein Webpack, das mit Brightcove Player verwendet werden kann. Details dazu finden Sie in der Player-Loader-Webpack-Plugin GitHub Repo.

    Installation

    Die Bibliothek wird unter Verwendung von NPM wie folgt installiert:

    npm install --save @brightcove/player-loader

    Einschließlich der Bibliothek

    Weitere Methoden zum Einschließen der Bibliothek für Ihre Verwendung finden Sie in der GitHub Repo für die Bibliothek.

    Implementierung mit einem Versprechen

    In diesem und den folgenden Beispielen werden Parameter aus der Bibliothek verwendet. Eine Tabelle mit vollständiger Auflistung der Parameter befindet sich in der Verfügbare Parameter Abschnitt unten.

    Sie können eine verwenden JavaScript-Versprechen mit der Bibliothek. Versprechen sind nicht erforderlich, damit diese Bibliothek funktioniert, aber sie werden empfohlen. Standardmäßig sucht diese Bibliothek nach einem globalen Versprechen. Sie können jedoch explizit eine Promise-Implementierung über das bereitstellen Promise Parameter, der später in diesem Dokument beschrieben wird.

    Das Folgende zeigt eine Implementierung der Bibliothek unter Verwendung eines Versprechens. Ein Schlüsselkonzept ist, dass Sie mithilfe von einen Verweis auf den Player erhalten var myPlayer = success.ref; in dem then Abschnitt der Behandlung des Versprechens:

    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js {
      height: 344px;
      width: 610px;
    }
  </style>

</head>

<body>
  <script src="brightcove-player-loader.min.js"></script>

  <div id="theDiv">
  </div>

  <script>
    brightcovePlayerLoader({
      refNode: document.querySelector('#theDiv'),
      accountId: '1752604059001',
      playerId: 'wHBRh9M3o',
      videoId: '4607357817001'
    })
    .then(function(success) {
      var myPlayer = success.ref;
      console.log('success', success);
      myPlayer.on('loadedmetadata',function(){
        myPlayer.muted(true);
        myPlayer.play();
      });
    })
    .catch(function(error) {
      console.log('error', error);
    })
  </script>

</body>

</html>

    Implementierung mit Rückrufen

    Sie können den Player Loader auch mithilfe von Rückrufen über das implementieren onSuccess und onFailure Rückruffunktionen.

    Das Folgende zeigt eine Implementierung der Bibliothek mithilfe von Rückrufen. Ein Schlüsselkonzept ist, dass Sie mithilfe von einen Verweis auf den Player erhalten var myPlayer = success.ref; in dem onSuccess Rückruffunktion:

    <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js {
      height: 344px;
      width: 610px;
    }
  </style>

</head>

<body>
  <script src="brightcove-player-loader.min.js"></script>

  <div id="theDiv">
  </div>

  <script>
  brightcovePlayerLoader({
    refNode: document.querySelector('#theDiv'),
    accountId: '1752604059001',
    playerId: 'wHBRh9M3o',
    videoId: '4607357817001',
    onSuccess: function(success) {
      var myPlayer = success.ref;
      console.log('success', success);
      myPlayer.on('loadedmetadata',function(){
        myPlayer.muted(true);
        myPlayer.play();
      });
    },
    onFailure(error) {
      console.log('error', error);
    }
  });
  </script>

</body>

</html>

    Verfügbare Parameter

    Name: accountId

    Datentyp: string | number

    Beschreibung:
    Eine Video Cloud-Konto-ID.

    Name: applicationId

    Datentyp: string

    Beschreibung:
    Die Anwendungs-ID, die auf die generierte Einbettung angewendet werden soll.

    Name: catalogSearch

    Datentyp: Zeichenfolge | Objekt

    Beschreibung:
    Eine durchzuführende Video Cloud-Katalogsuche. Dies kann eine einfache Zeichenfolgensuche oder ein Objekt sein, das mit dem übereinstimmt Katalog getSearch() Methode. Wenn ein Nicht-String-Wert angegeben wird, der nicht als JSON serialisierbar ist, wird dieser Parameter ignoriert.

    Name: catalogSequence

    Daten-Art: Array | Objekt


    Beschreibung:
    Eine durchzuführende Video Cloud Catalog-Sequenz. Siehe die Katalog getLazySequence() Methode für mehr Informationen. Wenn ein Nicht-String-Wert angegeben wird, der nicht als JSON serialisierbar ist, wird dieser Parameter ignoriert.

    Name: embedId

    Datentyp:Zeichenfolge

    Beschreibung:
    Die Brightcove Player-Einbettungs-ID für den Player. Der Standardwert ist 'default'. Der Standardwert ist für die meisten Benutzer korrekt.

    Name: embedOptions

    Daten-Art: Objekt

    Beschreibung:
    Wird verwendet, um bestimmte Optionen für die Einbettungsgenerierung bereitzustellen. Hierzu zählt Folgendes:

    embedOptions.pip Boolesch Wenn true , wird die Einbettung in a wickeln <div class="vjs-pip-container"> Element. Dies sollte verwendet werden, wenn Sie Unterstützung für die benötigen Brightcove Picture-in-Picture-Plugin. Der Standardwert ist false.
    embedOptions.playlist Boolesch Wenn true wird ein hinzufügen <div class="vjs-playlist"> Element nach dem Einbetten. Dies sollte verwendet werden, wenn Sie Unterstützung für die benötigen Brightcove Playlist UI Plugin. Der Standardwert ist false.
    embedOptions.responsive boolean | Objekt Wird verwendet, um den Einbettungscode anzupassen, um mithilfe des intrinsic ratio Wrapper-Ansatz. Wenn true, wird ein reaktionsfähiger Einbettungscode mit einem Seitenverhältnis von 16: 9 erstellt, der den Container füllt. Der Standardwert ist false.
    Ein Objekt kann bereitgestellt werden, um dies mit den folgenden Untereigenschaften anzupassen:
    • aspectRatio:: Eine Zeichenfolge, mit der das Seitenverhältnis auf einen anderen Wert als 16: 9 angepasst wird (z. B. '4: 3').
    • maxWidth:: Eine Zeichenfolge, mit der die maximale Breite des Players begrenzt wird. Dies sollte CSS-Einheiten verwenden, z. B. Pixel (z. B. '960px').
    embedOptions.unminified Boolesch Wenn true wird die nicht minimierte Version des Players verwenden. Dies kann für Debugging-Zwecke nützlich sein, kostet jedoch einen größeren Player-Download. Nicht für die Produktion empfohlen! Der Standardwert ist false.
    Name: embedType

    Datentyp: string

    Beschreibung:
    Der zu erzeugende Einbettungscodetyp. Der Wert dieses Parameters muss einer der folgenden sein:

    • 'in-page' oder brightcovePlayerLoader.EMBED_TYPE_IN_PAGE:: Dies wird auch als erweiterter Einbettungscode bezeichnet und fügt den Player direkt in die Webseite der obersten Ebene ein.
    • 'iframe' oder brightcovePlayerLoader.EMBED_TYPE_IFRAME:: Dies wird auch als grundlegender Einbettungscode bezeichnet und fügt den Player als <iframe> -Element ein.
    Der Standardwert ist 'in-page'.
    Name: onEmbedCreated

    Daten-Art: Funktion (Element)

    Beschreibung:
    Ein Rückruf zum Anpassen des Einbettungselements (entweder) video-js Element oder ein iframe Element), bevor es in das DOM eingefügt oder als Ergebnis von angepasst wird embedOptions und bevor der Player heruntergeladen und initialisiert wird. Das Einbettungselement kann mutiert sein, oder wenn dieser Rückruf ein Element zurückgibt, wird dieses Element als Einbettungselement verwendet. Mögliche Anwendungsfälle sind das Hinzufügen / Entfernen von Attributen oder das Hinzufügen von untergeordneten Elementen wie Quellen oder Spuren.

    Name: onFailure

    Daten-Art: Funktion (Fehler)

    Beschreibung:
    Eine Rückruffunktion, die die Behandlung von Fehlern ermöglicht, wenn Promise ist nicht verfügbar oder nicht erwünscht. Durch das Übergeben dieser Funktion wird verhindert, dass a Promise zurückgegeben wird. Es wird eine Single Error Objekt als Argument. Der Rückgabewert dieser Funktion wird ignoriert.

    Name: onSuccess

    Daten-Art: Funktion (Objekt)

    Beschreibung:
    Eine Rückruffunktion, mit der Erfolge behandelt werden können, wenn Promise ist nicht verfügbar oder nicht erwünscht. Durch das Übergeben dieser Funktion wird verhindert, dass a Promise zurückgegeben wird. Es wird eine Single Success Objekt als Argument. Der Rückgabewert dieser Funktion wird ignoriert.

    Name: options

    Daten-Art: Objekt

    Beschreibung:
    Ein Video.js Optionen Objekt während des Player-Erstellungsprozesses zu bestehen. Diese Optionen haben Vorrang vor allen Einstellungen, die in der Brightcove Player-Konfiguration angegeben sind. Dieser Parameter kann nicht mit iframe-Einbettungen verwendet werden.

    Name: playerId

    Datentyp: string

    Beschreibung:
    Eine Brightcove Player-Spieler-ID. Der Standardwert ist 'default'.

    Name: playlistId

    Datentyp: string | number

    Beschreibung:
    Eine Video Cloud-Wiedergabelisten-ID oder eine Wiedergabelisten-Referenz-ID.

    Name: playlistVideoId

    Datentyp: string | number

    Beschreibung:

    Eine Video Cloud-Video-ID, die in der resultierenden Wiedergabeliste von angegeben wird playlistId. Dieser Parameter wird ignoriert, wenn playlistId wird vermisst.

    Name: Promise

    Daten-Art: Funktion (Funktion)

    Beschreibung:
    Wird verwendet, um explizit a bereitzustellen Promise Implementierung. Wenn angegeben, wird dies anstelle eines globalen verwendet Promise. Der Standardwert ist window.Promise

    Name: refNode

    Daten-Art: Element | Zeichenfolge

    Beschreibung:
    Required
    Das DOM-Element, in das der Player eingebettet wird. Wenn es nicht als DOM-Element bereitgestellt wird, kann es als Zeichenfolge bereitgestellt werden, an die übergeben wird document.querySelector.

    Name: refNodeInsert

    Datentyp: string

    Beschreibung:
    Die Art und Weise, in der der Player relativ zum Referenz-DOM-Element eingefügt wird (angegeben durch refNode). Der Wert dieses Parameters muss einer der folgenden sein:

    • 'append' oder brightcovePlayerLoader.REF_NODE_INSERT_APPEND:: Der Spieler ist das letzte Kind des Referenzknotens.
    • 'prepend' oder brightcovePlayerLoader.REF_NODE_INSERT_PREPEND:: Der Spieler ist das erste Kind des Referenzknotens.
    • 'before' oder brightcovePlayerLoader.REF_NODE_INSERT_BEFORE:: Der Spieler ist das vorherige Geschwister des Referenzknotens.
    • 'after' oder brightcovePlayerLoader.REF_NODE_INSERT_AFTER:: Der Spieler ist das nächste Geschwister nach dem Referenzknoten.
    • 'replace' oder brightcovePlayerLoader.REF_NODE_INSERT_REPLACE:: Der Referenzknoten wird entfernt und vom Player ersetzt.

    Name: videoId

    Datentyp: string | number

    Beschreibung:
    Eine Video Cloud-Video-ID oder eine Video-Referenz-ID.
