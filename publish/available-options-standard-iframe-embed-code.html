    Verfügbare Optionen für den Standard (iframe) -Einbettungscode

    In diesem Thema erfahren Sie mehr über die Verwendung und Konfiguration von iframe Code einbetten. Diese Version der Player-Implementierung wird auch als bezeichnet Standard Spielercode.

    Einführung

    Das iframe Der Player-Implementierungscode erleichtert die Verwendung des Players als einzelne, enthaltene Einheit. Dies ist häufig für Social Media-Anwendungen und das Teilen erforderlich.

    Verwendung

    Hinweis: Wenn Sie ein Brightcove Player-Kunde sind, werden Sie das nicht verwenden videoId Abfrageparameter, um Ihrem Player ein Video zuzuweisen. Sie würden das Video entweder im Players-Modul zuweisen oder curl verwenden.

    So verwenden Sie die iframe Code folgen Sie diesen Schritten:

    1. Verwenden Sie das Modul SPIELER , um einen Spieler zu erstellen.
    2. Gehe zum MEDIA-Modul und veröffentliche ein Video mit dem neu erstellten Player.
    3. Kopiere das Standard Code einbetten. Der HTML-Code ähnelt dem folgenden: 
            <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
    4. Kopieren Sie den HTML-Code aus dem Browser und fügen Sie ihn in eine vollständige HTML-Seite ein. body
    5. Durchsuchen Sie die HTML-Seite, um zu sehen, wie der Player funktioniert.

    Parameter abfragen

    Für die stehen zahlreiche Abfrageparameter zur Verfügung iframe Tag, um zusätzliche Informationen darüber bereitzustellen, wie sich ein Spieler verhalten soll. In der folgenden Tabelle sind die verfügbaren Parameter aufgeführt.

    Bei Verwendung von Abfrageparametern, die nur wahr oder falsch sein können (Datentyp) Boolescher Wert fügen Sie einfach den Abfrageparameter ein, um ihn zu aktivieren, oder schließen ihn aus, um ihn auszuschalten. Ein Beispiel wird hier mit gezeigt muted.

    <iframe src="https://players...index.html?videoId=4784463159001&muted"

    Parameter Beschreibung Data Type (Datenart)
    adConfigId Eine Dynamic Delivery SSAI-Anzeigenkonfiguration ID. String
    applicationId Ermöglicht die Wiederverwendung eines einzelnen Players, unterscheidet jedoch Analysen pro Site oder pro Anwendung. Weitere Informationen finden Sie im Dokument „ Application ID zum Player Embed Code“ hinzufügen . String
    automatische Wiedergabe Gibt an, dass der Spieler sofort auf Plattformen spielen sollte, auf denen dies erlaubt ist.

    Wenn auf einen booleschen Wert gesetzt autoplay ist, wird das native Autoplay-Verhalten des Browsers verwendet. Wenn er auf einen der folgenden drei gültigen Zeichenfolgenwerte festgelegt ist, geschieht Folgendes:
    • play: Der Spieler wird manuell versuchen, das Spiel anzurufen.
    • muted: Der Spieler wird den Spieler manuell stummschalten und spielen.
    • any: Der Spieler wird zuerst versuchen, Play anzurufen. Wenn das fehlschlägt, schalten Sie den Spieler stumm und rufen Sie Play auf.
    Wenn einer der oben genannten Punkte fehlschlägt, zeigt der Spieler den „Big Play Button“ an, als ob Autoplay auf false gesetzt wäre.

    Ein Video zur automatischen Wiedergabe zu bringen, kann eine komplexe Situation sein. Weitere Einzelheiten finden Sie im Dokument „ Überlegungen zu Autoplay “.

Zum Beispiel, &autoplay=false stoppt die automatische Wiedergabe nicht. Tatsächlich würde der letzte Abfrageparameter die automatische Wiedergabe aktivieren, als ob die URL die Zeichenfolge enthält automatisches Abspielen Die automatische Wiedergabe wird aktiviert, unabhängig davon, welcher Wert ihr zugewiesen wurde.

    Details vorladen

    Der Preload-Parameter informiert den Browser darüber, ob die Videodaten heruntergeladen werden sollen, sobald das Video-Tag geladen ist. Die Optionen sind Auto, Metadaten und keine. Für die HLS-Wiedergabetechnologie bedeuten die Einstellungen etwas andere Dinge und sind unten aufgeführt.

    • Auto (Standard): Beginnen Sie sofort mit dem Laden des Videos (wenn der Browser zustimmt). Einige Mobilgeräte wie iPhones und iPads laden das Video nicht vor, um die Bandbreite ihrer Benutzer zu schützen. Deshalb heißt der Wert Auto und nicht etwas Endgültigeres wie „Wahr“.

      Für die HLS-Wiedergabetechnologie bedeutet Auto , dass der Player die Manifestdatei m3u8 und nur ein Segment anstelle von 3-4 herunterlädt, wie es vor Version 5.3 von Brightcove Player der Fall war.

    • Metadaten:: Laden Sie nur die Metadaten des Videos, die Informationen wie Dauer und Abmessungen des Videos enthalten.

      Für die HLS-Wiedergabetechnologie sind Metadaten die gleichen wie Auto.

    • keine: Laden Sie keine der Videodaten vor. Dies wird warten, bis der Benutzer auf Play klickt, um mit dem Herunterladen zu beginnen. Beachten Sie, dass unter iOS auch bei dieser Einstellung möglicherweise Segmentdownloaden. Weitere Informationen finden Sie im Dokument Brightcove Player und iOS .

      Für die HLS-Wiedergabetechnologie keiner bedeutet, dass der Spieler nichts mehr lädt, einschließlich des m3u8-Manifests, bis der Benutzer die Wiedergabetaste drückt.
          <video-js preload ...>
      or
      { "preload": "auto" }
    Hinweis: Bei Verwendung der serverseitigen Anzeigeneinfügung (SSAI) empfiehlt Brightcove die Einstellung preload=none. Wenn preload ist eingestellt auf auto oder metadata werden die ersten Frames des Videostreams angefordert, wodurch die Anzeige als geliefert gilt, auch wenn der Nutzer die Wiedergabe nicht startet.
