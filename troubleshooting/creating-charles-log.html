Erstellen eines Charles-Protokolls

{{ page.description }}

Charles ist ein Web-Proxy, der auf Ihrem Computer ausgeführt wird und es Ihnen ermöglicht, Daten aufzuzeichnen, die zwischen Ihrem Webbrowser und dem Server gesendet und empfangen werden. Verwenden von Charles macht es einfach, Probleme zu diagnostizieren und zu beheben. Einzelheiten finden Sie im Charles Web-Proxy Seite? ˅.

Beim Benutzen Charles , schließen Sie so viele Anwendungen wie möglich und schließen Sie alle unnötigen Browserfenster, um die Menge der erfassten Informationen zu begrenzen Charles.

Schritte zum Erstellen von a Charles Protokoll

Die folgenden Schritte führen Sie durch die Verwendung der Charles Web-Proxy, um ein aufgezeichnetes Protokoll zu erstellen.

  1. Herunterladen und installieren Charles auf Ihrer Maschine. Charles ist kompatibel mit Windows, Mac und Linux. Für plattformspezifische Details zur Installation Charles , siehe die Charles Installation Anmerkungen.

  2. Führen Sie auf Ihrem Computer aus Charles. Wählen Sie im Menü die Option Proxy> Proxy-Einstellungen. Stellen Sie sicher, dass ein Port zugewiesen wurde (z. B. 8888) und der Aktivieren Sie transparentes HTTP-Proxy Option ist ausgewählt.
  3. Wählen OK um Änderungen zu speichern und den Dialog zu schließen.
  4. Installiere das Charles SSL-Zertifikat. Wählen Sie im Menü Hilfe> SSL-Proxying> Installieren Charles Root-Zertifikat.
  5. In dem Zertifikat hinzufügen Dialog, wählen Sie Hinzufügen.
  6. Schlüsselbundzugang wird Öffnen. Suchen Sie in der Zertifikatsliste das Charles Stellvertreter... Zertifikat. Doppelklicken Sie darauf, um detaillierte Informationen anzuzeigen.
    Wenn Sie macOS nicht verwenden, lesen Sie die Charles Dokumentation für Details zu SSL-Zertifikaten.
  7. Erweitern Sie die Vertrauen Sektion. Für die Bei Verwendung dieses Zertifikats Option auswählen Immer Vertrauen.
  8. Schließen Sie das Zertifikatfenster und geben Sie Ihr Administratorkennwort ein, um die Einstellungen zu aktualisieren.
  9. SSL-Proxying aktivieren in Charles. Wählen Sie im Menü die Option Proxy> SSL-Proxy-Einstellungen. Sicherstellen, dass die Aktivieren Sie SSL-Proxy Option ist ausgewählt. In dem Ort Abschnitt, wählen Sie die Hinzufügen Taste.
  10. In dem Standort bearbeiten Navigieren Sie zum Dialogfeld Hafen Feld und eingeben 443. Wählen OK um diese Änderung zu speichern.
  11. Wählen OK zu schließen Standort bearbeiten Dialog.
  12. Wählen OK zu verlassen SSL-Proxy-Einstellungen Dialog.
  13. Es wird empfohlen, dass Sie Ihren Computer zu diesem Zeitpunkt neu starten, um sicherzustellen, dass Charles übernimmt die neuen Zertifikatseinstellungen.
  14. Halten Charles öffnen, während Sie die nächsten Schritte ausführen.
  15. Fügen Sie die URL auf die Seite mit dem Brightcove Player, den Sie beheben, in die Adressleiste des Browsers ein, und drücken Sie dann die Eingabetaste, um die Seite zu laden.
  16. Führen Sie die Schritte aus, die Sie zuvor verwendet haben, um den Fehler oder das Problem zu reproduzieren. Charles zeichnet automatisch auf, was passiert.
  17. Wenn Netzwerkverkehr benötigt wird, wenn der Player ein Video abspielt, starten Sie das Video.
  18. Kehren Sie nach Reproduzieren des Fehlers zu . zurück Charles. Wähle aus Struktur Zeigen Sie die URL für Ihren Player an und suchen Sie sie.
  19. Wählen Höre auf, aufzunehmen.
  20. Speichern Sie die aktuelle Sitzung in einer Datei.
  21. E-Mail an die Charles Sitzungsdatei an das entsprechende Supportpersonal weiterleiten.

Fügen Sie Akamai-Header hinzu

In diesem Abschnitt erfahren Sie, wie Sie Ihren Anforderungen Akamai-Header hinzufügen.

  1. Von der Hauptleitung Charles Fenster, wählen Werkzeuge und dann Umschreiben.
  2. In dem Einstellungen neu schreiben Dialog, wählen Sie Aktivieren Sie das Umschreiben.
  3. Fügen Sie ein neues Set mit dem Namen hinzu Akamai-Header.
  4. Fügen Sie eine neue Regel hinzu.

  5. In dem Rewrite-Regel Wählen Sie im Dialogfeld a Art von Header hinzufügen.

  6. In dem Ersetzen Abschnitt, fügen Sie Folgendes hinzu:

    Name: Pragma

    Wert:

          akamai-x-get-request-id, akamai-x-cache-on, akamai-x-cache-remote-on,
      akamai-x-check-cacheable, akamai-x-get-cache-key, akamai-x-get-extracted-values,
      akamai-x-get-nonces, akamai-x-get-ssl-client-session-id, akamai-x-get-true-cache-key,
      akamai-x-serial-no
  7. Wählen OK , und dann OK.
