    Chromecast Plugin für Brightcove Spieler

    In diesem Thema erfahren Sie, wie Sie Ihre Videos mithilfe des Chromecast-Plugins (2.x) für Brightcove Player auf ein mit Chromecast verbundenes Fernsehgerät übertragen.

    Übersicht

    Google Chromecast ist ein Gerät, das Sie an den HDMI-Anschluss Ihres Fernsehers anschließen. Wenn Sie Ihr Smartphone oder Ihren Computer als Fernbedienung verwenden, können Sie mit Chromecast auf Videoinhalte zugreifen. Mit dem Chromecast-Plugin für Brightcove Player kann der Player Videos von Ihrem Desktop oder Android Chrome-Browser auf das Chromecast-Gerät übertragen.

    Das Plugin unterstützt alle Video Cloud-Videos oder externen Streams mit HLS oder DASH, einschließlich DRM-verschlüsselter Streams mit Widevine. Werbung wird durch serverseitige Anzeigeneinfügung (SSAI) von Video Cloud unterstützt. Clientseitige Anzeigen und Playlists werden nicht unterstützt.

    Gehen Sie folgendermaßen vor, um das Plugin zu verwenden:

    1. Lade das Chromecast Plugin für Brightcove Player (siehe unten).
    2. Stellen Sie sicher, dass sich das Chromecast-Gerät im selben Netzwerk befindet wie Ihr Gerät, auf dem das Video abgespielt wird.
    3. Beginnen Sie mit dem Abspielen des
    4. Klicken Sie auf die Cast-Schaltfläche und wählen Sie Ihr Chromecast-Gerät aus.

    Chromecast-App-Komponenten

    Die Software hinter Chromecast besteht aus folgenden Komponenten:

    • Anwendung für Absender: Dies ist jede Anwendung, die eine Besetzungssitzung initiieren und mit einem Empfänger kommunizieren kann. Die Absenderanwendung kann als „kundenseitiger“ Teil des Chromecast-Ökosystems angesehen werden.

      Das Chromecast Plugin für Brightcove Player aktiviert die Absender-App im Brightcove Player. In diesem Thema erfahren Sie etwas darüber.

    • Anwendung des Empfängers: Dies ist eine benutzerdefinierte Webanwendung, die im öffentlichen Internet gehostet und auf einem Chromecast-Gerät ausgeführt wird. Es übernimmt die Kommunikation zwischen der Sender-App und dem Empfängergerät. Es kann als einseitige HTML-App mit CSS- und JavaScript-Assets angesehen werden.

      Damit Chromecast mit Playern mit Domainbeschränkungen arbeiten kann, sollten Sie die Domäne des Chromecast-Empfängers auf die Whitelist setzen players.brightcove.net.

    Anforderungen

    Die folgenden Anforderungen sind für das Chromecast-Plugin erforderlich:

    • Brightcove Player v6.45.0 oder neuer
    • Brightcove Chromecast Plugin v2.0.0 oder neuer

    Endgeräte

    Folgende Geräte werden unterstützt:

    • Chromecast (3. Generation)
    • Chromecast Ultra
    • Chromecast mit Google TV (vorläufige Unterstützung)
    Ältere Versionen oder Kombinationen funktionieren möglicherweise weiterhin, werden jedoch nicht unterstützt.

    Wie es funktioniert

    Die aktuelle Implementierung (Plugin / Receiver Version 2.x) verwendet die CAF-API (Cast Application Framework).

    Wenn das Chromecast-Empfänger-Plugin hinzugefügt wird, zeigt der Player eine Cast-Schaltfläche in der Player-Benutzeroberfläche an, wenn im lokalen Netzwerk ein Chromecast verfügbar ist.

    Wenn Sie auf diese Schaltfläche klicken oder darauf tippen, wird eine Besetzungssitzung mit dem aktuellen Video gestartet, das in den Brightcove Player geladen wird. So funktioniert's:

    1. Das Plugin sendet eine Reihe von Parametern an den Empfänger.
    2. Der Empfänger spiegelt den sendenden Player, indem er ihn vom CDN von Brightcove lädt.
    3. Nach dem Laden lädt der empfängerseitige Player die Video-ID von der Wiedergabe-API, die zu Beginn der Besetzungssitzung gesendet wurde.
    4. Dann beginnt der empfängerseitige Spieler die Wiedergabe an der Abspielkopfposition des sendenden Spielers.

    Dieser Vorgang ist für Nicht-Video-Cloud-Quellen identisch, außer dass keine Playback-API-Anforderung vorliegt.

    Analytics

    Aus analytischer Sicht wird die Besetzungssitzung derzeit als völlig neue Wiedergabesitzung behandelt.

    Aus UX-Sicht sieht der Betrachter, wie der Stream auf dem Sendergerät angehalten wird und auf dem Empfänger fortgesetzt wird.

    Aus Datenperspektive wird unsere Metrik, da sie standardmäßig anonymisiert ist, als neuer Viewer protokolliert, der einen neuen Stream auf einem separaten Gerät startet. Das ist es, was tatsächlich passiert.

    Verwenden des Plugins

    Das Chromecast Plugin für Brightcove Player wird wie alle anderen Brightcove Player-Plugins verwendet, dh Sie müssen das Plugin in den Player laden.

    Verwenden von Studio

    In den folgenden Schritten wird beschrieben, wie Sie das Plugin mit Studio laden.

    Bei der Konfiguration in Studio wählt die Player-Verwaltungs-API automatisch die beste Versionskombination für Ihren Player aus. Wenn du bekommst v1.x. des Chromecast-Plugins müssen Sie Ihren Player aktualisieren!
    1. Öffnen Sie das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Spieler, dem Sie das Plugin hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
    3. Wählen Sie im linken Navigationsmenü Plugins.

    4. Wählen Sie als Nächstes die Schaltfläche Plugin hinzufügen und dann Brightcove Pluginaus.

    5. Erweitern Sie die Brightcove Plugin Dropdown und auswählen Chromecast-Empfänger.

    6. Optional: Geben Sie die Konfigurationsoptionen im Textfeld Optionen (JSON) ein. Ein Beispiel für den Standardplayer im 1752604059001-Konto wird angezeigt. Sie müssen natürlich Informationen für Ihren Spieler eingeben. 

      {
"playerUrl" : "https://players.brightcove.net/1507807800001/default_default/index.min.js"
}

      Einzelheiten zu den Optionen finden Sie unten.

    7. Wählen Sie die Schaltfläche „ Speichern “. Sie werden jetzt die sehen Chromecast-Empfänger Plugin zur Liste der Plugins für Ihren Player hinzugefügt.

    8. Um den Player zu veröffentlichen, wählen Sie Veröffentlichungs- und Einbettungsänderungen einbetten >aus.
    9. Um den Öffnen-Dialog zu schließen, wählen Sie Schließenaus.

    10. Kehre zu MEDIA zurück und veröffentliche dein Video oder deine Playlist mit dem Player, den du gerade für Chromecast aktualisiert hast.

      Sobald das Plugin geladen ist, das Video mit der Wiedergabe begonnen hat und ein verfügbares Chromecast-Gerät in Reichweite ist, wird die Cast-Schaltfläche im Player angezeigt, wie hier in diesem Screenshot eines Players gezeigt:

      Die Cast-Schaltfläche des Chromecast-Plugins wird in der Kontrollleiste des Players angezeigt, wenn sich ein verfügbares Chromecast-Gerät in Reichweite befindet.

    Manuell konfigurieren

    Führen Sie die folgenden Schritte aus, um das Plugin mit dem JSON-Editor hinzuzufügen:

    1. Öffnen Sie das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Spieler, dem Sie das Plugin hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.

    3. Wählen Sie im linken Navigationsmenü JSON Editoraus.

    4. Fügen Sie dem ein Objekt hinzu plugins Array

      Ihr Code sollte ähnlich wie folgt aussehen: 

      {
  "compatibility": true,
  "video_cloud": {
    "policy_key": "your policy key"
  },
  "player": {
    "template": {
      "name": "single-video-template",
      "version": "6.45.4"
    }
  },
  "studio_configuration": {
    "player": {

      "scripts": ["https://players.brightcove.net/videojs-chromecast-receiver/2/videojs-chromecast-receiver.js"]
    }
  ]
}

      Einzelheiten zu den Optionen finden Sie unten.

    Optionen für die Konfiguration

    Unabhängig davon, welche Methode Sie zum Einrichten des Chromecast-Plugins für Brightcove Player auswählen, können Sie Optionen zum Ändern des Plugin-Verhaltens übergeben. Während das Festlegen dieser Optionen nicht erforderlich ist, sind zusätzliche Anpassungen für fortgeschrittene Benutzer möglich.

    Um Optionen im Code zu übergeben, würden Sie etwas Ähnliches wie das Folgende verwenden: 

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  options = {};
  options.playerUrl = '//players.brightcove.net/1752604059001/default_default/index.min.js';
  myPlayer.chromecastReceiver(options);
});

    Rat

    Wenn Ihre Web-Player eine große Anzahl von Plugins oder benutzerdefinierten Skripten enthalten, sollten Sie einen Lightweight-Player speziell für die Chromecast-Wiedergabe erstellen.

    Da der Web-Player und seine Wiedergabe-Engine etwas ressourcenintensiv sind, kann die Ausführung in einer Umgebung mit eingeschränkten Ressourcen (z. B. einem Chromecast) mit zusätzlichen umfangreichen Skripten die Benutzererfahrung negativ beeinflussen!

    Optionen

    Das Chromecast-Plugin unterstützt die folgenden Optionen:

    • options.playerUrl
      Art:: string
      Standard : ''

      Standardmäßig wird der senderseitige Player auf dem Empfänger gespiegelt. Mit dieser Option können Sie eine benutzerdefinierte empfängerseitige Brightcove Player-URL bereitstellen.

      Da es sich um einen Brightcove Player handelt, der auf den Receiver geladen wird, können Sie den Player außerhalb von Chromecast anpassen, gestalten und debuggen.

      Dies sollte eine URL zu einem sein index.min.js Datei auf dem Brightcove CDN
    • options.splashScreen
      Art:: string
      Standard : ''

      Der Begrüßungsbildschirm, der vor einem Video und beim Umschalten von Videos angezeigt werden soll.

    • options.authRequest
      Art :: string oder object oder function
      Standard : {}

      Dies ist eine ungewöhnliche Option, sodass Sie sie wahrscheinlich nicht benötigen werden!

      Sie können diese Option verwenden, wenn der Chromecast eine Anforderung zum Abrufen der Berechtigung zum Abspielen von Inhalten benötigt, bevor ein Video angefordert wird.

      Sie können dies auch verwenden, um Cookies zu erhalten, aber Ihr Server muss eingerichtet sein, um dies zuzulassen players.brightcove.net die Möglichkeit, domänenübergreifende Cookies zu setzen, wenn Sie Ihre eigenen Inhalte hosten.

      String-Verwendung

      Wann authRequest Ist eine Zeichenfolge, wird eine leere GET-Anforderung an diese URL gesendet.

      Objektnutzung

      Wann authRequest ist ein Objekt, mit dem bestimmte Eigenschaften der Anforderungsnutzlast angegeben werden können:

      Eigenschaft Art Erforderlich Default Beschreibung
      url string Ja '' Die anzufordernde URL
      method string Nein GET Die zu verwendende HTTP-Methode
      body string Nein '' Der Anfragetext
      headers object Nein {} Ein Objekt, das HTTP-Header beschreibt, bei denen die Schlüssel der Header-Name und die Eigenschaften die Header-Werte sind
      withCredentials boolean Nein false Ein Objekt, das HTTP-Header beschreibt, bei denen die Schlüssel der Header-Name und die Eigenschaften die Header-Werte sind
      timeout number Nein 15000 Wie viele Millisekunden müssen gewartet werden, bevor die Authentifizierungsanforderung aufgegeben wird?

      Funktionsnutzung

      Wann authRequest Ist eine Funktion, sollte sie für diese Verwendungen entweder eine Zeichenfolge oder ein Objekt zurückgeben.

    Einschränkungen und bekannte Probleme

    • HEVC / 4K-Inhalte werden nur auf Chromecast Ultra-Geräten und neueren Chromecasts mit Google TV-Geräten unterstützt.
    • Clientseitige Werbung wird derzeit nicht unterstützt, SSAI wird jedoch unterstützt!
    • Wir haben Probleme mit der Suche auf dem Absender festgestellt, nachdem eine Beschriftungsspur ausgewählt und im Empfänger angezeigt wurde.
    • Bei Chromecast mit Google TV-Geräten sind Probleme mit der Benutzeroberfläche sowie Probleme bei der Wiedergabe bei der Verwendung von Untertiteln aufgetreten.
    • Google hat die Unterstützung unsicherer Ursprünge (HTTP) mit Chromecast eingestellt, daher funktioniert das Plugin nicht in unsicheren Kontexten. In diesen Fällen wird die Chromecast-Schaltfläche des Spielers nicht angezeigt.
    • Da das Chromecast-Plugin auf die Unterstützung des Browsers für das Casting angewiesen ist, wird es nur in den folgenden OS/Browser-Kombinationen unterstützt:
      • Desktop/Chrome
      • Android/Chrom
    • Google unterstützt nicht mehrere Instanzen der Chromecast-Absenderschaltfläche auf einer einzigen Seite. Mögliche Problemumgehungen wären, den Brightcove Player mithilfe der Standard (iframe) -Player-Implementierung einzubetten oder Spieler nach Bedarf dynamisch zu instanziieren und zu vernichten. Siehe das Dokument Beispiel für Brightcove Player: Laden Sie den Player dynamisch , um Informationen über die letztere Möglichkeit zu erhalten.
    • Dieses Plugin wird nicht an Google Nest Hub übertragen. Das Plugin unterstützt nur tatsächliche Chromecast-Geräte (d. h. Chromecast und Chromecast Ultra).

    Änderungen von v1.x zu v2.x.

    Neben internen Änderungen wie der Verwendung der CAF-API anstelle der v2 Chromecast-API haben sich die folgenden Schnittstellen geändert:

    • options.css und options.js werden nicht mehr unterstützt.
    • customData.analyticsParams wurde hinzugefügt
    • customData.catalogData wurde umbenannt customData.catalogParams um die Konsistenz mit anderen Verwendungen zu verbessern
    • customData.policyKey wurde ersetzt durch customData.catalogParams.policyKey
    • customData.keySystems wurde hinzugefügt
