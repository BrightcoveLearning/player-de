    GoogleTag-Manager-Plugin für Brightcove Spieler

    In diesem Thema erfahren Sie, wie Sie das Google Tag Manager (GTM) -Plugin für Brightcove Player verwenden.

    Übersicht

    Mit dem Google Tag Manager (GTM) -Plugin können Sie GTM-Funktionen in Brightcove Player integrieren.

    Tag-Manager ermöglichen es Ihnen, Tags hinzuzufügen und zu veröffentlichen, ohne Code verwenden zu müssen. Da es sich um eine clientseitige Lösung handelt, können Sie mit Tag-Managern Daten über verschiedene Marketingtechnologien hinweg sammeln. Dies ermöglicht Vermarktern, indem sie ihnen die Möglichkeit geben, Benutzer- und Spielerdaten über eine Vielzahl von Sammelstellen zu sammeln.

    Wenn Sie bereits Google Analytics, AdWords und DoubleClick verwenden, möchten Sie möglicherweise auch Google Tag Manager verwenden, da dieser sich gut in diese Tools integrieren lässt. GTM bietet eine Vielzahl von Variablen- und Triggeroptionen.

    Das Google Tag Manager-Plugin aktiviert Sie zu verfolgen Helle Bucht Spieler Veranstaltungen mit die Google Analytics Instrumententafel. Das folgenden Spieler Veranstaltungen kann verfolgt werden: 

    • Laden des Players
    • Videoimpression
    • Videoaufruf
    • Video Engagement
    • Spielanforderung
    • Videowiedergabe beendet
    • Erweiterte Spielerereignisse sowie Anzeigenstart, Anzeigende
    • Fehler, die das stören Wiedergabeerfahrung
    Das Google Tag Manager-Plugin wird mit der Brightcove Player-Version 6.36.5+ unterstützt.

    Tag-Management-Terminologie

    Im Folgenden finden Sie einige häufig verwendete Begriffe, die bei der Verwendung von Google Tag Manager (GTM) auftreten.

    Term Beschreibung
    Behälter Ihr Tag-Management-Systemkonto kann mehrere Container (Profile) mit eigenen Tags und Variablen haben. Jeder Container kann mehrere Umgebungen haben, die normalerweise für Entwicklung, Bereitstellung und Produktion eingerichtet sind.
    Datenschicht Die Datenschicht bezieht sich auf die Sammlung von Daten, die im Code Ihrer Webseite enthalten sind. Dieses JavaScript-Objekt wird verwendet, um Informationen von Ihrer Website an Ihren Tag-Manager-Container zu übergeben. Sie können diese Informationen dann verwenden, um Variablen zu füllen und Trigger in Ihren Tag-Konfigurationen zu aktivieren.
    Veranstaltung Ereignisse sind Datenpunkte, die verwendet werden, um Daten an Ihr Berichtstool zu senden. Diese beziehen sich auf das Laden und die Wiedergabe von Videos.
    Etikett Tags sind Code-Snippets, die auf Ihrer Webseite platziert werden, um Daten zur Benutzerinteraktion zu sammeln. Tag-Management-Systeme vereinfachen das Hinzufügen von Tags zu Ihrer Site und erfordern, dass Sie nur die Tag-ID und den Tag-Ereignistyp angeben.
    Markt-Marktplatz Vorlagen für Tag Marketplace bieten Code-Snippets für die Bereitstellung eines Tags. Während dies die Menge an Codekenntnissen reduziert, ist für eine erfolgreiche Implementierung eine gewisse Vertrautheit mit JavaScript und HTML erforderlich.
    Tracker Tracker sind Objekte, die Daten sammeln, speichern und an Google Tag Manager senden.
    Auslösen Trigger geben an, wann Tag-Code zu einer Seite hinzugefügt werden soll. Dies sind benutzerdefinierte Bedingungen.

    Erste Schritte

    Gehen Sie folgendermaßen vor, um Google Tag Manager einzurichten:

    Die Schritte

    1. Lesen Sie die Dokumentation von Google zu Richten Sie den Tag Manager ein und installieren Sie ihn.

    2. Im Tag Manager , erstellen Sie ein Konto oder verwenden Sie ein vorhandenes Konto. Um ein neues Konto zu erstellen, wählen Sie die Option Benutzerkonto anlegen Taste.

      Standardmäßig wird ein neuer Container für ein neues Konto erstellt. Sie können in jedem Konto zusätzliche Container erstellen.

    3. Geben Sie Werte für Ihre neue Kontoseite ein, einschließlich Kontoname und Containername. Wählen Sie eine Zielplattform aus.

    4. Wählen Sie die Schaltfläche „ Erstellen “.

    Überprüfen von Ereignissen, die verfolgt werden sollen

    Überprüfen Sie die folgende Tabelle und bestimmen Sie, welche Ereignisse Sie verfolgen möchten.

    Art Ereignis Beschreibung
    Player Load (Playerladevorgänge) bcvideo_player_load Wird gesendet, wenn ein Player geladen wurde und zur Wiedergabe bereit ist
    Video Impression (Videoimpressionen) bcvideo_video_impression Wird gesendet, wenn ein Video in einen Player geladen wurde und zur Wiedergabe bereit ist
    Videoansicht bcvideo_video_view Wird gesendet, wenn die Wiedergabe eines Videos entweder durch automatische Wiedergabe oder aufgrund von Benutzerinteraktion gestartet wurde
    Video-Interesse bcvideo_video_engagement Ein Benutzer hat sich eine Reihe von Sekunden der Zeitleiste eines Videos angesehen. Dieses Ereignis ist ein Herzschlag für die Verfolgung der Videointeraktion und wird wahrscheinlich mehrmals während der Wiedergabe gesendet, abhängig von der Benutzerinteraktion und der Länge des Videos.
    Wiedergabe-Anfrage bcvideo_play_request Wird gesendet, wenn die Wiedergabe entweder durch den Benutzer ausgelöst wird, der ausdrücklich auf die Wiedergabetaste klickt, oder automatisch, wenn die Plattform die Wiedergabe in einem Auto-Play-Szenario auslöst
    Anzeigen-Start bcvideo_ad_start Wird gesendet, wenn die Steuerung von der Wiedergabeplattform an einen Werbeagentur übergeben wird
    Ad-End bcvideo_ad_end Wird gesendet, wenn die Kontrolle vom Werbeagentur an die Wiedergabeplattform zurückgegeben wird
    Fehler bcvideo_error Wird gesendet, wenn schwerwiegende Fehler auftreten, die die Wiedergabe stören
    Video abgeschlossen bcvideo_video_complete Wird gesendet, wenn die Videowiedergabe beendet ist

    Erstellen von Variablen, Tags und Triggern

    Jedes zu verfolgende Brightcove-Ereignis muss als erstellt werden Ereignis-Tag im Google Tag Manager. Die Ereignis-IDs von GTM werden im Player verwendet, sodass Brightcove-Ereignisdaten an GTM-Ereignisse übergeben werden können.

    Für jedes Tag müssen GTM-Trigger erstellt werden, um diese Ereignisse an Google Analytics (GA) zu senden. Die den Ereignissen zugeordneten benutzerdefinierten Parameter werden ebenfalls im GTM erstellt Variablen Tab.

    Hinzufügen von Variablen zu GTM

    Parameter enthalten zusätzliche Daten und werden mit Ereignissen übergeben. Jeder Brightcove-Parameter, der aufgezeichnet wird, muss im Google Tag Manager als Variable erstellt werden. Der genannte Parameter wird dann im Player verwendet, sodass Brightcove-Parameter an Google Tag Manager-Variablen übergeben werden können.

    Führen Sie die folgenden Schritte aus, um Variablen in Google Tag Manager zu erstellen

    1. Melden Sie sich bei Ihrem Google Tag Manager-Konto

    2. Wählen Variablen in der linken Navigation.
    3. Zum Benutzerdefinierte Variablen , wähle aus Neu Taste.

    4. Wähle aus Variablentyp wie Datenschichtvariable.

    5. Geben Sie die Details ein:

      • Name der Datenschichtvariablen - Name der Variablen
      • Datenschicht Ausführung - Version 2

    6. Zu erstellende Parameter:

      • Konto - Brightcove-Kontoname
      • Spieler - Spieler-ID
      • Spielername - Spielername
      • Plattformversion - Versionsnummer der Plattform
      • Session - Sitzungskennung
      • Video - Video ID
      • Videoname - Name des Videos
      • Angebot - Der Sekundenbereich, der im Format StartSecond..EndSecond angezeigt wird
      • Video Sekunden angezeigt - Anzahl der beobachteten Sekunden
      • Videodauer - Die Dauer des Videos in Sekunden
      • Video Prozent angezeigt - Prozentsatz des angesehenen Videos
      • Fehlercode - Plattformspezifischer Fehlercode für das Ereignis
      • Video Meilenstein - Meilenstein (25%, 50%, 75%, 100%) durch das Video erreicht
      • Video-Abspielkopf - Die Zeit im Video, zu der das Ereignis ausgelöst wird
      • Land - Ländercode ISO-3166 (Alpha 2), z.B: USA
      • Ländername - Vom Menschen lesbarer Ländername z.B: USA
      • Region - ISO-3166 (Alpha 2) -Regionscode
      • Regionsname - Vom Menschen lesbarer Regionsname
      • Stadt - Stadtname
      • Browser - Vollständiger Name und Version des verwendeten Browsers
      • Browsertyp - Allgemeiner Name des verwendeten Browsers (z. B. Chrome, Edge, Firefox)
      • Ziel - URI, die das Ereignis ausgelöst hat
      • Quelle - URI, der den Endbenutzer zum Ziel gesendet hat
      • Gerätetyp - Überschreiben, um das Betriebssystem des Geräts anzugeben, von dem das Ereignis stammt, wenn der User-Agent unzuverlässig ist
      • Gerätebetriebssystem - Der Name des vom Gerät verwendeten Betriebssystems
      • Gerätebetriebssystemversion - Die vom Gerät verwendete Betriebssystemversion

    Hinzufügen von Tags zu GTM

    Nachdem Sie festgelegt haben, welche Ereignisse Sie verfolgen möchten, müssen Tags im Google Tag Manager erstellt werden.

    Führen Sie die folgenden Schritte aus, um Tags in Google Tag Manager zu erstellen

    1. Melden Sie sich bei Ihrem Google Tag Manager-Konto

    2. Wählen Stichworte Klicken Sie in der linken Navigation auf die Option Neu Taste.

    3. Wähle aus Tag-Typ wie Google Analytics: Universal Analytics.

    4. Geben Sie die Tag-Details ein:

      • Spurtyp - Einstellen Veranstaltung
      • Kategorie - Ereigniskategorievariable
      • Aktion - Ereignisaktionsvariable
      • Etikette - Ereignisbezeichnungsvariable
      • Wert - Null
      • Nicht-Interaktionstreffer - Einstellen Wahr
      • Google Analytics-Einstellungen - Voreinstellung
      • Tracking ID - Tracking-ID des GA-Dashboards. Wählen Sie Tracking ID Variable.

    5. Gehen Sie zu Erweiterte Einstellungen und legen Sie die zugehörigen benutzerdefinierten Bemaßungsindizes und Bemaßungswerte fest. Diese Indizes sollten mit den Angaben in Google Analytics übereinstimmen.

    6. Ihre Tags sollten in GTM ähnlich aussehen:

    Hinzufügen von Triggern zu GTM

    Für jedes Ereignis, das Sie verfolgen möchten, benötigen Sie außerdem einen Auslöser im Google Tag Manager.

    Führen Sie die folgenden Schritte aus, um Trigger in Google Tag Manager zu erstellen

    1. Melden Sie sich bei Ihrem Google Tag Manager-Konto

    2. Wählen Löst aus Klicken Sie in der linken Navigation auf die Option Neu Taste.

    3. Wähle aus Auslösertyp wie Benutzerdefiniertes Ereignis.

    4. Geben Sie den Ereignisnamen ein.

    5. Erstellen Sie einen Trigger für jedes Ereignis, das Sie verfolgen möchten.

    Integration in Brightcove

    Um GTM in Brightcove Player zu integrieren, fügen Sie Ihrem Player das Google Tag Manager-Plugin hinzu.

    Verwenden von Studio

    In den folgenden Schritten wird beschrieben, wie Sie das Plugin mit Video Cloud Studio laden.

    1. Öffnen Sie in Video Cloud Studio das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Player, dem Sie das Plugin hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
    3. Wähle im linken Navigationsmenü Plugins aus.

    4. Wählen Sie als Nächstes die Schaltfläche Plugin hinzufügen und dann Brightcove Pluginaus.

    5. Erweitern Sie die Brightcove Plugin Dropdown und auswählen Google Tag Manager.

    6. Erstellen Sie ein JSON-Objekt, um dem Player mitzuteilen, welche Ereignisse und benutzerdefinierten Dimensionen Sie verfolgen möchten Optionen (JSON) Textfeld.

      Beschreibungen der Felder:

      • tracker- Ihre Google Tag Manager-Tracking-ID (GTM-XXXXXXX)
      • eventsToTrack- Ereignisse, die Sie verfolgen möchten
      • paramsToTrack- Benutzerdefinierte Abmessungen, die Sie verfolgen möchten

      Beispiel:

      {
  "tracker":"your Google Tag Manager id",
  "eventsToTrack": {
    "error":"Error",
    "play_request":"Play Request",
    "video_engagement":"Video Engagement"
  },
  "paramsToTrack": {
    "bcvideo_video_seconds_viewed":"dimension1",
    "bcvideo_video_playhead":"dimension2",
    "bcvideo_account":"dimension3",
    "bcvideo_player":"dimension4",
    "bcvideo_player_name":"dimension5",
    "bcvideo_session":"dimension8",
    "bcvideo_video":"dimension6",
    "bcvideo_video_name":"dimension7",
    "bcvideo_platform_version":"dimension9",
    "bcvideo_video_duration":"dimension11",
    "bcvideo_range":"dimension10",
    "bcvideo_video_percent_viewed":"dimension12",
    "bcvideo_error_code":"dimension13",
    "bcvideo_video_milestone":"dimension14",
    "bcvideo_destination":"dimension15"
  }
}
      Stellen Sie sicher, dass Ihre benutzerdefinierten Dimensionen im JSON-Objekt mit dem Namen und dem Index in Google Analytics übereinstimmen.

    7. Wählen Sie die Schaltfläche „ Speichern “. Sie werden jetzt die sehen Google Tag Manager Plugin zur Liste der Plugins für Ihren Player hinzugefügt.

    8. Um den Player zu veröffentlichen, wählen Sie Veröffentlichungs- und Einbettungsänderungen einbetten >aus.
    9. Um den Dialog zu schließen, wählen Sie Schließenaus.

    10. Gehe zurück zum MEDIEN und veröffentlichen Sie Ihr Video oder Ihre Wiedergabeliste mit dem Player, den Sie gerade für Google Tag Manager aktualisiert haben.

      Wenn Ihr Spieler geladen wird und der Benutzer mit dem Spieler interagiert, werden die angegebenen Ereignisse und Parameter verfolgt.

    Benutzerdefinierte Dimensionen erstellen

    Benutzerdefinierte Dimensionen sind Metriken, die Sie selbst in Google Analytics erstellen. Weitere Informationen finden Sie unter Google Benutzerdefinierte Dimensionen und Metriken Dokument.

    Mithilfe von Google Scripts können Sie das Hinzufügen benutzerdefinierter Dimensionen in Google Analytics automatisieren.

    Die Schritte

    Führen Sie die folgenden Schritte aus, um mithilfe eines Google App-Skripts benutzerdefinierte Abmessungen zu erstellen:

    1. Melden Sie sich bei Google an.

    2. Navigieren Sie zu G Suite Developer Hub und wählen Sie das neue leere Projekt Apps Script aus.

    3. Wählen Projekt ohne Titel und geben Sie Ihrem Projekt einen Namen.

    4. Kopieren Sie den folgenden benutzerdefinierten App-Skriptcode in einen Editor und aktualisieren Sie die folgenden Werte:

      • accountId- Ihre Google Analytics-Konto-ID
      • webPropertyId- Ihre Google-Konto-Tracking-ID
      • dimensions- Die benutzerdefinierten Abmessungen, die hinzugefügt werden sollen. Die aktuelle Liste enthält alle benutzerdefinierten Abmessungen. Beachten Sie, dass den Dimensionen ein Präfix vorangestellt ist bcvideo_. Dies soll sicherstellen, dass keine Namenskonflikte mit diesen Variablen bestehen. 
      //Set values for GA Account ID and web property ID to which you have to set custom dimensions.
var accountId='your account id';
var webPropertyId='your tracking code';
//Add required custom dimensions to be created to this list
var dimensions=['bcvideo_video_seconds_viewed','bcvideo_video_playhead','bcvideo_account','bcvideo_player','bcvideo_player_name','bcvideo_video','bcvideo_video_name','bcvideo_session','bcvideo_platform_version','bcvideo_range','bcvideo_video_duration',
'bcvideo_video_percent_viewed','bcvideo_error_code','bcvideo_milestone','bcvideo_destination'];


//Function to create custom dimensions in GA
function createGACustomDimensions() {

  //This loop iterates through the custom dimensions in order to set the custom dimension scope, active values into our GA property
  for(var index=0;index<dimensions.length;index++){
  var customDimensions ={
    'name': dimensions[index],
    'scope':'HIT',
    'active':true,
  }

  //This create a request to the new custom dimension
  Analytics.Management.CustomDimensions.insert(customDimensions,accountId,webPropertyId);

  // Pause for 1 second between adding each custom dimension, this ensure that the GA property has time
  // to process the request fully before moving to the next custom dimension.
  Utilities.sleep(1000)
  }
}

    Ereignisse und Parameter

    Die folgenden Ereignisse und Parameter können an Google Tag Manager gesendet werden.

    Veranstaltungen

    Diese Ereignisse stammen von deinem Spieler.

    • player_load
    • video_impression
    • play_request
    • video_view
    • video_engagement
      • video_engagement- 25% Meilenstein
      • video_engagement- 50% Meilenstein
      • video_engagement- 75% Meilenstein
    • ad_start
    • ad_end
    • error
    • video_complete- 100% Meilenstein

    Der Prozentsatz des angesehenen Videos (video_engagement ) kann wie folgt berechnet werden (ähnlich dem Video_engagement-Ereignis in Google und Adobe Analytics): 

    Video watched percentage = duration the user actually watched / total video duration

    In einem 20-Sekunden-Video kann beispielsweise der Meilenstein von 25% in beiden folgenden Fällen erreicht werden

    • Benutzer sieht von 0-5 Sekunden zu
    • Benutzer sieht von 0 bis 1 Sekunde und springt auf 11. Sekunde und wacht bis 15. Sekunde

    In beiden oben genannten Fällen hat sich der Benutzer 5 Sekunden angesehen, sodass der Meilenstein als 25% gemeldet wird.
