    Werbung mit dem FreeWheel Plugin Changelog

    Im Folgenden werden Änderungen am Plugin verfolgt:

    Changelog

    30. Mrz 2020

    v3.5.2

    • Bugfix: ermöglichen play_request vor dem Start

    18. März 2020

    v3.5.1

    • Unterstützung für hinzugefügt minDuration und maxDuration zu temporalSlots
    • Kompatibilität mit behoben Video.js 7.7.x.

    3 Dez 2019

    v3.5.0

    • Unterstützung für hinzugefügt minDuration und maxDuration zu temporalSlots

    4. September 2019

    v3.4.0

    • Die Größe des Anzeigencontainers wurde geändert, wenn die playerresize Ereignis tritt ein
    • Aktualisieren Sie die FW SDK-URL
    • Ein Größenproblem mit VPAID-Anzeigen wurde behoben
    Hinweis: Wie im aktuellen FreeWheel-Plugin-Dokument erwähnt, finden Sie in der Verwendung der richtigen SDK-URL In diesem Abschnitt hat FreeWheel die dynamische SDK-URL zugunsten einer versionierten URL abgeschafft. Dies bedeutet, dass das Brightcove Player-Plugin jetzt standardmäßig an eine bestimmte Version gebunden ist und nicht mehr an die neueste Version. Brightcove wird versuchen, mit den FreeWheel SDK-Versionen Schritt zu halten, kann jedoch zurückbleiben.

    Möglicherweise möchten Sie Ihre eigene SDK-Version in der Brightcove Player-Integration verwalten. Dies beinhaltet die Übergabe der Html5.sdkurl Option an das FreeWheel Plugin:

    player.FreeWheelPlugin({
  Html5: {
    sdkurl: 'preferred-sdk-url'
  }
});

    v3.3.1

    • Bugfix: Stellen Sie sicher, dass die Dauer ein Float ist, wenn Sie sie an das FreeWheel SDK senden, um sie richtig einzustellen vdur Parameter in der URL
    • Unterstützt neuere Codierungsstandards (vjsstandard 8)
