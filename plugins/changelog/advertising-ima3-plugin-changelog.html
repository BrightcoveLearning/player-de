7. Juli 2017

v2.19.1

Hinweis: Diese Version enthält Korrekturen zur Unterstützung der Autoplay von HMTL5-Anzeigen auf iOS 10- und Android-Geräten.

  • Autoplay HTML5-Anzeigen in Spielern, die nicht stumm geschaltet sind und/oder nicht enthalten sind, playsinline werden korrekt wiedergegeben, wenn der Benutzer die Wiedergabe per Tippen oder Klicken auf iOS 10- und Android-Geräte initiiert
  • Autoplay HTML5-Anzeigen in Spielern, die muted UND enthalten, playsinline werden auf iOS 10- und Android-Geräten korrekt autoplay
  • Behebt die Stummschaltfläche bei iOS 10 und Android HTML5-Anzeigen
    • Wenn das muted Attribut vorhanden ist, wird die Stummschaltfläche angezeigt, mit der der Benutzer die Stummschaltung der Anzeige aufheben kann
    • Die Stummschaltungstaste wird nicht mehr auf Geräten angezeigt, die die programmgesteuerte Steuerung der Lautstärke nicht unterstützen
    • Der Stummschaltknopf stumm-/stummschaltet die Anzeige korrekt stumm-/stummschaltet, wenn Sie
    • Die Einstellung Stummschaltung bzw. Stummschaltung bleibt von der Anzeige bis zum Hauptinhalt bestehen

Darüber hinaus veröffentlichte das IMA HTML5 SDK eine Funktion zum Deaktivieren der benutzerdefinierten Wiedergabe auf iOS 10+-Geräten. Diese Funktion bestimmt, ob die Inline-Wiedergabe oder die native Wiedergabe von Anzeigen verwendet wird, einschließlich der Unterstützung für überspringbare TrueView-Anzeigen. Weitere Informationen zu dieser Einstellung finden Sie in der IMA HTML5 SDK-Dokumentation.

Informationen zur Anwendung dieser Einstellung mit dem Brightcove IMA SDK-Plugin finden Sie in unserer Dokumentation zu Werbung mit dem IMA3-Plugin unter Ima3sdkSettings.

v2.19.0

Version 2.19.0 könnte dazu führen, dass einige programmatische Implementierungen des IMA3-Plugins fehlschlagen. Bitte lesen Sie den Abschnitt Verwenden von Code implementieren in diesem Dokument, um sicherzustellen, dass Ihr Code den empfohlenen Best Practices folgt. Der folgende Fehler zeigt an, wann Sie Ihren aktuellen Code aktualisieren müssen:

VIDEOJS: FEHLER: videojs-contrib-ads hat 5 Sekunden nach der Initialisierung kein Loadstart-Ereignis gesehen, aber eine Quelle ist vorhanden. Dies deutet darauf hin, dass Videojs-Contrib-Anzeigen zu spät initialisiert wurden. Es muss sofort nach video.js im selben Tick initialisiert werden. Infolgedessen werden einige Anzeigen nicht abgespielt und einige Medienereignisse sind falsch.

Weitere Informationen finden Sie in diesem Beitrag im Brightcove Web Player Developer Forum. Allgemeine Informationen zum gemeinsamen Testen von Players und Plugins finden Sie im Thema Testen von Player/Plugins.
