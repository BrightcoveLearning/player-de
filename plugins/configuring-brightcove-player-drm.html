    Den Brightcove Player für DRM konfigurieren

    In diesem Thema erfahren Sie, wie Brightcove Player die Verwaltung digitaler Rechte (DRM) implementiert. Das Dokument zeigt zunächst, wie das DRM-Plugin implementiert wird, gefolgt von Details zum Plugin und seiner Implementierung.

    Brightcove nutzt die folgenden Technologien, um DRM-geschützte Inhalte für eine möglichst große Vielfalt von Browsern und Geräten bereitzustellen:

    • MPEG-DASH mit Native / EME-unterstützten CENC-DRMs
    • HLS mit FairPlay

    Um DRM mit Brightcove Player verwenden zu können, müssen Sie:

    • Produzieren Sie DRM-fähige Inhalte
    • Aktivieren Sie Brightcove Player, um das DRM-Plugin zu verwenden
    • Konfigurieren Sie in einigen Fällen das DRM-Plugin für die Verwendung Ihres Lizenzservers
    Hinweise:
    1. DRM muss von Ihrem Account Manager aktiviert werden.
    2. Es wird empfohlen, die Brightcove Player-Version zu verwenden 6.18.1+.
    3. Weitere Informationen zur Brightcove-Unterstützung für DRM finden Sie unter Überblick: Verwaltung digitaler Rechte (DRM) in der Video Cloud.
    Brightcove unterstützt derzeit keine Nur-Video- oder Nur-Audio-DRM-Wiedergabe.

    Terminologie

    Überprüfen Sie diese in diesem Dokument verwendeten Schlüsselbegriffe.

    Term Definition
    DRM Aus Wikipedia: Digital Rights Management (DRM) ist eine Klasse von Kopierschutztechnologien, die von Hardware- und Softwareherstellern, Herausgebern, Copyright-Inhabern und Einzelpersonen verwendet werden, um die Verwendung digitaler Inhalte und Geräte nach dem Verkauf zu kontrollieren.
    MPEG-DASH Aus Wikipedia: Dynamisches adaptives Streaming über HTTP (DASH), auch als MPEG-DASH bekannt, ist eine adaptive Bitraten-Streaming-Technik, die ein qualitativ hochwertiges Streaming von Medieninhalten über das Internet ermöglicht, das von herkömmlichen HTTP-Webservern bereitgestellt wird. Ähnlich wie bei der HTTP Live Streaming (HLS) -Lösung von Apple teilt MPEG-DASH den Inhalt in eine Folge kleiner HTTP-basierter Dateisegmente auf, wobei jedes Segment ein kurzes Intervall der Wiedergabezeit eines Inhalts enthält, dessen Dauer möglicherweise viele Stunden beträgt. wie ein Film oder die Live-Übertragung eines Sportereignisses.
    CENC Aus der ISO-Norm: Das allgemeine Verschlüsselungsschema 'CENC' spezifiziert Standardverschlüsselungs- und Schlüsselzuordnungsmethoden, die von einem oder mehreren digitalen Rechten und Schlüsselverwaltungssystemen (DRM-Systemen) verwendet werden können, um die Entschlüsselung derselben Datei unter Verwendung verschiedener DRM-Systeme zu ermöglichen.
    Verschlüsselte Medienerweiterungen (EME) Aus Wikipedia: EME ist ein W3C-Spezifikationsentwurf zur Bereitstellung eines Kommunikationskanals zwischen Webbrowsern und DRM-Agentensoftware (Digital Rights Management). Dies ermöglicht die Verwendung von HTML5-Videos zur Wiedergabe von DRM-umhüllten Inhalten, ohne dass Medien-Plugins von Drittanbietern wie Microsoft Silverlight erforderlich sind.

    Implementieren mit dem Modul Spieler

    Gehen Sie folgendermaßen vor, um das DRM-Plugin in Studio zu implementieren und DRM basierend auf Ihrer Kontoeinrichtung zu aktivieren:

    1. Öffne das SPIELER Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Player, zu dem Sie DRM-Funktionen hinzufügen möchten.
    2. Klicken Sie auf den Link für den Spieler, um die Eigenschaften des Spielers zu öffnen.
    3. Klicken Sie im linken Navigationsmenü auf Wiedergabe .
    4. Überprüfen Sie als nächstes die DRM aktivieren Kontrollkästchen.
    5. Um den Player zu veröffentlichen, klicken Sie auf Veröffentlichen & Einbetten von > Veröffentlichungsänderungen.
    6. Um den Öffnen-Dialog zu schließen, klicken Sie auf Schließen.

    DRM-Plugin-Architektur

    Das DRM-Plugin (videojs-drm ) ist ein Wrapper um zwei Plugins:

    • videojs-silverlight
    • videojs-contrib-eme

    Das videojs-drm Plugin Version 5 verwendet die integrierten DASH-Wiedergabefunktionen des Players. Hierbei wird VHS verwendet, die nächste und umbenannte Version des integrierten Geräts videojs-contrib-hls Plugin.

    Ab Spieler 6.26.0 unterstützt der Spieler DASH Multi-Periode. Für frühere Versionen des Players ist der Shaka-Player für DASH-Mehrperioden erforderlich.

    Wenn Sie den Shaka Player für die DASH-Wiedergabe wie in Version 4 verwenden möchten, können Sie das folgende Skript zusammen mit hinzufügen videojs-drm Skript der Version 5:

        https://players.brightcove.net/videojs-shaka/1/videojs-shaka.js
     

    Das videojs-silverlight Das Plugin ermöglicht die Wiedergabe von DASH-Inhalten in bestimmten Internet Explorer-Browsern.

    Das videojs-contrib-eme Das Plugin ermöglicht die Wiedergabe von FairPlay HLS-Inhalten.

    Hinweise:

    • Bei der Angabe von FairPlay-Optionen wird der Name des Plugins angezeigt videojs-contrib-emePlugin Name, eme.

    Verwendete Wiedergabetechnologien

    Brightcove Player verwendet verschiedene DRM-Wiedergabetechnologien mit verschiedenen Browsern. Im Folgenden werden die mit Brightcove Player verwendeten DRM-Technologien beschrieben:

    • Faires Spiel:: Apples DRM-System
    • PlayReady:: Microsoft DRM-System
    • Widevine:: Googles DRM-System

    In der folgenden Tabelle wird die Beziehung zwischen dem in Brightcove Player verwendeten Browser (neueste Version), Format und Wiedergabetechnologie beschrieben:

    Browser Format Playback-Technologie Wiedergabetyp verwendet
    DRM-Inhalte zu liefern
    Chrom-Desktop STRICH mit Widevine Mutter/EME MPEG-DASH
    Chrome Mobile1 STRICH mit Widevine Mutter/EME MPEG-DASH
    Internet Explorer2 STRICH mit PlayReady Silverlight MPEG-DASH
    Rand STRICH mit PlayReady Mutter/EME MPEG-DASH
    Safari HLS mit FairPlay Native HLS HLS
    Firefox STRICH mit Widevine Mutter/EME MPEG-DASH

    1Die DRM-Wiedergabe mit Chrome Mobile unter iOS wird nicht unterstützt.

    2IE11 unter Windows 10 und 8.1 und IE11 unter Win8 Metro verwenden Native / EME. Alle anderen IE-Versionen und -Konfigurationen verwenden das Silverlight-Plugin.

    Produzieren Sie DRM-Inhalte

    Es gibt zwei Schritte, die Sie ausführen müssen, um DRM-fähige Inhalte zu erstellen:

    1. Wenden Sie sich an Ihren Account Manager, um Ihr Konto DRM-fähig zu machen. Anschließend können Sie Ihr Konto mit den richtigen Lizenzschlüsseln konfigurieren und Profile aufnehmen, um die Erstellung von DRM-geschützten Inhalten zu ermöglichen.

    2. Produzieren Sie DRM-geschützte Inhalte. Sie können entweder neuen Inhalt hochladen oder vorhandenen Inhalt als DRM neu codieren. Dies erfolgt durch Auswahl der Profil aufnehmen das erzeugt Ihre gewünschte Verschlüsselungstechnologie.

      Weitere Informationen finden Sie in der Inhalte mit DRM aufnehmen Dokument.

    Sie müssen DRM-geschützten Inhalt erstellen, der entweder MPEG-DASH-Manifeste mit segmentierten und verschlüsselten Videos oder HLS FairPlay-Inhalt verwendet.

    Implementierung der FairPlay-Wiedergabe im Code

    Wenn Sie die FairPlay-Wiedergabe mithilfe des Einbettungscodes auf der Seite implementieren möchten, müssen Sie die folgenden Schritte ausführen, um das DRM-Plugin zu verwenden:

    1. Fügen Sie in der Kopfzeile Ihrer HTML-Seite das Stylesheet des Plugins ein: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Füge das JavaScript des Plugins ein: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>

    3. Wenn Ihr Konto für eingerichtet ist Dynamische Lieferung Rufen Sie einfach das EME-Plugin auf, um es zu initialisieren.

          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme();
    });
    </script>

       

      Wenn Ihr Konto NICHT eingerichtet ist für Dynamische Lieferung Anschließend müssen Sie das EME-Plugin mit FairPlay-Anmeldeinformationen aufrufen und konfigurieren.

          videojs.getPlayer('myPlayerID').ready(function() {
      var myPlayer = this;
      myPlayer.eme({
        "applicationId": "YOUR_APPLICATION_ID",
        "publisherId": "YOUR_PUBISHER_ID"
      });
    });
    </script>

      Brightcove Player sendet die FairPlay-Anmeldeinformationen an einen Video Cloud FairPlay-Zertifikatpfad.

    Führen Sie die folgenden Schritte aus, um das DRM-Plugin mit FairPlay-geschütztem Inhalt zu verwenden:

    1. Fügen Sie in der Kopfzeile Ihrer HTML-Seite das Stylesheet des Plugins ein: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Füge das JavaScript des Plugins ein: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>
    3. Rufen Sie in einem JavaScript-Block auf der Seite das EME-Plugin mit FairPlay-Anmeldeinformationen auf und konfigurieren Sie es, indem Sie Folgendes angeben getCertificate , getContentId und getLicense Funktionen zum EME-Plugin. Diese Funktionen sind spezifisch für Ihre eigene FairPlay-Lizenzserverimplementierung und können vor a direkt an das EME-Plugin übergeben werden src ist eingestellt: 
          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme({
          keySystems: {
            'com.apple.fps.1_0': {
            getCertificate: function (emeOptions, callback) {
            // request certificate
            // if err, callback(err)
            // if success, callback(null, certificate) where certificate
            // is a Uint8Array
          },
          getContentId: function (emeOptions, initData) {
            // return content ID as a string
            },
            getLicense: function (emeOptions, contentId, keyMessage, callback) {
              // request key
              // if err, callback(err)
              // if success, callback(null, key) as an arraybuffer
            }
          }
        }
      });
    </script>

    Beachten Sie, dass die FairPlay-Informationen auch als Teil von jedem übergeben werden können src Objekt. Das emeOptions werden als Parameter für alle Funktionen bereitgestellt. Sie beziehen sich auf Plugin-Optionen, die mit den Quelloptionen der aktuellen Quelle zusammengeführt (von diesen überschrieben) wurden. Es ist verfügbar, um den Zugriff auf Optionen zu vereinfachen, sodass Sie diese nicht selbst warten müssen.

        player.src({
      type: 'application/vnd.apple.mpegurl',
      src: 'http://www.example.com/path/to/master.m3u8',
      keySystems: {
        "com.apple.fps.1_0": {
          getCertificate: function(emeOptions, callback) { ... },
          getContentId: function(emeOptions, initData) { ... },
          getLicense: function(emeOptions, contentId, keyMessage, callback) { ... }
        }
      }
    });

    Zum Beispiel, wenn Sie eine verwenden müssen applicationId und publisherId für die getCertificate Anfrage können Sie Plugin-Optionen folgendermaßen übergeben:

        {
      keySystems: {
        "com.apple.fps.1_0": {
          getCertificate: function(emeOptions, callback) {
          var applicationId = emeOptions.applicationId; // 'application-id'
          var publisherId = emeOptions.publisherId; // 'publisher-id'
          // ...
        }
          // ...
        }
      },
      applicationId: 'application-id'
      publisherId: 'publisher-id'
    }

    Oder wenn Sie eine quellenspezifische benötigen publisherId Sie können es über die Quelloptionen überschreiben:

        // plugin options
    {
      keySystems: {
        "com.apple.fps.1_0": {
        getCertificate: function(emeOptions, callback) {
        var applicationId = emeOptions.applicationId; // 'application-id'
        var publisherId = emeOptions.publisherId; // 'source-specific-publisher-id'
        // ...
      },
        // ...
      }
    },
    applicationId: 'application-id'
    publisherId: 'publisher-id'
    }
    // source options
    player.src({
      src: '<URL>',
      type: 'application/vnd.apple.mpegurl',
      publisherId: 'source-specific-publisher-id'
    });

    Das Folgende ist eine Beispielimplementierung, bei der die Optionen als übergeben werden src Optionen, Abrufen der Inhalts-ID vom Hostnamen, festen Lizenz- und Zertifikat-URIs sowie einem Lizenz-URI, für den ein POST mit einem Hauptteil der Schlüsselnachricht erforderlich ist:

        var uint8ArrayToString = function(array) {
    return String.fromCharCode.apply(null, new Uint16Array(array.buffer));
    };
    var getHostnameFromUri = function(uri) {
      var link = document.createElement('a');
      link.href = uri;
      return link.hostname;
    };
    var getCertificate = function(emeOptions, callback) {
      videojs.xhr({
        uri: emeOptions.certificateUri,
        responseType: 'arraybuffer'
        }, function(err, response, responseBody) {
        if (err) {
          callback(err);
          return;
        }
        callback(null, new Uint8Array(responseBody));
      });
    };
    var getContentId = function(emeOptions, initData) {
      return getHostnameFromUri(uint8ArrayToString(initData));
    };
    var getLicense = function(emeOptions, contentId, keyMessage, callback) {
      videojs.xhr({
        uri: emeOptions.licenseUri,
        method: 'POST',
        responseType: 'arraybuffer',
        body: keyMessage,
        headers: {
          'Content-type': 'application/octet-stream'
        }
        }, function(err, response, responseBody) {
        if (err) {
          callback(err);
          return;
        }
        callback(null, responseBody);
      });
    };
    player.src({
      type: 'application/vnd.apple.mpegurl',
      src: 'http://www.example.com/path/to/master.m3u8',
      keySystems: {
        "com.apple.fps.1_0": {
        getCertificate: getCertificate,
        getContentId: getContentId,
        getLicense: getLicense
      }
      },
      certificateUri: 'http://example.com/fairplay/certificate/endpoint',
      licenseUri: 'http://example.com/fairplay/license/endpoint'
    });

    Implementierung der Widevine-Wiedergabe im Code

    Wenn Sie die Widevine-Wiedergabe mithilfe des Einbettungscodes auf der Seite implementieren möchten, müssen Sie die folgenden Schritte ausführen, um das DRM-Plugin zu verwenden:

    1. Fügen Sie in der Kopfzeile Ihrer HTML-Seite das Stylesheet des Plugins ein: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Füge das JavaScript des Plugins ein: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>
    3. Rufen Sie als Nächstes einfach das EME-Plugin auf, um es zu initialisieren. 
          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme();
      });
    </script>

    Für Widevine Modular-Inhalte müssen Sie Ihren Player so konfigurieren, dass er Ihren Widevine-Lizenzserver mithilfe des Quellhandlers des Players verwendet.

    Um eine Instanz eines Players auf Ihrer Webseite zu aktualisieren, können Sie den Quellhandler des Players verwenden. Hier ist ein Beispiel für die Verwendung von keySystemsOptions Array mit dem player.src() Funktion:

        player.src({
      src: 'http://example.com/my/manifest.mpd',
      type: 'application/dash+xml',
      keySystemOptions: [
      {
        name: 'com.widevine.alpha',
        options: {
        licenseUrl: 'http://m.widevine.com/proxy'
      }
      }]
    });

    Sie können auch die verwenden updateSourceData Funktion wie folgt:

        videojs.Html5DashJS.updateSourceData = function(source) {
    source.keySystemOptions = [{
    name: 'com.widevine.alpha',
    options: {
    serverURL:'https://example.com/anotherlicense'
    }
    }];
    return source;
    };

    Weitere Einzelheiten finden Sie in der videojs-contrib-dash Informationen zu GitHub.

    Unterstützung anderer DRM-Anbieter

    Das Plugin hat einen Pfad implementiert, über den Kunden Support für andere DRM-Anbieter implementieren können. Dies ist vor allem für Fairplay nützlich, da für Fairplay eine benutzerdefinierte Logik erforderlich ist, um Lizenzinformationen abzurufen. Hinzufügen von a vendor.name zu keySystems Bei einer bestimmten Quelle wird versucht, diese Logik zu verwenden und die erforderlichen Zertifikat- und Lizenzinformationen anzugeben. Ein Beispiel für Azure folgt:

        player.ready(function(){
      player.eme();
      player.src({
      src: 'http://example.com/src-url.m3u8'
      type: '',
      keySystems: {
        'com.apple.fps.1_0': {
        vendor: {
        name: 'azure'
      },
      certificateUri: 'https://example.com/your-certificate-uri.cer',
      licenseUri: 'https://example.com/your-license-uri'
      }
      }
      });
    });

    Hier ist ein Beispiel für castLabs:

        var player = videojs.getPlayer('myPlayerID');
    player.ready(function(){
      player.eme();
      player.src({
        src: 'http://example.com/src-url.m3u8'
        type: '',
        keySystems: {
          'com.apple.fps.1_0': {
          vendor: {
          name: 'castlabs',
          options: {
          authToken: 'your-auth-token',
          customData: 'your-custom-data'
        }
        }
        certificateUri: 'https://example.com/your-certificate-uri.cer',
        licenseUri: 'https://example.com/your-license-uri'
        }
        }
      });
    });

    In diesem Beispiel werden Widevine- und PlayReady-Quellen geladen:

        player.src({
      type: 'application/dash+xml',
      src: '<some src>',
         keySystems: {
      'com.widevine.alpha': '<license url>',
      'com.microsoft.playready': '<license url>'
      }
    });

    Bekannte Probleme

    • Das Standard (Auto Display) Beschriftungseinstellung in der Medien Module TEXTVERFOLGUNG Abschnitt wird nicht unterstützt, wenn er in Verbindung mit DRM- und In-Manifest-Untertiteln verwendet wird. In-manifeste Bildunterschriften werden beispielsweise mit den Brightcove Produkten Dynamic Delivery und SSAI verwendet. Eine Problemumgehung für dieses Problem besteht darin, das <track> Tag mit der Advanced Brightcove Player-Implementierung zu verwenden. Dies ist in der Programmgesteuertes Hinzufügen von Untertiteln zu Videos Dokument. Hinweis: Sie müssen das default Attribut mit dem <track> Tag verwenden.
    • DRM-Assets und Chrome:: Wenn Sie die Standard (iframe) Player-Implementierung mit dem DRM-Plugin verwenden, allow="encrypted-media" ist erforderlich, um DRM-Assets in Chrome abspielen zu können. 
          <iframe src="https://players.brightcove.net/123456789/BydO6uuuu_default/index.html?videoId=5783262319001"
    allowfullscreen
    webkitallowfullscreen
    mozallowfullscreen
    width="640" height="360"
    allow="encrypted-media"></iframe>
    • Sonderveranstaltung für Silverlight / IE11:: In den meisten Fällen, wenn Sie programmgesteuert mit dem Player interagieren möchten, warten Sie entweder auf den ready oder loadedmetadata  zu versendendes Ereignis. Wenn Sie jedoch programmgesteuert mit dem Player interagieren möchten, wenn Sie die Silverlight-Technologie in IE11 verwenden UND DRM-Inhalte abspielen, sollten Sie auf die warten canplay Veranstaltung.
    • Unter iOS bietet nur Safari die erforderlichen DRM-Browser-APIs (EME). WebViews unterstützen EME derzeit nicht. Daher funktioniert Fairplay DRM nur auf Safari.
