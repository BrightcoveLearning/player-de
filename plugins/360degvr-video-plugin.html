    360°/VR-Video-Plugin

    In diesem Thema erfahren Sie, wie dieses Plugin ein ordnungsgemäß aufgezeichnetes Videoelement in ein HTML5-Panorama-360 ° -Video verwandelt, das in Brightcove Player abgespielt werden kann.

    Dieses Plugin ist mit Brightcove Player Version 6.17.0 oder höher kompatibel.

    Beispiel Spieler

    Das folgende Beispielvideo zeigt die Verwendung des 360 ° / VR-Plugins. Sobald Sie auf das Video klicken, um es abzuspielen, können Sie entweder Ihre Maus oder die Pfeiltasten verwenden, um die Ansicht auf dem Desktop oder das Gyroskop zu steuern, wenn Sie es auf einem mobilen Gerät anzeigen. Die Wiedergabetaste für das 360 ° -Videologo verschwindet bei der Wiedergabe.

     

    Einführung

    Features

    Das 360 ° / VR-Plugin bietet folgende Funktionen:

    • Gyroskop-Integration auf Mobilgeräten:: Die Perspektive ändert sich, wenn Sie Ihr Telefon drehen.
    • Unterstützung für Karton / VR-Modus:: Auf Geräten, die dies unterstützen, wird das Cardboard Viewer-Symbol in der Steuerleiste angezeigt. Wenn diese Taste gedrückt wird, wird die Projektion so angepasst, dass sie mit VR-Brillen im Kartonstil funktioniert.
    • Video Cloud Media 360 ° -Einstellung:: Wenn Sie Video Cloud-Videos verwenden, berücksichtigt das Plugin die Einstellung von Video Cloud für 360 ° auf Video Cloud-Medien. Wenn der Wert nicht vorhanden ist, wird das Video als normales Video abgespielt. Mit dem 360 ° / VR-Plugin müssen Sie nicht sicherstellen, dass nur 360 ° -Videos auf Playern abgespielt werden.
    • Nicht-Video-Cloud-Medien:: Wenn du bist Keine Medien verwenden, die aus der Video Cloud bereitgestellt werden können Sie das Plugin weiterhin verwenden. Sie müssen die anrufen vr Plugin, das ein Objekt mit a projection Wert wie folgt eingestellt: 
      <script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.vr({projection: '360'});
  });
</script>
      Einstellen der projection Eigentum an 360 , equirectangular oder Sphere sind alle logisch gleichwertig.

      Der obige Code setzt natürlich voraus, dass Sie die JavaScript- und CSS-Dateien für das Plugin eingefügt haben, wie später in diesem Dokument beschrieben.

    Browser-Kompatibilität

    Für eine optimale Leistung in allen Browsern sollten Sie MP4-Wiedergaben gegenüber HLS für Videos bevorzugen, die dieses Plugin verwenden.

    Das Plugin ist mit den neuesten Versionen dieser gängigen Browser kompatibel:

    • Chrom-Desktop
    • Chrome Mobile (auf Android 6+)
    • Firefox Desktop
    • Rand
    • Safari Desktop *
    • Safari Mobile (iOS 11.2+) *
    * Wenn Sie das Plugin in Safari verwenden und ein Nicht-Video Cloud-Video verwenden (ein Video, das NICHT über Video Cloud aufgenommen wurde), müssen Sie das einstellen crossorigin Konfigurationseigenschaft zu anonymous.

    Das Plugin tut es NICHT mit Internet Explorer arbeiten.

    Einschränkungen

    Wenn Sie das 360 ° / VR-Plugin verwenden, sollten Sie Folgendes beachten:

    • MP4-Videowiedergaben werden für Videos, die dieses Plugin verwenden, gegenüber HLS bevorzugt.
    • Single Bitrate:: Du musst wählen Single Bitrate High als Aufnahmeprofil für 360 ° -Videos. Dies liegt daran, dass bei der 360 ° / VR-Wiedergabe jeweils nur ein kleiner Teil des Frames angezeigt wird (weitere Details hierzu finden Sie im nächsten Abschnitt dieses Dokuments). Jede andere Einstellung führt zu einer sehr schlechten Qualität. Wenn Sie Video Cloud nicht verwenden, empfehlen wir die Verwendung einer einzelnen hochauflösenden HLS-Wiedergabe oder eines MP4-Videos, um die Umschaltung der Wiedergabe zu deaktivieren.

    Überblick über die Implementierung

    Die Implementierung des Plugins ist unkompliziert. Führen Sie einfach die beiden folgenden Schritte aus:

    • Nehmen Sie das Video richtig auf.
    • Fügen Sie das Plugin dem Player hinzu.

    Nehmen Sie das 360 ° -Video auf

    Hinweis: Weitere Informationen zum Erstellen von 360 ° -Videos finden Sie im Arbeiten mit 360 ° -Videos Dokument.

    Sobald Sie das 360 ° -Video aufgenommen haben, müssen Sie es in die Video Cloud aufnehmen. Ein wenig Verständnis der Funktionsweise von 360 ° -Videos hilft Ihnen dabei, einige Gründe für die Auswahl des Aufnahmeprofils zu verstehen. Das 360 ° -Video wird gefilmt, um auf das Innere einer Kugel projiziert zu werden. Dies bedeutet, dass der sichtbare Bereich des Videos im Player nur einen Bruchteil des gesamten Videos ausmacht. Der folgende Screenshot gibt Ihnen eine ungefähre Vorstellung von diesem Konzept. Der Bereich innerhalb des roten Rechtecks ist im Player sichtbar, aber nur ein kleiner Teil des gesamten Videos.

    Da sich immer nur ein kleiner Teil des Videos im Player befindet, möchten Sie eine Wiedergabe von sehr hoher Qualität abspielen, ohne die Chance zu haben, auf eine Wiedergabe mit niedrigerer Qualität zurückzugreifen. Im Wesentlichen „zoomen“ Sie auf einen Teil des Videos und eine Wiedergabe von schlechter Qualität würde ein sehr schlechtes Seherlebnis bieten. Aus diesem Grund sollten Sie während des Aufnahmeprozesses ein Aufnahmeprofil von hoher Qualität und wenigen Wiedergaben verwenden. Brightcove empfiehlt die Verwendung von Single-Bitrate-High (2 Wiedergaben) Profil für 360 ° -Videos aufnehmen. Dieses Profil enthält eine hochwertige MP4-Wiedergabe, die bei Verwendung des 360 ° / VR-Plugins gegenüber HLS-Wiedergaben bevorzugt wird.

    Stellen Sie sicher, dass das aufgenommene Video für eine 360 ° -Projektion markiert ist, wie hier im Informationsbereich des Videos gezeigt:

    Implementieren mit dem Modul Spieler

    Um das 360 ° / VR-Plugin zu implementieren, fügen Sie den Namen des Plugins und die URLs zu den JavaScript- und CSS-Dateien des Plugins hinzu:

    1. Öffnen Sie das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Spieler, dem Sie das Plugin hinzufügen möchten.
    2. Klicken Sie auf den Link für den Spieler, um die Eigenschaften des Spielers zu öffnen.
    3. Klicken Sie im linken Navigationsmenü auf Plugins .
    4. Klicken Sie als Nächstes in den drei Optionen auf Plugins .
    5. Wählen Sie aus dem Plugin hinzufügen die Option Benutzerdefiniertes Plugin
    6. Für die Plugin Name eingeben vr.
    7. Geben Sie für die JavaScript-URL ein: 
      https://players.brightcove.net/videojs-vr/1/videojs-vr.min.js
    8. Geben Sie für die CSS-URL Folgendes ein: 
      https://players.brightcove.net/videojs-vr/1/videojs-vr.css
    9. Sie müssen keine Optionen für dieses Plugin eingeben.
    10. Klicken Sie auf Save.
    11. Klicken Sie im linken Navigationsmenü auf Styling .
    12. Wenn Sie der Meinung sind, dass dies ein besseres Seherlebnis ist, können Sie das Titel- und Beschreibungsdock deaktivieren. Deaktivieren Sie dazu das Kontrollkästchen Titel und Beschreibung anzeigen Kontrollkästchen.
    13. Um den Player zu veröffentlichen, klicken Sie auf Veröffentlichen & Einbetten von > Veröffentlichungsänderungen.
    14. Um den Öffnen-Dialog zu schließen, klicken Sie auf Schließen.

    Optionen

    forceCardboard

    Typ: boolean Standardwert: false

    Erzwingen Sie, dass die Kartontaste auf allen Geräten angezeigt wird.

    motionControls

    Typ: boolean Standardwert: true unter iOS und Android

    Legt fest, ob Bewegungs- / Kreiselsteuerungen aktiviert werden sollen.

    Projektion

    Typ: string Standardwert: auto

    Videotyp Wert für projection Möglichkeit
    Kugel 360 , Kugel oder gleichwinklig
    Würfel Würfel oder 360_CUBE
    KEIN 360 ° Video Keiner
    Wenn Sie überprüfen möchten, ob das aktuelle Video ein 360 ° -Video über ist player.mediainfo.projection AUTO
    Side-by-Side-360 ° -Video 360_LR
    360 ° -Video von oben nach unten 360_TB

    player.mediainfo.projection

    Typ: string Standardwert: undefiniert

    Dieser Wert sollte Quelle für Quelle festgelegt werden, um 360 ° -Videos je nach Video ein- und auszuschalten.

    debuggen

    Typ: boolean Standardwert: false

    Aktivieren Sie die Debug-Protokollierung für dieses Plugin.

    Advertising

    Derzeit ist die Unterstützung für Werbung in diesem Plugin nur auf IMA3-Pre-Rolls beschränkt. Andere Arten von Anzeigen und andere Anzeigenanbieter werden derzeit nicht unterstützt.

    Bekannte Probleme

    • Das Mischen von VR- und Nicht-VR-Inhalten im selben Player wird nicht unterstützt. Die Empfehlung für diese Art von Viewer-Erfahrung besteht darin, den Player nach Bedarf zu entfernen und neu zu erstellen. Sehen Brightcove Player Beispiel: Den Spieler dynamisch laden
    • Unter iOS 13 bittet die App den Benutzer um Erlaubnis, die Bewegungs- und Orientierungsfunktion (Gyroskop) des Geräts verwenden zu dürfen. Entwickler müssen die Berechtigungs-API verwenden, um diese Funktionalität zu implementieren. Siehe Apples Erlaubnis anfordern Dokument für weitere Informationen.
    • In Safari 12.x ist die Gyroskopintegration für mobile Geräte standardmäßig deaktiviert. Das Bewegungs- und Orientierungszugang Die Einstellungen müssen geändert werden, damit die Kreiselfunktion funktioniert.
    • Obwohl das 360 ° / VR-Plugin möglicherweise mit Live-Streams funktioniert, wird es offiziell nicht unterstützt.
    • Das neueste 360 ° / VR-Plugin ist mit Brightcove Player Version 6.17.0 und höher kompatibel.
    • Da das 360 ° / VR-Plugin eine MP4-Wiedergabe verwendet, ist dies der Fall nicht unterstützt mehrere Audiospuren (da Browser mit MP4s nicht mehrere Audiospuren unterstützen). Sie können dieses Problem vermeiden, indem Sie Aufnahmeprofile verwenden, die keine MP4-Wiedergaben für 360 ° -Videos mit mehreren Audiospuren enthalten.
    • Das 360 ° / VR-Plugin funktioniert nicht mit Internet Explorer.
    • Das 360 ° / VR-Plugin unterstützt DRM nicht.
    • Stereoskopische / gestapelte Videos werden vom Plugin nicht unterstützt.
    • Auf Mobilgeräten funktioniert nur horizontales Wischen (dies ändert die Ansicht horizontal), aber vertikales Wischen des Bildschirms bewirkt nichts. Das vertikale Wischen funktioniert zum Desktop-Browser.
    • Die Standardimplementierung (iframe) von Brightcove Player ändert NICHT die Ausrichtung.
    • Unter iOS kann das 360 ° -Plugin Videos korrekt rendern Laden Sie das Video vor der Wiedergabe vor Einstellung muss auf eingestellt sein Video vorladen (am schnellsten) im Spieler:
    • Unter iOS ist die Schaltfläche für den Vollbildmodus deaktiviert, da das 360 ° / VR-Plugin nicht mit dem nativen iOS-Player funktioniert. Aus diesem Grund wird empfohlen, die playsinline Attribut für die Nicht-Vollbild-Videowiedergabe, wie in der iOS und Brightcove Player Dokument.
