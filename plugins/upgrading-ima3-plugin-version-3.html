Überblick

Im Mai 2018 veröffentlichte Brightcove Version 3 des IMA Advertising Plugins. Das Update enthält viele wesentliche Verbesserungen der Zuverlässigkeit und anderer Bereiche, die im nächsten Abschnitt dieses Dokuments beschrieben werden. Das Update gilt nur für Brightcove Player Version 6 und höher. Bestehende Player der Version 6 werden nicht automatisch auf Version 3 migriert, da einige Integrationen möglicherweise aktualisiert werden müssen, um die neue Version zu unterstützen. Brightcove empfiehlt Ihnen, dieses Dokument vollständig zu lesen und die Änderungen vor dem Upgrade zu testen.

Verbesserungen

Das Folgende ist eine Liste der Verbesserungen, die in Version 3 des Plugins vorgenommen wurden:

  • Die Behandlung von Änderungen an Videoquellen durch das Plugin wurde grundlegend überarbeitet, was zu einer weitaus höheren Zuverlässigkeit bei der Verwendung mit Wiedergabelisten führt.
  • Das Festlegen der Server-URL kann jetzt verzögert werden, sodass asynchrone Workflows wie Header-Gebote oder Makros basierend auf Medieninformationen einfach integriert werden können. Das Folgende sind zwei Beispiele:
    • Das Starthilfe: Header Bidding Ad Dokument.
    • Das Implementieren Sie mit Code Abschnitt des IMA3-Plugin-Dokuments.
  • Es wurde ein Fehler behoben, der zu einer längeren Zeitüberschreitung als erwartet hätte führen können.
  • Die Flash-Unterstützung wurde entfernt, um die Leistung zu verbessern.
  • In der neuen Version sind verschiedene zusätzliche Fehlerbehebungen und Zuverlässigkeitsverbesserungen enthalten.

  • Sie können Anzeigenmakros Standardwerte angeben. In einem Makro kann ein Standardwert angegeben werden, in diesem Fall wird dieser Wert verwendet, wenn eine Variable nicht definiert ist. Die Syntax lautet:

    {macro=default}

    Zum Beispiel

    http://example.com/ad/{pageVariable.adConf=1234}

    würde auf Folgendes auflösen, wenn nicht definiert window.adConf ist:

    http://example.com/ad/1234

Technische Details

Um die Zuverlässigkeit zu verbessern, hat das Brightcove IMA-Plugin seine Abhängigkeit von aktualisiert Videojs-Contrib-Anzeigen , das Open Source Video.js-Anzeigenframework, auf Version 6. Dies ist ein wichtiges Wartungsupdate. Sie können mehr darüber erfahren Videojs-Contrib-Anzeigen Version 6 auf der Video.js Blog.

Dieses Update beseitigt eine Grundursache für einige Inhaltsblitze, die vor den Vorläufen aufgetreten sind. Die Lösung verwendet Video.js Middleware um die Notwendigkeit einer schnellen Wiedergabe zu beseitigen und zu Beginn der Wiedergabe anzuhalten.

Migration auf Version 3

Bestehende Player werden NICHT automatisch auf Version 3 des Brightcove IMA-Plugins aktualisiert. In der Standard (Legacy) Studio-Oberfläche wird wie in der Vergangenheit immer Version 2 des IMA-Plugins verwendet. In der Beta (neuen) Studio-Oberfläche wird die Werbung In diesem Abschnitt finden Sie eine Dropdown-Liste, in der Sie auswählen können, welche Version des IMA-Plugins verwendet werden soll. Diese neue Oberfläche kann zum Migrieren von Spielern verwendet werden. Brightcove empfiehlt dringend, Ihre Integration zu überprüfen und Ihren Player beim Upgrade sorgfältig zu testen.

Neu erstellte Player verwenden standardmäßig Version 3, wenn Werbung aktiviert ist. Sie können bei Bedarf manuell auf Version 2 eingestellt werden.

Im Folgenden finden Sie eine Liste der Änderungen, für die möglicherweise eine Aktualisierung der Integrationen erforderlich ist:

  • Möglicherweise möchten Sie testen, um sicherzustellen, dass Sie nicht länger von einem Zeitlimit abhängig sind als in Ihrer Konfiguration angegeben.
  • Das ended Ereignisse werden nicht länger um eine Sekunde verzögert. Wenn Ihre Integration reagiert ended Bei Ereignissen sollten Sie testen, ob die Integration weiterhin wie erwartet funktioniert. Im Allgemeinen bedeutet dies, dass solche Integrationen eine bessere Leistung erbringen.
  • Das player.ads.state Eigentum ist jetzt veraltet. Brightcove hat es in einigen Integrationen verwendet, um zu überprüfen, ob sich der Player im Anzeigenmodus befindet, aber es wird im Allgemeinen falsch ausgeführt. Eine bessere Alternative ist die Verwendung der player.ads.isInAdMode() Methode.
  • Das contentplayback Ereignis wurde entfernt. Es ist ein Implementierungsdetail, das nicht mehr benötigt wird, aber wir haben festgestellt, dass einige Integrationen es verwenden. Verwenden Sie die playing Ereignis stattdessen.
  • Der Beginn der Anzeigenwiedergabe wird durch das angezeigt ads-ad-started Ereignis, das genau einmal ausgelöst wird, wenn jede Anzeige beginnt. Während adplaying Das Ereignis wird weiterhin ausgegeben. Es kann jetzt mehrmals pro Anzeigenwiedergabe ausgegeben werden. Brightcove empfiehlt daher die Verwendung von ads-ad-started Ereignis stattdessen.

Warum ein Hauptversionsupdate?

Obwohl die Änderungen bescheiden erscheinen mögen, haben sich einige wichtige Aspekte der Benutzeroberfläche geändert, was zu einer größeren Versionsaktualisierung geführt hat. Insbesondere die Beseitigung der contentplayback Ereignis, sowie das spezifische Verhalten der timeout Parameter. Während diese geringfügig erscheinen mögen, können einige Integrationen von ihrem jeweiligen Verhalten abhängen. Da bereits kleine Änderungen bei einigen Kundenimplementierungen einen großen Unterschied bewirken können, ist Brightcove der Ansicht, dass ein größeres Versionsinkrement angemessen war, um anzuzeigen, dass Integrationen vor der Verwendung in der Produktion gründlich getestet werden sollten.
