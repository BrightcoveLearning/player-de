    Werbung mit dem FreeWheel Plugin

    In diesem Thema erfahren Sie, wie Sie das FreeWheel-Plugin verwenden, um Werbung mit Videowiedergabe zu aktivieren, und wie es mit Studio und durch benutzerdefinierte Codierung implementiert werden kann.

    Übersicht

    Dieses Plugin ermöglicht die FreeWheel-Anzeigentechnologie für Flash oder HTML5 im Brightcove Player. Die Standard-Rendering-Option für das FreeWheel-Plugin ist festgelegt Html5 (Weitere Informationen finden Sie in der adTechOrder Option unten).

    Verwenden von Versionen des Plugins

    Wenn Sie das FreeWheel-Plugin verwenden, müssen Sie versionierte URLs verwenden.

    Version Art Versionierte URL
    Neueste Erscheinung JS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    CSS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    SWF * https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf

    Als Referenz werden die Versionen des Plugins und kompatible SWFs in der verfolgt Plugin Versionsreferenz Dokument.

    Testen des Anzeigenservers

    Das erste, was Sie tun sollten, ist die Gültigkeit des von Ihnen geplanten Anzeigen-Tags zu überprüfen. Stellen Sie sicher, dass Sie die URL kopiert haben, und navigieren Sie zur folgenden Seite: Anzeigenvorschau (Wenn Sie auf diesen Link klicken, wird die Seite in einem neuen Fenster oder einer neuen Registerkarte geöffnet.)

    Fügen Sie Ihre Anzeigen-Tag-URL in das angegebene Formular-Eingabefeld ein. Klicken OPEN IN AD PREVIEWER. Ein Popup mit dem Titel Open In Ad Previewer wird angezeigt, und Sie klicken dann auf OPEN Schaltfläche zum Testen Ihrer Anzeige. Klicken PLAY Bei ordnungsgemäßer Funktion wird Ihre Anzeige mit einem Video durchsetzt angezeigt. Wenn Ihr Anzeigen-Tag in dieser Testumgebung nicht funktioniert, funktioniert es nicht mit Brightcove Player.

    Verwendung der richtigen SDK-URL

    Vor SDK Version 6.24 stellte FreeWheel eine URL für die automatische Aktualisierung auf die neueste Version des JS AdManger SDK bereit. Diese URL zeigte immer auf die aktuellste Version des SDK. Dies ist nicht mehr der Fall.

    Die nächsten beiden Abschnitte bieten Alternativen zur Implementierung von FreeWheel. In beiden Abschnitten verwenden Sie a sdkurl. In der Vergangenheit konnte man einfach verwenden http://adm.fwmrm.net/p/[]/AdManager.js für den Wert der Option und wissen, dass es auf die neueste Version des FreeWheel SDK verweisen würde. Jetzt müssen Sie eine URL eingeben, die auf eine bestimmte Version verweist, z https://mssl.fwmrm.net/libs/adm/6.24.0/AdManager.js. Und dazu Nach der Ersteinrichtung müssen Sie diesen Wert auch aktualisieren, wenn eine neuere Version des SDK von FreeWheel veröffentlicht wird, wenn Sie die neueste Version verwenden möchten. Beachten Sie, dass Sie nach dieser Änderung Ihren Brightcove Player erneut veröffentlichen müssen.

    Wenn Sie die URL nicht aktualisieren und die URL für das automatische Upgrade weiterhin verwenden, verweisen Sie für immer auf Version 6.23.

    Möglicherweise möchten Sie Ihre eigene SDK-Version in der Brightcove Player-Integration verwalten. Dies beinhaltet die Übergabe der Html5.sdkurl Option an das FreeWheel Plugin:

    player.FreeWheelPlugin({
  Html5: {
    sdkurl: 'preferred-sdk-url'
  }
});

    Implementieren mit dem Modul Spieler

    Gehen Sie folgendermaßen vor, um das FreeWheel-Plugin mithilfe des Players-Moduls zu implementieren:

    1. Öffnen Sie das PLAYSER-Modul und erstellen Sie entweder einen neuen Player oder suchen Sie den Spieler, dem Sie das Plugin hinzufügen möchten.
    2. Klicken Sie auf den Link für den Spieler, um die Eigenschaften des Spielers zu öffnen.
    3. Klicken Sie im linken Navigationsmenü auf Plugins .
    4. Klicken Sie als Nächstes in den drei Optionen auf Plugins .
    5. Wählen Sie aus dem Plugin hinzufügen die Option Benutzerdefiniertes Plugin
    6. Für den Plugin-Namen geben Sie ein FreeWheelPlugin.
    7. Geben Sie für die JavaScript-URL ein: 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    8. Geben Sie für die CSS-URL Folgendes ein: 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    9. Geben Sie die Konfigurationsoptionen im Textfeld Optionen (JSON) ein. Die im Formular bereitgestellten Optionen konfigurieren die Wiedergabetechnologie wo Flash und Html5 Konfigurationsabschnitte werden von FreeWheel bereitgestellt. Eine leicht veränderte (aus Sicherheitsgründen) vollständige Konfiguration ist unten dargestellt. Ein Stub-Beispiel wird hier gezeigt: 
      {
  "adTechOrder": ["html5"],
  "Flash": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  },
  "Html5": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  }
}
    10. Klicken Sie auf Save.
    11. Um den Player zu veröffentlichen, klicken Sie auf Veröffentlichen & Einbetten von > Veröffentlichungsänderungen.
    12. Um den Öffnen-Dialog zu schließen, klicken Sie auf Schließen.

    Rendering-Optionen

    Die folgende Tabelle zeigt die Technologie, in der das FreeWheel-Plugin eine Anzeige (Spalte 3) basierend auf der technischen Auswahl in Brightcove Player (Spalte 1) und der adTechOrder im Plugin einstellen (Spalte 2).

    Brightcove-Player
    Ausgewählte Tech    		 FreeWheelPlugin adTechOrder FreeWheel Plugin
    Ausgewählte Ad Tech
    html5 ["Flash“, „html5"] Flash
    html5 ["html5", "Flash"] html5
    html5 ["html5] html5
    html5 ["blinken"] Flash
    Flash ["Flash“, „html5"] Flash
    Flash ["html5", "Flash"] Flash
    Flash ["html5] Nicht möglich
    Flash ["blinken"] Flash
    Hinweis: Um die bestmögliche Benutzererfahrung bei der Bereitstellung von Midrolls zu erzielen, sollten Sie ein segmentiertes Bereitstellungsformat wie HLS verwenden und Ihre Inhalte von einem CDN bereitstellen, das Byterange-Anforderungen unterstützt.

    Konfiguration

    Die folgenden Einstellungen werden verwendet, um das FreeWheel-Plugin zu konfigurieren:

    • adTechOrder::
      • Typ: Array Default-Wert: ["html5"]
      • Das Array enthält die Anzeigenintegrationstechnologien, die in absteigender Reihenfolge verwendet werden sollen.
    • Flash::
      • Typ: Object
      • Ein Objekt für die technisch spezifische FreeWheel-Konfiguration (bereitgestellt von FreeWheel). Das Objekt enthält die folgenden Optionen:
        Diese Optionen können entweder dynamisch ( hier dargestellt) oder als Teil der Erstkonfiguration ( hierdargestellt) festgelegt werden.
      • swfurl::
        • Typ: string Standardwert: https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf (kompatibel mit FW Flash SDK v6.4.6)
        • Optionale Werte können basierend auf der FreeWheel Flash Plugin-Version verwendet werden. Diese Werte finden Sie in der Using versions of the plugin Abschnitt oben in diesem Dokument.
        • Muss eine untergeordnete Eigenschaft des Flash Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Flash-Anzeigentechnologie verwendet wird.
      • autoplay::
        • Typ: boolean Default-Wert: true
        • Gibt an, ob der Inhalt automatisch wiedergegeben wird.
        • Muss eine untergeordnete Eigenschaft des Flash Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Flash-Anzeigentechnologie verwendet wird.
      • unattendedPlay::
        • Typ: boolean Default-Wert: false
        • Wann autoPlay ist auf true gesetzt, unattendedPlay Gibt an, ob dem Benutzer die automatische Wiedergabe bekannt ist.
        • Muss eine untergeordnete Eigenschaft des Flash Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Flash-Anzeigentechnologie verwendet wird.
      • visitorCustomId:
        • Typ: string Default-Wert: unset
        • Ermöglicht Publishern, eine benutzerdefinierte Besucher-ID für die Anzeigenanforderung zu übergeben. Wenn festgelegt, wird dieser Wert im Besucherblock der Anforderungsnutzlast als übergeben customId="CustomID_001". Die ID wird in der Antwort auf die Anzeigenanforderung im öffnenden Besuchertag als Attribut zurückgegeben: customId="CustomID_001".
        • Muss eine untergeordnete Eigenschaft des Flash Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Flash-Anzeigentechnologie verwendet wird.
    • Html5::
      • Typ: Object
      • Ein Objekt für die technisch spezifische FreeWheel-Konfiguration (bereitgestellt von FreeWheel). Das Objekt enthält die folgenden Optionen:
        Diese Optionen können entweder dynamisch ( hier dargestellt) oder als Teil der Erstkonfiguration ( hierdargestellt) festgelegt werden.
      • autoPlayType::
        • Typ: integer Default-Wert: 1
        • Zeigt an, dass das Inhaltsvideo automatisch abgespielt wird und der Benutzer weiß, dass der Inhalt automatisch wiedergegeben wird.
        • Die an autoPlayType übergebenen Werte setzen Flags in der an den FreeWheel-Anzeigenserver gesendeten Anzeigenanforderungs-URL und können einer von drei ganzzahligen Werten sein, die den zugehörigen FreeWheel SDK-Konstanten entsprechen:
          • 1: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_ATTENDED:: Meint autoPlay ist true und unattendedPlay ist false. Dies setzt flag = +play-uapl.
          • 2: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_UNATTENDED: meint autoPlay ist true und unattendedPlay ist auch true. Dies setzt flag = +play+uapl.
          • 3: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_NONE: meint autoPlay ist false und unattendedPlay ist unset. Dies setzt flag = -play.
        • Muss eine untergeordnete Eigenschaft des Html5 Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Html5-Anzeigentechnologie verwendet wird.
      • visitorCustomId:
        • Typ: string Default-Wert: (optional)
        • Das visitorCustomId Mit dieser Option können Publisher eine benutzerdefinierte Besucher-ID für die Anzeigenanforderung übergeben. Wenn festgelegt, wird dieser Wert in der Anzeigenanforderungs-URL als queryString übergeben vcid=CustomID_001. Die ID wird als Antwort auf die Anzeigenanforderung im Besucher-JSON-Block als zurückgegeben "customId" : "CustomID_001".
        • Muss eine untergeordnete Eigenschaft des Html5 Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Html5-Anzeigentechnologie verwendet wird.
      • subsessionToken::
        • Typ: int Default-Wert: (optional)
        • Das subsessionToken Mit dieser Option können Publisher eine Subsession mit dem angegebenen Token starten. Wenn gesetzt, wird dieser Wert in der Anzeigenanforderungs-URL als QueryString übergeben: 
          ssto=1234567890
        • Muss eine untergeordnete Eigenschaft des Html5 Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Html5-Anzeigentechnologie verwendet wird.
          Hinweis: Für die subsessionToken Um ordnungsgemäß zu funktionieren, müssen Sie auch eine Funktion in die Anzeigenanforderung von aufnehmen [flag=+sync] als Parameter.
      • compatibleDimensions::
        • Typ: array Default-Wert: (optional)

        • Das compatibleDimensions Mit dieser Option können Publisher ein Array kompatibler Dimensionen übergeben. Beispiel:

          compatibleDimensions: [ {width: A, height: B}, {width: X, height: Y} ]

          Wenn gesetzt, wird dieser Wert in der Anzeigenanforderungs-URL als QueryString übergeben:

          cd=widthA,heightA|widthX,heightY
        • Muss eine untergeordnete Eigenschaft des Html5 Objekts sein. Dies wird direkt auf das Objektelement angewendet, wenn die Html5-Anzeigentechnologie verwendet wird.
    • loadWithCookie::
      • Typ: boolean Default-Wert: false
      • Aktiviert oder deaktiviert Cookies für VAST-Anforderungen.
    • requestAdsMode::
      • Typ: string Default-Wert: onload
      • Mögliche Werte für die Eigenschaft sind:
        • onload: Anzeigen werden angefordert, wenn der Player vollständig geladen ist.
        • onplay: Anzeigen werden angefordert, wenn das Video abgespielt wird, entweder vom Benutzer initiiert oder automatisch abgespielt.
        • oncue:: Anzeigen werden zum Startzeitpunkt des Cue-Points angefordert. Beachten Sie bei Verwendung dieser Eigenschaft auch:
          • Nur funktionsfähig, wenn die useMediaCuePoints Eigenschaft ist auch auf gesetzt true.
          • Video Content-Cue-Points müssen für Content-Videos eingerichtet werden. Siehe die Arbeiten mit Cue-Points im Medienmodul Dokument mit Anweisungen zum Erstellen von Cue-Points im Inhaltsvideo.
          • Wenn da ein ... ist temporalSlots Option unter der Flash oder Html5 Eigenschaften wird die Einstellung ignoriert.
      Hinweis: Bei Verwendung von Cue-Point-Anzeigen mit dem FreeWheel-Plugin werden Post-Roll-Anzeigen nicht unterstützt.
    • useMediaCuePoints::
      • Typ: boolean Default-Wert: false
      • Muss auf true gesetzt sein, um Video Cloud Ad Cue Points zum Auslösen von Anzeigen zu verwenden. Muss in Verbindung mit verwendet werden requestAdsMode: 'oncue' Eigenschaft und zugewiesener Wert.

    {mediainfo.tags} Tags (Metadaten), die mit dem Video verknüpft sind
    {mediainfo.reference_id} Reference ID
    {mediainfo.duration} Dauer des Videos wie von Video Cloud berichtet

    Veranstaltungen

    Das Plugin gibt während des Ladens, der Initialisierung und der Wiedergabe einige benutzerdefinierte Ereignistypen aus. Sie können die Anzeigenframework-Ereignisse wie jedes andere Ereignis abhören:

    player.on('ads-ad-started', function(event) {
  console.log('event', event);
});
    Ereignis Wird versandt wenn:
    Werbeanfrage Auf Anfrage Anzeigendaten.
    ads-load Wann Anzeigendaten nach einer Anzeigenanfrage verfügbar sind.
    werbestart Eine Anzeige hat begonnen zu spielen.
    mit Werbung beendete Eine Anzeige ist mit dem Abspielen fertig.
    erklagen-pausieren Eine Anzeige ist pausiert.
    ads-play Eine Anzeige wird aus einer Pause fortgesetzt.
    aufstigs-first-quartile Die Anzeige hat 25% ihrer Gesamtdauer gespielt.
    werfmittelpunkt Die Anzeige hat 50% ihrer Gesamtdauer gespielt.
    ads-dritte-quartile Die Anzeige hat 75% ihrer Gesamtdauer gespielt.
    ads-klick Ein Zuschauer klickte auf die abspielende Anzeige.
    ads-volumechange Das Volumen der abgespielten Anzeige wurde geändert.
    Ads-Pod gestartet Die erste Anzeige in einem linearen Ad-Pod (einer sequenzierten Gruppe von Anzeigen) hat begonnen.
    Ads-Pod beendet Die letzte Anzeige in einem linearen Ad-Pod (einer sequenzierten Gruppe von Anzeigen) ist beendet.
    ads-allpods-abgeschlossen Alle linearen Anzeigen haben die Wiedergabe beendet.
    fw-before-ad-request Dieses Ereignis wird auf dem Player-Objekt angezeigt und ausgelöst, bevor eine Anzeigenanforderung gesendet wird. Normalerweise wird es im Kontext von Wiedergabelisten verwendet, um Aktualisierungen der FreeWheel-Konfigurationseinstellungen vorzunehmen über: player.FreeWheelPlugin.settings.Html5 oder player.FreeWheelPlugin.settings.Flash

    Dynamische Server-URL-Zuweisung

    Du kannst den ... benutzen fw-before-ad-request Ereignis zum dynamischen Zuweisen der Server-URL. Sie werden die verwenden on() Methode, um auf die Anzeigenanforderung zu warten und dann die gewünschte Server-URL zuzuweisen. Natürlich müssen Sie Ihre gewünschten Server-URLs für die Platzhalter im Code angeben:

    player.on('fw-before-ad-request', function () {
  player.FreeWheelPlugin.settings.Html5.serverUrl = '[your server url]'
  player.FreeWheelPlugin.settings.Flash.serverUrl = '[your server url]'
})

    Wenn Sie die Server-URLs zuvor konfiguriert haben, überschreibt der angezeigte Code die vorherige Konfiguration.

    Name Data Type (Datenart) Beschreibung

    Bekannte Probleme

    Dynamische Wiederverwendung des gleichen Players mit FreeWheel

    Wenn Sie Videos dynamisch laden und denselben Player erneut verwenden, enthalten die FreeWheel-Konfigurationseinstellungen mit zwei Ausnahmen dieselben Werte wie zum Zeitpunkt der Instanziierung des FreeWheel-Plugins. Die erste Ausnahme ist, wenn Sie Anzeigenmakros verwenden. Die zweite Ausnahme ist, dass Sie die Einstellungen mit aktualisieren können player.FreeWheelPlugin.settings.Html5 oder player.FreeWheelPlugin.settings.Flash auf einem fw-before-ad-request Veranstaltung.

    Ändern der Größe des Players während der Anzeigenwiedergabe

    Wenn die Größe des Players während der Wiedergabe von Anzeigen oder Videos geändert wird, wird die Größe des Anzeigeninhalts nur geändert, wenn die Dimensionsfunktion des Players aufgerufen wird, um die Größe des Players zu ändern. Durch Ändern der Größe des Players mithilfe anderer Methoden (z. B. Stilbreite und -höhe) wird die Größe der Anzeige nicht geändert.

    Beim Benutzen player.dimensions(width,height) Um die Größe des Players zu ändern, müssen Sie das auslösen fw-resizeplayer Ereignis, damit das Plugin weiß, dass sich die Abmessungen geändert haben. Dies liegt daran, dass einige VPAID-Anzeigen die Standardabmessungen der Anzeige beibehalten und die Größe nicht automatisch ändern, wenn die Größe des Players geändert wird.

    Beispiel:

    player.dimensions(960,540);
player.trigger('fw-resizeplayer');

    Anzeigen überlagern

    Wenn das FreeWheel-Plugin eine Anzeige in Flash rendert, können Overlay-Anzeigen nicht angeklickt werden. Wenn das FreeWheel-Plugin eine Anzeige in HTML5 rendert, werden Overlays nicht an den richtigen Koordinaten angezeigt. Durch Klicken auf eine HTML5-Overlay-Anzeige wird der Content Player nicht angehalten, wenn die Klick-URL befolgt wird.

    FreeWheel Live

    FreeWheel-Anzeigen in einem Live-Stream können mit Unterstützung von implementiert werden Brightcove Global Services (BGS).

    VPAID

    Es ist nicht möglich, Anzeigen über VPAID-Steuerelemente fortzusetzen, wenn die Anzeigensteuerungsleiste aktiviert ist.
