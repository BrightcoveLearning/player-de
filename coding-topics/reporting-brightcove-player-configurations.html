Dieses Beispiel und der zugehörige Code werden als Leitfaden für Ihre Produktionsentwicklung bereitgestellt. Der Code ist weder für die Verwendung in der Produktion zertifiziert noch offiziell unterstützt.

Berechtigung abrufen

Um diese Konfigurationsberichts-App verwenden zu können, benötigen Sie die richtigen Anmeldeinformationen.

Der einfachste Weg, um Anmeldeinformationen zu erhalten, ist in den meisten Fällen der Abschnitt Studio-Admin-API-Authentifizierung (erfordert Administratorberechtigungen für Ihr Konto). Weitere Informationen finden Sie unter Verwalten von API-Authentifizierungsdaten. Sie erhalten die folgende Einzelberechtigung:

Wenn die von Ihnen benötigten Berechtigungen in Studio nicht verfügbar sind oder Sie sie lieber direkt von der OAuth-API beziehen möchten, verwenden Sie Ihre Wahl der Kundenanmeldeinformationen abrufen unten aufgeführten Dokumente. Für welche Option Sie sich auch entscheiden, Sie müssen nach den richtigen Betriebsberechtigungen fragen. Folgendes kann mit cURL oder Postman verwendet werden, um alle Berechtigungen für die Player-Verwaltungs-API abzurufen:

  "operations": [
    "video-cloud/players/read"
  ]
  • OAuth: Client-Anmeldeinformationen mit cURL abrufen
  • OAuth: Client-Anmeldeinformationen mit Postman abrufen

Beachten Sie das Folgende:

  • Der Bericht, in dem alle in die Player eines Kontos geladenen Plugins angezeigt werden, funktioniert nicht mit Playern der Version 5 und älter.
  • Auch im Plugins-Bericht Plugins, die mit beginnen Brightcove / sind von Brightcove gelieferte Plugins. Die anderen sind entweder benutzerdefinierte Plugins oder von Brightcove bereitgestellte Plugins, die manuell über Studio oder Curl geladen werden.

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (The proxies used by Brightcove Learning Services only accept requests from Brightcove domains.) Ein Beispielproxy, der dem von uns verwendeten sehr ähnlich ist, aber ohne die Prüfungen, die Anfragen von Nicht-BrightCove-Domänen blockieren, kann in diesem GitHub-Repository gefunden werden. Dort finden Sie auch grundlegende Anweisungen zur Verwendung und eine ausführlichere Anleitung zum Erstellen von Apps rund um den Proxy unter Verwenden der REST-APIs.
