    Kurzanleitung: Erstellen Sie einen Livestream mit der Brightcove Live API

    Dieses Tutorial führt Sie durch die Erstellung eines Live-Stream-Jobs mit der Live-API.

    Was du brauchst

    Um dieses Tutorial abzuschließen, benötigen Sie Folgendes.

    • Ein Konto, das für die Live-API eingerichtet wurde
    • Ein API-Schlüssel für die Live-API (wird Ihnen bei der Einrichtung des Kontos zur Verfügung gestellt)
    • Ein Video Cloud-Konto (nicht erforderlich, um den Livestream-Auftrag zu erstellen, aber zum Einrichten der Wiedergabe im Brightcove Player erforderlich)
    • Wenn Sie die verwenden, CMS API um das Video Cloud-Video anstelle von Studio zu erstellen, benötigen Sie Client-Anmeldeinformationen mit Video-/Lese- und Video-/Schreibrechten. Wenn Sie mit dem Erhalten von Client-Anmeldeinformationen nicht vertraut sind, lesen Sie die Abschnitt zur Authentifizierung unten.
    • Die Möglichkeit, REST-API-Anfragen entweder über cURL oder einen REST-Client wie Insomnia oder Postmanzu stellen.

      Um Ihnen zu helfen, haben wir einige Insomnien- und Postman-Importe vorbereitet, die Sie für die Übung verwenden können (bei Brightcove Learning Services bevorzugen und verwenden wir Insomnia, aber Postman wird auch funktionieren, wenn Sie das haben):

      In jeder der ZIP-Dateien finden Sie eine Readme-Seite, die Ihnen dabei hilft.

      • Insomnia Workspace importieren
      • Briefträger-Sammlung und Umwelt-Im
    • Um einen tatsächlichen Livestream zu erstellen, benötigen Sie entweder einen Software- oder Hardware-Encoder für Live-Streaming. Die folgenden Anweisungen enthalten Schritte zum Einrichten des Livestreams mit Wirecast ( für eine Testversion anmelden). Wenn Sie einen anderen Encoder verwenden, ist der Prozess unterschiedlich, sollte jedoch in Bezug auf die Informationen, die Sie aus der Live-API-Antwort benötigen, ähnlich sein.

    Einrichten von Video Cloud-Video

    Jetzt, da wir die Live-Stream-Informationen haben, werden wir in Video Cloud ein Remote-Video einrichten, damit wir es in einem Brightcove Player abspielen können.

    In diesen Schritten richten wir das Video mit der CMS-API ein, dies kann jedoch auch in Video Cloud Studio erfolgen, wenn Sie es vorziehen - sehen Sie sich die folgenden alternativen Schrittean.

    Wenn Sie Insomnia oder Postman verwenden, müssen Sie die OAuth2-Authentifizierung mit Ihren Client-Anmeldeinformationen einrichten. Lesen Sie eine der folgenden Anleitungen, wenn Sie Hilfe benötigen:

    • Verwenden Sie Insomnia für API-Anfragen
    • Verwenden Sie Postman für API-Anfragen

    1. Zunächst müssen Sie ein neues Video erstellen, indem Sie eine POST Anfrage an folgende Adresse stellen:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos

    2. Der Anfragetext kann mehrere Metadaten enthalten (Einzelheiten finden Sie in der API-Referenz ), aber das einzige, was erforderlich ist, ist ein name:

      {
    "name": "some name for your live stream"
}

    3. Die Antwort enthält alle Video-Metadaten, aber das wichtige Feld ist das id, das Sie benötigen, um das Remote-Asset hinzuzufügen.

    4. Um den HLS-Stream hinzuzufügen, stellen Sie eine weitere POST Anfrage an:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id/assets/hls_manifest

    5. Diesmal ist der Anfragetext:

      {
    "remote_url": "playback_url for your live stream"
}

    Video zu einem Spieler hinzufügen

    Jetzt erstellen wir eine einfache HTML-Seite und fügen mit dem gerade erstellten Video einen Player hinzu.

    Auch hier werden wir Studio verwenden, um den Player-Veröffentlichungscode zu erhalten - Sie können dies auch mit der Player-Management-API tun, wenn Sie dies bevorzugen.
    1. Erstellen Sie eine einfache HTML-Seite, um den Live-Stream-Player zu testen. Sie können jeden anderen oder gar keinen Inhalt einschließen.
    2. Wählen Sie im Studio-Medienmodul das in den vorherigen Schritten erstellte Video aus und klicken Sie auf > Web Player veröffentlichen und einbetten:
    3. Wählen Sie im Dialogfeld „Veröffentlichen“ einen Player aus (der Brightcove Default Player ist in Ordnung):
    4. Scrollen Sie nach unten, legen Sie die Größe des Players nach Ihren Wünschen fest und klicken Sie dann auf den Codeblock, um den Standard-Player-Code auszuwählen - kopieren Sie diesen Code und fügen Sie ihn in den Hauptteil Ihrer HTML-Seite ein:
    5. Speichern Sie Ihre HTML-Seite auf einem Remote- oder lokalen Webserver, damit Sie sie durchsuchen können.
    6. Öffnen Sie die Seite in Ihrem Webbrowser. Da wir noch nicht mit dem Dämpfen begonnen haben, wird ein Medienfehler angezeigt:

    Verwenden von Wirecast zum Streamen eines Ereignisses

    In diesem Abschnitt wird die Konfiguration und Verwendung des Codierers von Telestream Wirecast zur Unterstützung einer Live-Streaming-Veranstaltung erläutert. Die Software von Telestream Wirecast ist eine Desktopanwendung, die die Eingabe Ihrer Kamera erfasst und einen Stream erzeugt, der in einem CDN bereitgestellt werden kann. Beachten Sie, dass eventuell andere Codierungslösungen auf Hardware- und Softwarebasis verfügbar sind, die sich möglicherweise besser für die Bereitstellung Ihrer Live-Streaming-Veranstaltung eignen. Laden Sie eine Testversion der Wirecast-Software auf der Website von Telestream herunter.

    Hinweis: Installieren Sie die Wirecast-Software und verbinden Sie eine Kamera mit Ihrem Computer, bevor Sie fortfahren.

    Führen Sie die folgenden Schritte aus, um Wirecast für eine Live-Veranstaltung zu konfigurieren:

    1. Öffnen Sie Wirecast.
    2. Bewegen Sie den Mauszeiger über +, klicken Sie auf Capture Devices und wählen Sie dann Ihre Kamera
    3. Klicken Sie auf die Pfeilschaltfläche ( ) , damit die Kamera den Liveshot aufgenommen hat.
    4. Klicken Sie auf die Schaltfläche Stream ( ). Sie werden aufgefordert, die Ausgabeeinstellungen zu konfigurieren.
    5. Legen Sie als Ziel den RTMP-Server fest. Klicken Sie auf OK.
    6. Verwenden Sie für die Adresse die von der Live-API zurückgegebene stream_url( Schritt 3).
    7. Verwenden Sie für den Stream lebendig.
    8. (Optional) Klicken Sie auf Add..., um weitere Ausgangsstreams hinzuzufügen. Legen Sie beim Erstellen von weiteren Ausgangsstreams ein eigenes benutzerdefiniertes Codierungsprofil an, um die Bitrate der Ausgabe zu steuern. Achten Sie darauf, dass der Name des Streams die neue Bitrate widerspiegelt.
    9. Klicken Sie auf OK, um die Ausgabeeinstellungen zu speichern.
    10. Klicken Sie auf die Schaltfläche Stream ( ). Der Live-Stream sollte starten.

    Fazit

    Sie sind jetzt bereit, eine Live-Streaming-Sitzung zu erstellen. Sie müssen Ihrer Encoder-Konfiguration Folgendes hinzufügen:

    • stream_url: Füge das stream_url von deinem Job hinzu (die rtmp Adresse)
    • stream_name: Der Streamname muss alive

    Sobald Sie den Live-Stream gestartet haben, sollten Sie sehen, dass er im Brightcove-Player angezeigt wird, den Sie in den vorherigen Schritten erstellt haben.
