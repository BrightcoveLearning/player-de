    Übersicht: Brightcove Live API

    In dieser Übersicht erfahren Sie, wozu die Live-API dient und wie sie verwendet wird. Zu den in diesem Dokument enthaltenen Themen gehören unterstützte AWS-Regionen und CDNs, Live-Channels und Ereignisse sowie das Einfügen von ID3-zeitgesteuerten Metadaten in einen Live-Stream.

    Einführung

    Die Live API ist eine REST-basierte API, mit der Sie Live-Streaming-Ereignisse erstellen und verwalten können. Zu den optionalen Funktionen gehören:

    • Serverseitige Anzeigeneinfügung ( SSAI )
    • AES-128 Verschlüsselung
    • Erstellen von Video-on-Demand-Assets aus Clips aus dem Live-Stream
    • DVR Fähigkeit
    • Mehrere CDNs

    Siehe auch die API-Referenz.

    Basis-URL

    Die Basis-URL für den Live API lautet:

          https://api.bcovlive.io

    Header

    Alle Anfragen werden mit einem API-Schlüssel authentifiziert, der Ihnen bei der Einrichtung Ihres Kontos zur Verfügung gestellt wird. Der Schlüssel wird in einem X-API-KEY Header übergeben. Ein Content-Type Header ist ebenfalls erforderlich:

          X-API-KEY : YOUR_APIKey
      Content-Type: application/json

    Unterstützte AWS-Regionen

    Die folgenden AWS-Regionen werden unterstützt:

    Unterstützte AWS-Regionen
    Ort AWS Name SSAI-Unterstützung
    Oregon us-west-2
    Virginia us-east-1
    Tokio ap-northeast-1
    Singapur ap-southeast-1
    Sydney ap-southeast-2
    Mumbai ap-south-1
    Frankfurt eu-central-1
    Irland eu-west-1

    Beachten Sie, dass SEP-Jobs auf ein Konto beschränkt sind, bei dem das Standardlimit 3 beträgt, mit Ausnahme des US-West-2 , dass seine Beschränkung bis zu 10 beträgt. Alle Einschränkungen sind nach Konto und nicht nach Regionen festgelegt.

    Unterstützte CDNs

    Die folgenden CDNs werden für Live-Streaming unterstützt:

    • Akamai
    • Cloudfront

    Andere dateibasierte CDNs sollten funktionieren, wurden aber nicht getestet und werden nicht aktiv unterstützt.

    Kanäle und Event-Stunden

    Es gibt zwei Kaufoptionen für Live:

    • Erwerben Sie Ereignisstunden der Streaming-Zeit
    • Streaming-Kanäle kaufen

    Sie können auch sowohl Event-Streaming-Stunden als auch Kanäle erwerben. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu den Angeboten zu erhalten.

    Wenn Sie sowohl Event-Streaming-Stunden als auch Kanäle erworben haben, müssen Sie Folgendes beachten:
    • Standardmäßig werden Live-Jobs gegen Event-Streaming-Stunden abgerechnet.
    • Wenn Sie möchten, dass ein Job stattdessen als Channel-Job abgerechnet wird, müssen Sie beim Erstellen des Live-Jobs das folgende Feld in den Anfragetext aufnehmen:

      "channel_type": "24x7"

    • Nur-Channel-Kunden müssen beim Erstellen des Live-Jobs immer das folgende Feld in den Anfragetext aufnehmen:

      "channel_type": "24x7"

    • Kunden, die nur Event-Stunden Streaming-Zeit gekauft haben, müssen nicht einschließen "channel_type": "24x7" in Anfragen.

    Token-Authentifizierung

    Brightcove bietet die Möglichkeit, Token-Authentifizierung zu Wiedergabe-URLs für Live-Videostreams hinzuzufügen. Wenn Sie eine Token-Authentifizierung hinzufügen möchten, wenden Sie sich an den Brightcove Support. Es kann bis zu drei Tage dauern, bis die Token-Authentifizierung eingerichtet ist.

    Die TTL (Time-to-Live) für die Token kann auf einen beliebigen Wert von einer Stunde bis zu 365 Tagen eingestellt werden. Wie lange Sie die TTL festlegen, hängt davon ab, welche Arten von Live-Streams Sie bereitstellen. Beachten Sie jedoch, dass die TTL eine kontoweite Einstellung ist und für alle Live-Streams gilt.

    DVR Fähigkeit

    Brightcove Live-Streams haben DVR die Fähigkeit. Um diese Funktion nutzen zu können, müssen Sie:

    • Verwenden Sie die playback_url_dvr URL für die Wiedergabe
    • Verwenden Sie einen Spieler, der hat DVR Fähigkeit

    Die DVR-Funktion ist auf 86.400 Sekunden begrenzt.

    Der DVR Stream bleibt nach Abschluss des Live-Streams 7 Tage lang verfügbar.

    Endpunkte und Vorgänge

    Die Hauptoperationen für Live API sind das Erstellen und Verwalten von Live-Streams und das Generieren von VOD-Clips aus Live-Streams. Diese Vorgänge werden durch Anfragen an folgende Endpunkte durchgeführt, die im Rest des Dokuments ausführlicher erläutert werden.

    Beachten Sie, dass Brightcove Live eine eigene Konto-ID hat, die von allen anderen Konto-IDs getrennt ist. account_id Verweist in den API-Vorgängen auf Ihre Live-Konto-ID.

    Schaffung und Verwaltung von Arbeitsplätzen

    Mit diesen Vorgängen können Sie einen Live-Job erstellen, die Details dazu abrufen und ihn stoppen. Es gibt auch einen Endpunkt, um einen sofortigen Cue-Point für eine Werbeunterbrechung zu erstellen.

    Erstellen Sie einen Live-Job

          POST https://api.bcovlive.io/v1/jobs

    Dieser Endpunkt wird verwendet, um Livestreams über eine POST Anfrage zu erstellen. Neben der Angabe von Eigenschaften des Live-Streams selbst kann die Anforderung auch VOD-Clips angeben, die aus dem Live-Stream generiert werden sollen (dies kann auch später über den Endpunkterfolgen). Details zu den Feldern, die in den Anfragetext aufgenommen werden können, finden Sie in der API-Referenz.

    Input-Protokoll

    Brightcove Live unterstützt mehrere Eingabeprotokolle. Verwenden Sie das protocol Feld im Anforderungsnachrichtentext beim Erstellen des Auftrags, um das Feld anzugeben, das Sie verwenden möchten. Unterstützte Werte sind:

    • rtmp ( Die Standardeinstellung)
    • rtp
    • rtp-fec
    • srt

    Das RTMP-Protokoll dient zur Bereitstellung eines Streams im FLV-Format. Die anderen Protokolle dienen zur Lieferung von MPEG2-TS.

    Wenn du benutzt rtp , rtp-fec oder srt müssen Sie auch a angeben cidr_whitelist (sehen Classless Inter-Domain Routing).

    Wenn Sie verwenden rtmp, können Sie stattdessen eine ip_whitelist für die Eingabe angeben, dies ist jedoch nicht erforderlich.

    Beispiel Anfragetext für RTP+FEC-Job:

    {
    "live_stream":true,
    "region":"us-west-2",
    "reconnect_time":300,
    "outputs":[
      {
        "label": "hls720p",
        "live_stream": true,
        "height": 720,
        "video_bitrate": 800,
        "segment_seconds": 6,
        "keyframe_interval": 90
      }
    ],
    "protocol": "rtp-fec",
    "cidr_whitelist": ["127.0.0.1/32"]
}

    Der Live API Quick Start führt Sie durch die Erstellung eines Live-Stream-Jobs und die Einrichtung eines Brightcove Players zum Abspielen.

    Live-Jobs auflisten

          GET https://api.bcovlive.io/v1/jobs

    Dieser Endpunkt wird verwendet, um Ihre Live-Streams über eine GET Anfrage aufzulisten. Der Endpunkt unterstützt Paginierung, Sortierung und Suchfilterung. Details zu den Feldern, die in den Anfragetext aufgenommen werden können, finden Sie in der API-Referenz und einige zusätzliche Informationen finden Sie unter Liste der Live- oder VOD-Jobs abrufen.

    Details zu Live-Jobs abrufen

          GET https://api.bcovlive.io/v1/jobs/:jobId

    Dieser Endpunkt ermöglicht es Ihnen, detaillierte Informationen zu einem Live-Stream abzurufen, der auch zurückgegeben wird, wenn Sie den Job ursprünglich erstellt haben. Einzelheiten zu den Antwortfeldern finden Sie in der API-Referenz .

    Manuelle Ad Cue-Point-Ein

          POST https://api.bcovlive.io/v1/jobs/:jobId/cuepoint

    In der Regel sendet Ihr Encoder Cue-Points für Werbeunterbrechungen, aber Sie können auch eine sofortige Werbeunterbrechung erstellen, indem Sie eine Anfrage an diesen Endpunkt senden. Einzelheiten finden Sie in der API-Referenz .

    Beachten Sie, dass ein timecode im Formular für den Cue-Point erforderlich DD:HH:MM:SS ist.

    Stoppen Sie einen Live-Job

          PUT https://api.bcovlive.io/v1/jobs/:jobId/cancel

    Verwenden Sie diesen Endpunkt, um einen Live-Stream sofort anzuhalten. Nach der Kündigung kann ein Live-Stream nicht mehr neu gestartet werden. Einzelheiten finden Sie in der API-Referenz .

    Erstellen von Clips

    Sie können Video-on-Demand-Clips aus einem Live-Stream erstellen und in einem Video Cloud-Konto speichern oder an S3-Bucket oder FTP-Adresse senden. Sie können die Clips definieren, wenn Sie den Live-Stream erstellen, oder sie später mit dem unten beschriebenen Endpoint erstellen. Siehe auch den Leitfaden zum Erstellen von Clips .

    VOD-Clip erstellen

          POST https://api.bcovlive.io/v1/vods

    Die Start- und Endpunkte für die Clips können in Bezug auf Offsets ab dem Start des Streams oder UNIX-Zeitstempeln definiert werden. Details zu den Feldern des Anforderungskörpers finden Sie in der API-Referenz.

    Beachten Sie, dass Clips aus Livestreams bis zu 7 Tage nach Ende des Events erstellt werden können.

    Holen Sie sich eine Liste von VOD (Clip) Jobs

    Informationen zum Abrufen einer Liste Ihrer VOD-Jobs für Clips finden Sie unter Abrufen einer Liste von Live- oder VOD-Jobs und die API-Referenz.

    Managen SSAI

    Mithilfe der serverseitigen Anzeigeneinfügung ( SSAI) können Sie beliebig viele Werbeblöcke in Ihren Live-Stream einfügen. Sie können auch Slate-Assets (VOD-Clips) aufnehmen, um jede ungenutzte Werbezeit mit einer Willensnachricht oder was immer Sie möchten zu füllen.

    Weitere Details zur Einrichtung SSAI finden Sie unter Serverseitige Anzeigeneinfügung mit dem Brightcove Live API und der API-Referenz.

    Konto-Anzeigenkonfigurationen ab

          GET https://api.bcovlive.io/v1/ssai/applications/:account_id

    Mit diesem Endpunkt können Sie alle Anzeigenkonfigurationen abrufen, die für ein Konto eingerichtet wurden. Details zu den Antwortfeldern finden Sie in der API-Referenz.

    Anzeigenkonfiguration erstellen

          POST https://api.bcovlive.io/v1/ssai/application

    Erstellen Sie eine Anzeigenkonfiguration, die definiert, wie Anzeigen abgerufen werden SSAI. Details zu den Feldern des Anforderungskörpers finden Sie in der API-Referenz.

    Anzeigenkonfiguration holen

          GET https://api.bcovlive.io/v1/ssai/application/:application_id

    Verwenden Sie diesen Endpunkt, um die Details einer von Ihnen erstellten Anzeigenkonfiguration abzurufen. Details zu den Antwortfeldern finden Sie in der API-Referenz.

    Anzeigenkonfiguration aktualisieren

          PUT https://api.bcovlive.io/v1/ssai/application/account/:application_id

    Aktualisieren Sie die Details einer Anzeigenkonfiguration. Details zu den Feldern des Anforderungskörpers finden Sie in der API-Referenz.

    Erhalten Sie Slate Media Source Assets

          GET https://api.bcovlive.io/v1/ssai/slates/:ACCOUNT_ID

    Holen Sie sich die Slate-Medien-Assets, die für ein Konto definiert wurden. Slate-Medien-Assets werden verwendet, um Werbepausen zu füllen, die nicht durch Anzeigen gefüllt werden. Details zu den Antwortfeldern finden Sie in der API-Referenz.

    Intest Slate Medienquellen-Asset

          POST https://api.bcovlive.io/v1/ssai/slates

    Fügen Sie ein Medien-Asset für Schiefer hinzu, um die unausgefüllte Werbepause zu füllen. Details zu den Feldern des Anforderungskörpers finden Sie in der API-Referenz.

    Löschen von Slate Media Source Asset

          DELETE https://api.bcovlive.io/v1/ssai/slates/:SLATE_MSA_ID

    Löscht ein Slate-Medienasset.

    Statische Eintritt

    Die Static Entry Points (SEP) -Funktion ermöglicht einen lang andauernden Live-Job, der aktiviert und deaktiviert werden kann, während die Einstiegspunkt-URLs und Wiedergabe-URLs statisch und wiederverwendbar bleiben. Diese Funktion ermöglicht es Kunden, ihren Encoder in ihren Einrichtungen oder vor Ort zu konfigurieren, und ermöglicht es dem Kunden, eine eigene Planungslogik für Live-Kanäle oder -Programme zu erstellen. Siehe Statische Einstiegspunkte für Details.

    Untertitel

    Wenn sich Beschriftungen innerhalb des h264-Eingangssignals befinden (im user_data-Paket korrekt signalisiert), werden diese an die h264-Ausgänge weitergegeben.

    Wenn Sie einen Live-Encoder für Broadcast Elemental verwenden, können Sie Untertitel von SDI (EIA-608/CEA-608) oder anderen Quellen (SCTE-20, SCC, Teletext, DVB-Sub, Ancillary, ARIB, TTML, STL, SRT, SMI) erhalten und sie in den h264-Stream einfügen, den Sie uns senden. Andere Encoder für Broadcasts können wahrscheinlich das Gleiche tun, aber wir haben sie nicht offiziell getestet.

    Einfügen von zeitgesteuerten ID3-Metad

    Diese Informationen wurden in zeitgesteuerte Metadaten in ID3 einfügenverschoben.

    Einschränkungen

    • Damit Live-Jobs, die mit der API erstellt wurden, im Live-Modul angezeigt werden und nicht verwendet werden können, müssen Sie das videocloud Objekt beim Erstellen des Auftrags in den Anforderungskörper aufnehmen.

      Beispiel:

      {
  "live_stream": true,
  "region": "eu-central-1",
  "reconnect_time": 1800,
  "live_sliding_window_duration_ms": 0,
  "outputs": [
    { "label": "hls720p", "live_stream": true, "width": 1280, "height": 720, "video_codec": "h264", "h264_profile": "high", "video_bitrate": 2100, "segment_seconds": 4, "keyframe_interval": 60 }
    ,
    { "label": "hls540p", "live_stream": true, "width": 960, "height": 540, "video_codec": "h264", "h264_profile": "main", "video_bitrate": 1500, "segment_seconds": 4, "keyframe_interval": 60 }
    ,
    { "label": "hls360p", "live_stream": true, "width": 640, "height": 360, "video_codec": "h264", "h264_profile": "main", "video_bitrate": 800, "segment_seconds": 4, "keyframe_interval": 60 }
  ],
  "videocloud": {
    "video":
      { "name": "live event UI", "description": "live event UI", "long_description": "", "tags": [], "reference_id": "", "state": "ACTIVE" }
    }
  }
    • Die anfängliche Verbindung vom Encoder liefert die Bandbreiteninformationen, die mit der Live-Playlist erstellt werden sollen. Wenn die anfängliche Verbindung niedrig ist, selbst wenn die Auftragskonfiguration eine hohe Ausgabe hatte, behält die Wiedergabeliste weiterhin die gleichen Informationen in der Wiedergabeliste bei, bis Folgendes abgeschlossen ist:
      • Encoder wird neu gestartet
      • Der CDN-Cache muss möglicherweise auch geleert werden
    • Derzeit ist die Framerate für Input-Streams auf 30 FPS beschränkt. Wenn Sie an einer höheren Bildrate interessiert sind, wenden Sie sich bitte an den Support.
    • Standardmäßig ist die Auflösung des Eingabestreams auf 1080p begrenzt.
    • Bei der Trennung und Wiederherstellung der Verbindung müssen die Streameinstellungen gleich bleiben. Jede Änderung der Anzahl von Audiokanälen, Auflösungen oder Codeceinstellungen kann zu unerwartetem Verhalten führen.
    • Obwohl Sie DASH und MP4 für Remote-Quellen für Video Cloud-Videos hinzufügen können, unterstützt Live derzeit HLS nur.
    • Für Eingangsströme wird nur AAC-Audio unterstützt.
    • Maximal 5 aktive Warte- und nicht gestartete Jobs sind jederzeit erlaubt.

      Zusätzliche Einschränkungen bei gleichzeitigen Jobs:

      • Die Anzahl der channel (24x7) Jobs ist auf 0 oder eine geringe Anzahl pro Region beschränkt (abhängig vom Kontotyp).
      • Die Anzahl der gleichzeitig Laufen event Arbeitsplätze sind regional begrenzt, in der Regel auf 100.
      • Die Anzahl der gleichzeitig Warten auf eine Verbindung event Jobs ist auf 5 begrenzt.
      • Die Anzahl der SEP-Jobs pro Region ist auf 3 oder 10 beschränkt (siehe Unterstützte AWS-Regionen).

      Jedes dieser Limits kann vom Support auf Kontoebene angepasst werden. Wenden Sie sich an Ihren Account Manager, wenn Sie zusätzliche Kapazität benötigen.

    • Die als stream_url for Live-Jobs zurückgegebene „RTMP“ -Adresse ist ein Akamai HD Live-Stream und kein veralteter FMS RTMP-Stream - sie wird von älteren Versionen von Internet Explorer nicht unterstützt.
    • Live-Streams werden über HTTPS bereitgestellt, und wenn Ihr Brightcove-Konto nicht für HTTPS aktiviert ist, kann der Brightcove-Player den Live-Stream nicht laden. Wenn in Ihrem Konto keine HTTPS-Unterstützung für Origin-Dienste aktiviert ist, wenden Sie sich an den Brightcove-Support , um HTTPS-Unterstützung für Origin-Dienste zu aktivieren, um Wiedergabeprobleme zu vermeiden.
    • Wenn Sie eine transmuxierte Formatvariante innerhalb einer Multi-Bitrate-HLS-Ausgabe verwenden, segment_size kann bei der Transmuxing eingeschlossen werden, sollte jedoch so eingestellt werden, dass sie ein Vielfaches der GOP Größe des Eingangsstroms ist. Wenn die Eingabe also 30 fps mit Keyframes alle 60 Frames beträgt, beträgt die GOP Größe 2 Sekunden und die Segmentgröße sollte ein Vielfaches von 2 sein. Wenn Sie dies nicht tun, haben die Stream-Segmente unterschiedliche Größen.

      Außerdem keyframe_interval sollte nicht für irgendwelche Ausgaben angegeben werden.

    • Wenn Sie Ihren eigenen FTP- oder S3-Ursprungsstandort verwenden, muss Ihr CDN so konfiguriert sein, dass es auf Ihren Ursprungsstandort zurückfällt. Das Brightcove Live-System überprüft nicht, dass die Ursprungsstandorte für die in der Job-Anfrage bereitgestellten CDNs vorhanden sind.
