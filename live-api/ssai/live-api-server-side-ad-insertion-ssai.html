    Brightcove Live-API: Serverseitige Anzeigeneinfügung (SSAI)

    In diesem Thema erfahren Sie, wie Sie serverseitige Anzeigeneinfügung (SSAI) für Ihre Live-Stream-Jobs einrichten.

    Übersicht

    SSAI ermöglicht es Ihnen, Anzeigen während eines Live-Streaming-Events zu bestimmten Zeiten anzuzeigen. Zu den Hauptmerkmalen gehören:

    • Fügen Sie Anzeigen mit Cue-Points ein, die von Ihrem Encoder gesendet wurden, oder erstellen Sie mit dem Live API
    • Erhalten Sie „Slate“ -Assets, um ungenutzte Werbezeit zu füllen
    • Vermeiden Sie Werbeblocker mit Anzeigen, die auf der Serverseite in den Livestream eingefügt werden
    Bevor Sie es verwenden können SSAI, muss es für Ihr Konto aktiviert sein. Ihr Kontomanager teilt Ihnen gerne weitere Details hierzu mit.

    Gehen Sie folgendermaßen vor, um einen Live-Stream mit serverseitigen Anzeigen zu erstellen:

    1. Überprüfen Sie die allgemeinen Informationen über die Live-API
    2. Erstellen Sie eine Live-Anzeigenkonfiguration. Sie können dies auch in Video Cloud Studiotun. In den folgenden Abschnitten finden Sie Details zur Verwaltung Ihrer Anzeigenkonfigurationen mit der Live-API.
    3. Erstellen Sie Slate-Assets. Wenn Sie eine Benutzeroberfläche bevorzugen, können Sie Schiefer in Studio verwalten.
    4. Optional: Fügen Sie Cue-Points und Ad Beaconsein.
    5. Jetzt können Sie mit der Live-API einen Live-Stream erstellen. Wenn Sie es vorziehen, Studio zu verwenden, können Sie serverseitige Anzeigen im Live-Modul implementieren

    Im Rest dieses Dokuments finden Sie Details zu benutzerdefinierten Headern, Anzeigenkonfigurationsvariablen, der Verwendung von DFP- und Werbemakros.

    Allgemeine Informationen

    Die folgenden Informationen beziehen sich auf alle Live-APIs mit SSAI-Anfragen.

    Basis-URL

    Die Basis-URL für die Live-API mit SSAI lautet:

        https://api.bcovlive.io/v1/ssai

    Authentifizierung

    Die Authentifizierung für Anfragen erfordert einen Header mit einem API-Schlüssel:

        X-API-KEY: your API KEY

    Wenden Sie sich an Ihren Kundenbetreuer, um einen API-Schlüssel für Ihr Konto zu erhalten.

    Verwaltung von Schiefertchen

    Schiefer sind Ihre eigenen Vermögenswerte, die verwendet werden, um ungenutzte Werbezeit zu füllen. Sie können Schiefer verwenden, um eine „zurück sein“ -Nachricht oder einen beliebigen Inhalt bereitzustellen.

    Im Folgenden finden Sie Details für die API-Anfragen zum Hinzufügen und Verwalten von Slate-Assets.

    Slate-Asset hinzufügen

    Um ein neues Slate-Medienquellen-Asset aufzutragen, senden Sie eine POST Anfrage:

    -Methode POST
    URL https://api.bcovlive.io/v1/ssai/slates
    Überschrift X-API-KEY: your API KEY

    Muster-Anfrage

        {
      "source_url": "https://somesourceasset.com/slate-to-ingest.mp4",
      "account_id": "Account_ID [Optional]",
      "source_description": "User identifiable description for the slate  [Optional]"
    }
    Das account_id hier ist die Live-Konto-ID. Wenn nicht angegeben, wird die Konto-ID des anfragenden Benutzers verwendet.

    Sample-Antwort

        {
      "media_source_asset_id": "New_UUID",
      "account_id": "Account_ID",
      "media_source_asset_default": false,
      "media_source_asset_type": "slate",

    Hinweise zu DFP

    Wenn Sie Anzeigen von DFP erhalten, sollten Sie Folgendes beachten, um Probleme zu vermeiden.

    Anzeigen-Tag

    Wenn Sie ein Anzeigen-Tag für Live erstellen, achten Sie darauf, dass Sie die richtigen Richtlinien befolgen und einschließen /live . Weitere Informationen finden Sie im DFP-Dokument Erstellen eines Mastervideo-Tags manuell .

    Parallelität

    Wenn Sie eine hohe Parallelität erwarten, empfehlen wir Ihnen, mit Ihrem DFP-Kontoteam zu sprechen.

    Einzelne/mehrfache Werbeantworten

    Das singleadresponse und multiadresponse Parameter werden derzeit nicht von verwendet SSAI.

    Live führt SSAI nur einen einzelnen Anzeigeserver-Aufruf durch und erwartet, dass die Antwort alle Anzeigen für die Pause enthält, mit der Ausnahme, dass sie allen erforderlichen Werbebuchungen mit einem Limit von 5 Weiterleitungen pro Anzeige folgt. Die folgenden Anzeigenantwortformate werden akzeptiert und wie folgt analysiert:

    • VAST - Einzelne Antwort oder ein Pod mit Anzeigen in einer einzigen Antwort
    • DFP Ad Rules - Aggregiert alle verfügbaren Anzeigen in der Antwort, einschließlich vor-, mittel- und postroll-definierter Anzeigen
    • Smart XML - Aggregiert alle verfügbaren Anzeigen in der Antwort, einschließlich vor-, mittel- und postroll-definierter Anzeigen

    Benutzerdefinierte Header für Anzeigenanfragen

    Die SSAI Plattform kann benutzerdefinierte Header mit den Anzeigenaufrufen und allen von der Backend-Plattform verwendeten Beacons übergeben. Einige Werbeserver wie VideoPlaza benötigen benutzerdefinierte Header.

    Benutzerdefinierte Header werden als Satz von Schlüssel-Wert-Paaren in einem ad_configuration_headers Objekt angegeben, das Teil des ist application_ad_configuration (siehe Abschnitt Anzeigenkonfiguration erstellen ).

    Hinweise

    • Standard-Header werden standardmäßig behandelt wie:
      • X-Forwarded-For
      • X-Device-User-Agent
    • Header-Werte können die Anzeigenkonfigurationsvariablenverwenden
    • Header-Werte können auch statische Strings sein
    Header werden an alle Downstream-Anfragen weitergegeben, einschließlich Ad-Wrapper oder Impressionen/Quartiles, daher sollten Sie sie mit Vorsicht verwenden.

    Targeting von Anzeigen mit Werbemakros

    Wenn Sie eine Anzeigenkonfiguration erstellen, sieht Ihr Anzeigen-Tag normalerweise in etwa so aus:

          https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}}&
      num={{random.int32}}&ses={{session.session_id}}&IDFA={{deviceid}}&
      sitesection={{sitesection}}&breakid={{breakid}}&breaktheme={{breaktheme}}

    Die Elemente innerhalb der doppelten geschweiften Klammern sind Variablen, auch Werbemakros genannt, die Brightcove Live beim Senden der Anzeigenanfrage durch Werte ersetzt, falls sie existieren.

    Die Variablen, die nützlich sein werden, hängen vom Ad-Server ab.

    Anzeigenmakrowerte können auf folgende Weise bereitgestellt werden:

    • Verwenden von Header-Informationen
    • Anhängen der URL
    • Hinzufügen von URLs dynamisch
    • Verwenden von manuellen Cue-Po

    Verwenden von Header-Informationen

    Header-Informationen, die im obigen Abschnitt Ad-Konfigurationsvariablen beschrieben werden, sind für jede Anfrage verfügbar. Geben Sie einfach die gewünschten Variablen mit den entsprechenden Schlüsselnamen in Ihrer Anzeigenkonfiguration an.

    Anhängen der URL

    Zusätzliche Sitzungswerte können wie folgt an die URL für den Livestream angehängt werden:

          http://playback.bcovlive.io/e058d9f2737e18/us-west-2/NA/
      playlist.m3u8?deviceid=123456&sitesection=services"

    Hinzufügen von URLs dynamisch

    Sie können URLs dynamisch hinzufügen, indem Sie Javascript und die Brightcove Player-API verwenden:

          <!DOCTYPE html>
      <body>
        <video
        id="myPlayerID"
        data-video-id="5975635167001"
        data-account="3737230870001"
        data-player="tIG7lVKBm"
        data-embed="default"
        data-application-id
        class="video-js"
        controls
        width="640"
        height="360"></video>
        <script src="//players.brightcove.net/3737230870001/tIG7lVKBm_default/index.min.js"></script>

        <script type="text/javascript">
        var player = videojs("myPlayerID");

        player.one("loadstart", function(){
          var sourceUrl = player.currentSource();
          console.log(sourceUrl);
          var liveUrlWithParams = sourceUrl.src + "?player_width={width}&player_height={height}&deviceid={deviceid}";

          player.src([{
            "type": "application/vnd.apple.mpegurl",
            "src": liveUrlWithParams
          }]);
        });
      </script>

      </body>

    Beachten Sie, dass die Schlüsselnamen in diesem Beispiel den Variablennamen im oben gezeigten Anzeigen-Tag entsprechen.

    Variablen für die Anzeigen

    Anzeigenkonfigurationsvariablen, auch bekannt als Beacons, können in Anfragen zur Verwaltung von Anzeigenkonfigurationen verwendet werden. Einzelheiten finden Sie in der Live-API: Cue-Points und Ad Beacons mit SSAI-Dokument .

    Verwenden von manuellen Cue-Po

    Werte für bestimmte Werbeblöcke können mit der manuellen Cue-Point-Einfügeanforderung gesendet werden. Einzelheiten finden Sie in der Live-API: Dokument zu Cue-Points und Ad Beacons .

    Bekannte Probleme

    • SSAI erfordert, dass das Live-Streaming-Video eine Audiospur hat.
    • Wenn die zurückgegebene VAST dieselbe Anzeigenkennung hat, fordert der Server den Anzeigeninhalt nicht an, obwohl die Anzeigen-URL dynamische Variablen verwendet, um sie zu einer eindeutigen URL zu machen. Dies gilt nicht , wenn Sie DFP für Werbeanzeigen verwenden.
    • Mit DFP, obwohl Sie dieselbe Anzeigenkennung verwenden können, muss es immer noch eine andere kreative ID geben - mit anderen Worten, Sie können die MediaFile nicht einfach austauschen.

    • Wenn eine Werbeunterbrechung eine Dauer hat, die unter der maximalen Dauer der Anzeigen-URL (min_ad_duration=0&max_ad_duration=30000) liegt, wird die Anzeige nicht abgespielt, wenn die Anzeige länger als die Werbeunterbrechung zurückgegeben wird.

    • VPAID- oder anklickbare Anzeigen werden nicht unterstützt Brightcove Live SSAI.

    • Wenn ein Werbeblock die verbleibende Zeit kürzer als die Segmentsekunden des Streams hat und ein Slate angezeigt wird, wird der Slate für seine Segmentdauer angezeigt und der tatsächliche Werbeblock ist länger als erwartet.

    • Wenn die Anzeige zum ersten Mal von unserem System gesehen wird, wird sie erst wiedergegeben, wenn sie transcodiert und bereitsbereit ist.
    • VMAP for Live SSAI wird derzeit nicht unterstützt.
