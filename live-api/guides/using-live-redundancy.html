    Verwenden von Live-Redundanz

    Redundante Gruppen ermöglichen es Ihnen, 2 oder mehr Live-Jobs in einem einzigen Stream mit automatischem Failover für eine ununterbrochene zuverlässige Wiedergabe zu kombinieren. Dieser Leitfaden führt Sie durch das Erstellen einer redundanten Gruppe, das Hinzufügen von Jobs und erzwingt ein Failover mit der Live-API.

    Einführung

    Die Redundanzfunktion von Brightcove Live trägt dazu bei, eine zuverlässige Leistung für Ihre Live-Ereignisse zu gewährleisten, indem ein Backup-Stream erstellt wird, auf den Live automatisch ein Failover findet, falls der erste Stream nicht mehr funktioniert.

    Beachten Sie, dass alle Anfragen an die Live-API die folgenden Header benötigen:

    API-Anfrage-Header
    Schlüssel Wert Hinweise
    X-API-KEY {dein API Key} Ihr Schlüssel sollte angegeben worden sein, als Sie Ihr Brightcove Live-Konto eröffnet haben
    Content-Type anwendung/json Technisch gesehen ist der content-type Header nur für Schreibanforderungen erforderlich, die einen Anfragetext enthalten, sollte jedoch bei Leseanforderungen keinen Schaden anrichten.

    Erstellen Sie Ihre Live-Jobs

    Für das redundante Setup müssen Sie 2 oder mehr Brightcove Live Jobs erstellen. Die einzige Voraussetzung für die Jobs ist, dass sie mit dem erstellt werden gleich output die Einstellungen. Der einfachste Weg, dies zu erreichen, besteht darin, einen Auftrag mit den gewünschten Ausgabespezifikationen zu erstellen und dann den copy_outputs_from_job Parameter zu verwenden, um die zusätzlichen Jobs zu erstellen.

    Es gibt keine Beschränkung für die Regionen, in denen diese Arbeitsplätze geschaffen werden können. Es wird jedoch empfohlen, dass sie sich relativ nahe beieinander befinden.

    Nachdem die Live-Jobs erstellt wurden, halten Sie die Job-IDs für später fest. Der Anfragetext, mit dem Sie diese Jobs einer redundanten Gruppe hinzufügen werden (siehe die folgenden Abschnitte), sieht folgendermaßen aus:

        [
      {
        "job_id": "0b76bc73f92f46dc917bbe5061c0c633"
      },
      {
        "job_id": "0ae5a4a71dc54b3181af0f98ee407c27"
      }
    ]

    Erstellen Sie die redundante Gruppe

    Um eine Redundante Gruppe zu erstellen, senden Sie eine POST Anfrage an:

        https://api.bcovlive.io/v1/redundantgroups

    Hier ist ein Musteranfragetext:

        {
      "ad_insertion": true,
      "processing_regions": ["us-west-2"],
      "storage_regions": ["us-west-2", "us-east-1"],
      "label": "Test RG",
      "live_dvr_sliding_window_duration": 1800
    }

    Die folgende Tabelle enthält eine vollständige Liste der Felder für den Anfragetext. In mehreren Fällen sind diese identisch mit Feldern, die zur Erstellung eines Live-Jobs verwendet werden. Weitere Informationen zu den Feldern finden Sie in der Live-API-Referenz .

    Erstellen von redundanten Gruppenfeldern
    Feld Art Obligatorisch? Beschreibung
    ad_insertion Boolesch fakultativ Auf „true“ setzen, wenn dieser Stream SSAI-fähig sein soll
    add_cdns Array fakultativ Array zusätzlicher CDN-Anbieter, die für die Manifestgenerierung verwendet werden sollen. Für jedes bereitgestellte CDN wird das Manifest entsprechend vorangestellt
    drm Objekt fakultativ Noch nicht unterstützt
    encryption Objekt fakultativ Noch nicht unterstützt
    label string erforderlich Ein Label zur Identifizierung der Gruppe
    live_dvr_sliding_window_duration ganzzahlig fakultativ
    notifications Array fakultativ Array von Zielobjekten oder Strings für Benachrichtigungen
    processing_regions Array erforderlich Verarbeitungsregionen für die Redundante Gruppe. Dies wird bestimmen, welche AWS-Regionen die Manifeste generieren werden. Es wird empfohlen, dass dies zu storage_regions und den Regionen passt, in denen die Live-Jobs erstellt werden.
    storage_regions Array erforderlich Medienchunks und Wiedergabelisten für Speicherregionen werden in S3 hochgeladen. Empfohlen entspricht dies processing_regions und den Regionen, in denen die Live-Jobs erstellt werden.
    videocloud Objekt fakultativ Video Cloud-Kunden haben die Möglichkeit, ein Video zu erstellen, das für den Live-Stream verwendet werden kann.

    Fügen Sie der redundanten Gruppe Live-Jobs hinzu

    Nachdem Sie eine redundante Gruppe erstellt haben, können Sie ihr Aufträge hinzufügen, indem Sie eine POST Anfrage senden an:

        https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/jobs

    Die Live-Job-IDs werden im Anfragetext wie folgt angegeben:

        [
      {
        "job_id": "0b76bc73f92f46dc917bbe5061c0c633"
      },
      {
        "job_id": "0ae5a4a71dc54b3181af0f98ee407c27"
      }
    ]

    Es gibt ein paar zusätzliche optionale Eigenschaften für die Job-Objekte - die folgende Tabelle zeigt alle Felder:

    Felder für Anforderung von Live-Jobs hinzufügen
    Feld Art Obligatorisch? Beschreibung
    job_id string erforderlich ID des Jobs, der zur Gruppe hinzugefügt werden soll Wenn playlist weder noch angegeben streams sind, outputs wird alles verwendet.
    playlist string fakultativ Beschriftung der Wiedergabeliste, die als Ausgaben für den Stream verwendet werden soll. Wenn definiert playlist ist, streams muss undefiniertsein.
    streams Array fakultativ Liste der Stream-Labels, die als Ausgaben für den Stream verwendet werden sollen. Wenn definiert streams ist, playlist muss undefiniertsein.

    Die Erfolgsantwort für diese Anfrage gibt nur die redundante Gruppen-ID zurück.

    Manuelles Hinzufügen von Cuepoints

    Das manuelle Hinzufügen von Cuepoints zu einem Live-Stream mit Redundanz ist dem Hinzufügen von Cuepoints zu einem normalen Live-Stream sehr ähnlich. Um dies für eine redundante Gruppe zu tun, stellen Sie eine POST Anfrage an:

        https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/cuepoint

    Fügen Sie einen Anfragetext wie den folgenden ein:

        {
      "ad_server_data": {
        "subject": "wildlife"
      },
      "duration": 30,
      "timecode": "09:23:18:05"
    }

    Wenn Sie das timecode Feld weglassen, wird der Cuepoint sofort eingefügt.

    Erzwingen eines Job-Failover

    Failover wird automatisch vom Brightcove Live-System verwaltet und sollte kein Eingreifen Ihrerseits erfordern. Wenn Sie aus irgendeinem Grund das Failover auf einen anderen Job erzwingen möchten, ist es am einfachsten, den Encoder des on_air Jobs einfach anzuhalten.

    Sie können das Failover auch mit der API erzwingen, indem Sie eine PUT Anfrage an folgende Adresse stellen:

        https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/switch

    Der Anfragetext sieht so aus:

        {
      "mode": "manual",
      "on_air": "0ae5a4a71dc54b3181af0f98ee407c27"
    }

    Hier mode wird entweder manual oder auto (der Standard) sein, und der on_air wird den Wert eines der sekundäre Job-IDs

    Hinweis: Einmal im manual Modus wird kein automatisches Job-Failover durchgeführt. Sie müssen den Modus wieder auf einstellen, damit auto das automatische Failover fortgesetzt werden kann. Sie können den On-Air-Job ändern, ohne auf manuell zu wechseln, indem Sie den Modus vom Hauptteil weglassen und Ihrer Anfrage den ?force=true Force-Abfrage-Parameter hinzufügen. Dies zwingt den Dienst, den Job zu wechseln, kann jedoch jederzeit zurückwechseln, wenn Probleme erkannt werden.

    Beenden eines redundanten Streams

    Es gibt zwei Möglichkeiten, wie du deinen redundanten Stream beenden kannst. Redundante Gruppen sind implizit SEP-Streams . Sie können eine Redundante Gruppe in STANDBY Modus versetzen, indem Sie alle Jobs aus der Redundanten Gruppe entfernen.

    Um dies zu tun, verwenden Sie eine DELETE Anfrage an:

        https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}/jobs/{job_id}

    für jeden Auftrag, den Sie entfernen möchten. Beachten Sie dies, um den aktuellen Job zu entfernen on_air müssen Sie die hinzufügen ?force-true Abfrageparameter.

    Die zweite Möglichkeit, den Stream zu beenden, besteht darin, die redundante Gruppe vollständig zu löschen, indem Sie eine DELETE Anfrage senden an:

        https://api.bcovlive.io/v1/redundantgroups/{redundant_group_id}
