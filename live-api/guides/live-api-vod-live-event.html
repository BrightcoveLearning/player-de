    Live-API: VOD von Live-Event

    In diesem Thema erfahren Sie, wie Sie nach Abschluss der Veranstaltung ein Video-on-Demand (VOD) aus einem Live-Streaming-Event erstellen.
    Live to VOD kann nicht mit verschlüsselten Streams verwendet werden und gibt beim Erstellen einer Live-Job-Veranstaltung einen Fehler zurück.

    Einführung

    Bei kürzeren Live-Streaming-Ereignissen möchten Sie möglicherweise nach Abschluss der Veranstaltung eine Aufzeichnung der Veranstaltung als Video-on-Demand (VOD) in Ihrem Video Cloud Konto verfügbar machen. Sie können dies ganz einfach tun, indem Sie beim Erstellen Ihres Live-Jobseine Konfiguration für den VOD hinzufügen.

    Fast Live to VOD ermöglicht die schnelle Veröffentlichung von Clips mit Segmentgenauigkeit, während ein für den Kunden vollständig transparenter Backend-Prozess die Clip-Grenzen aufräumt und erneut veröffentlicht.

    Das Ergebnis ist eine schnellere Markteinführung mit einem Clip, der automatisch mit den gewünschten Clip-Grenzen veröffentlicht wird.

    Wichtig:

    • Wenn das Live-Streaming-Ereignis abgeschlossen ist, stoppen Sie Ihren Encoder, aber brechen Sie den Auftrag nicht ab. Warten Sie, bis der VOD-Clip erstellt wurde, bevor Sie den Auftrag abbrechen, oder er wird nicht erfolgreich erstellt.
    • Die Erstellung des VOD beginnt erst, wenn der reconnect_time für den Live-Job abgelaufen ist.
    • Wenn die Live-Ereignisdauer größer als 24 Stunden ist, werden nur die letzten 24 Stunden im VOD erfasst.

    Schnelle VOD-Option

    Mit der schnellen VOD-Option können Sie die VOD-Version des Live-Events schneller abrufen. Fast Live to VOD wird (unter Verwendung eines Transmux) von Live-Wiedergabeversionen veröffentlichen, die segmentgenau sind.

    Die output Felder für schnelles VOD sind:

    • mode ( wird für schnelles VOD benötigt) - für schnelles VOD wird der Wert sein instant
    • playlist_label ( optional) - Gilt nur für Sofortmodus-VODs. Gibt an, welche Playlist verwendet werden soll, um eine VOD-Ausgabe (aus dem Live-Job) zu erstellen. Nur notwendig, wenn benutzerdefinierte Wiedergabelisten bei der Erstellung von Live-Jobs definiert wurden.

    Die Umsetzung

    senden an Video Cloud

    Beachten Sie, dass es zwei Möglichkeiten gibt, wie videocloud Objekte verwendet werden, wenn Sie einen neuen Live-Job erstellen:

    1. videocloud In den Feldern auf Jobebene wird ein Video im Video Cloud Konto erstellt, das den Live-Stream als Remote-Asset verwendet: 
      {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
    "videocloud": {
  		"video": {
  			"name": "test_live_stream"
  		}
  	}...

      In diesem Fall gibt es kein ingest Objekt innerhalb des videocloud Objekts, da nichts aufgenommen wird.

    2. Wenn Sie das videocloud Objekt in die Eigenschaften eines outputs Clips oder vollständigen VOD-Objekts einbeziehen, wird ein Video Cloud Video für den Clip erstellt - der Clip wird aufgenommen Video Cloud und wird Teil der VOD-Bibliothek:

      {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
	"videocloud": {
		"video": {
			"name": "test_live_stream"
		}
	},
  "outputs": [
      { "videocloud": {
          "video": {
              "name": "video name",
          },
          "ingest": {
              "profile": "valid-ingest-profile-name"
          }
      },
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
      } 
  ]
}

    Wenn Sie eine VOD-Version des Ereignisses in Ihrem Video Cloud Konto speichern möchten, fügen Sie unten eine Version der hervorgehobenen Ausgabe hinzu.

    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 600,
    "ad_insertion": false,
    "outputs": [
        { OTHER_JOB_OUTPUT_OBJECT },
        { OTHER_JOB_OUTPUT_OBJECT_2 },
        { "videocloud": {
            "video": {
                "name": "video name",
                "description": "video description",
                "long_description": "long description",
                "tags": [
                "tag1", "tag2"
                ],
                "schedule":
                { "starts_at": "2017-01-01T12:00:00-04:00", "ends_at": "2099-01-01T12:00:00-04:00" }
                ,
                "custom_fields": {
                    "someCustomField": "value"
                }
            },
            "ingest": {
                "profile": "valid-ingest-profile-name"
            }
        },
        "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
        } 
    ]
}
    Wenn Sie noch keine Anmeldeinformationen für Ihren S3-Bucket eingerichtet haben, können Sie über die APIneue Anmeldeinformationen hinzufügen.

    Beachten Sie, dass Sie für die Video-Metadaten alle Felder einschließen können, die für den Vorgang „Video erstellen “ der CMS-API aufgeführt sind.

    Aufgabe-Profile definieren die Formatvarianten, die für den VOD erstellt werden. Für valid-ingest-profile-name können Sie den Namen eines benutzerdefinierten oder standardmäßigen Aufgaufsichtsprofils für Ihr Konto ersetzen. Welche Standardprofile Sie haben, hängt davon ab, ob Ihr Konto für Dynamic Delivery aktiviert ist oder nicht. Siehe die Standardprofilen für die dynamische Bereitstellung und die kontextbezogige Codierung oder die standardmäßigen Ingest-Profile für Legacy-Transcodierung.

    Beachten Sie, dass das profile Feld optional ist. Wenn Sie kein Profil angeben, wird das Standardprofil für das Konto verwendet.

    An S3 senden

    Um eine VOD-Version des Ereignisses in Ihrem S3-Bucket zu speichern, fügen Sie unten eine Version der hervorgehobenen Ausgabe hinzu.

    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 600,
    "ad_insertion": false,
    "outputs": [
        { OTHER_JOB_OUTPUT_OBJECT },
        { OTHER_JOB_OUTPUT_OBJECT_2 },
        {
        "url":"s3://YOUR_BUCKET/path/filename.mp4",
        "credentials": "YOUR_CREDENTIAL_LABEL",
        "notifications":    [{
            "url": "http://httpbin.org/post?vodStateChange"
        },
        {
            "url": "http://httpbin.org/post?vodFinished",
            "event": "output_finished"
        }]
    }
    ]
}
    Ein Anmeldeinformationslabel für einen gültigen Benutzernamen+Passwort für Ihren S3-Bucket sollte erstellt worden sein, als Ihr Live-Konto eingerichtet wurde und das an Sie gesendete Anmeldeinweisetikett gesendet wurde. Wenn Sie es nicht haben, wenden Sie sich bitte an den Brightcove Support.

    Einschränkungen

    • Wenn Sie den Live-Job stoppen (abbrechen) , bevor der VOD erstellt wurde, wird er nicht erstellt. Stoppen Sie stattdessen Ihren Encoder, um die Erstellung des VOD zu initiieren, und warten Sie, bis dies abgeschlossen ist, bevor Sie den Job beenden.
    • Zum Statischer Einstiegspunkt Streams nur , Live to VOD kann nicht so eingestellt werden, dass der Stream nach Abschluss archiviert wird.

      Alternativen:

      • Verwenden Sie Clipping , um das vollständige Live-Event als VOD zu archivieren. Das Fenster zum Ausschneiden ist für die Standarddauer nach einem Ereignis oder bis zur nächsten Aktivierung des SEP verfügbar (je nachdem, was kürzer ist).
