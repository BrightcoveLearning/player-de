    Live-API: Erstellen von VOD-Clips

    In diesem Thema erfahren Sie, wie Sie Video-on-Demand-Clips (VOD) aus Ihren Live-Streams erstellen.

    Übersicht

    Clips sind Videos, die aus einem Live-Stream extrahiert wurden. Sie können an einen S3-Bucket, eine FTP-Site oder ein Video Cloud Konto gesendet werden. Der Clip wird als MP4-Video erstellt und wird in jedem Fall an das Ziel gesendet. Im Fall von Video Cloud wird der MP4 vom Ingest-System transcodiert, und welche Arten von Wiedergabeversionen für das Video erstellt werden, hängt vom verwendeten Aufgabeprofil ab.

    Definitionen für Clips werden mit dem /vods Endpoint erstellt.

    Clips können auf verschiedene Arten erstellt werden:

    • Mit stream_start_timecode und/oder stream_end_timecode definiert in SMPTE Timecodes für das Livestream-Ereignis - beachten Sie, dass dies erfordert, dass der Encoder Timecode-Informationen sendet
    • Mit start_time und/oder end_time definiert relativ zur Startzeit ( stream_start_time ) des gesamten Livestream-Ereignisses
    • Mit start_time und/oder end_time definiert in Epoch (Unix) Zeit (in Sekunden)
    • Mit duration
    • Die VOD-API kann nicht mit verschlüsselten Aufträgen verwendet werden.

    Hinweise

    1. Legacy-Ingest-Profile unterstützen das Live-Clipping nicht mehr. Um einen Videoclip aus einem Live-Stream zu erstellen, stellen Sie sicher, dass Ihr Konto für Dynamic Delivery aktiviert ist, und verwenden Sie ein Aufgabe-Profil für dynamische Lieferungen.
    2. Um Clips so schnell wie möglich verfügbar zu machen, wird zuerst ein segmentgenauer Clip erstellt und dann durch einen rahmengenauen Clip ersetzt, sobald er verfügbar ist.
    3. Wenn Sie angeben duration, lautet der resultierende Clip wie folgt:
      • Wenn der Job aktiv ist und noch gültig ist: (Anforderungszeit - Dauer) bis (Anforderungszeit)
      • Wenn der Job beendet ist: ( finished_at - Dauer) bis ( finished_at)
    4. Wenn Sie sowohl start_time AND als auch angeben end_time:
      • Wenn der Job aktiv ist und noch live ist: Solange das Epoch-Zeitfenster vollständig eindrauf fällt created_at und die Anforderungszeit erfolgt, wird der Clip erstellt
      • Wenn der Auftrag beendet ist: Solange das Epoch-Zeitfenster vollständig einfällt created_at und finished_at der Clip erstellt wird
    5. Clips von Livestreams mit SSAI enthalten keine Werbung.
    6. Clips können bis zu 7 Tage nach einer Veranstaltung erstellt werden. Für SEP können sie bis zur nächsten Aktivierung oder 7 Tage (je nachdem, was kürzer ist) erstellt werden.
    7. Die VOD-API fügt keine Inhalte außerhalb dessen hinzu, was im Stream vorhanden ist. Wenn Sie 350 in einem 300 Sekunden langen Live-Stream angeben, wird die Ausgabe 300 Sekunden lang sein.
    8. Sie müssen keinen DVR-fähigen Live-Stream verwenden, um zu clippen, um zu funktionieren, da der Live-Stream während der Übertragung gespeichert wird und sofort und 7 Tage nach Ende der Veranstaltung verfügbar ist.
    9. Brightcove Live Clipping erzeugt nur einen Clip mit der gleichen Auflösung wie die Ausgabe mit der höchsten Auflösung. Es stimmt nicht mit der Auflösung der Quelleingabe überein (es sei denn, dies entspricht der Ausgabe mit der höchsten Auflösung).

    Clips können auch an mehrere Ziele gesendet werden:

    • Ein Video Cloud Konto
    • Ein FTP-Server
    • Ein S3-Eimer

    Wenn Sie einen Clip angeben, muss die Ausgabe entweder ein url Ziel oder ein videocloud Objekt enthalten, um die Erstellung des Video und Aufnahme des Clip-ins Video Cloud.

    Hinweis: Clips können erstellt werden, während der Live-Stream läuft. Dazu müssen Sie die Start- und Endzeiten des Clips in Epoch-Zeit oder relativ zur Startzeit des Live-Streams definieren.

    Anmeldedaten

    Wenn das Ziel, an das Sie den Clip senden, Anmeldeinformationen für den Zugriff benötigt, können Sie diese mithilfe der Anmeldeinformationsoperationen der Live-API erstellen. Weitere Informationen finden Sie unter Anmeldeinformationen für die Live-API verwalten .

    Beachten Sie, dass Sie die Anmeldeinformationen nicht in den Clipping-Job einbeziehen sollten, wenn Sie den Clip an ein Video Cloud-Konto senden, das mit dem Live-Konto verknüpft ist.

    Endpunkt

    Clips werden erstellt, indem eine POST Anfrage gesendet wird an:

    https://api.bcovlive.io/v1/vods

    Body anfragen - Video Cloud

    Hinweis: Wir empfehlen dringend, kein Ingest-Profil anzugeben, das für die Transkodierung des Clips verwendet werden sollte, und stattdessen das Standardprofil des Kontos zu verwenden. Dies verringert die Wahrscheinlichkeit von Fehlern beim Erstellen des Clips Video Cloud

    Beispiel 1: Start-/Endzeiten relativ zum Streamstart

    Der Anfragetext enthält Start- und Endzeiten sowie Details darüber, wohin der Clip gesendet werden soll. Hier ist ein Beispielanforderungskörper, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud Konto sendet:

    {
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

    In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute und senden ihn an Video Cloud . Wir geben dem Clip einen Namen und ein paar Tags, die nicht ingest profile für die Retranscodierung angeben, damit der Kontostandardwert verwendet wird, und weisen an, während des Clips Miniaturansichten und Posterbilder aus dem Clip Video Cloud zu erfassen Transkodierung.

    Der Anfragetext enthält Start- und Endzeiten in Epoch-Zeit sowie Details darüber, wohin der Clip gesendet werden soll. Hier ist ein Beispielanforderungskörper, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud Konto sendet:

    {
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
    "outputs":[
      {
        "label": "60 secs - epoch time",
        "start_time": 1516652694,
        "end_time": 1516652754,
        "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
        "videocloud": {
          "video": {
          "name": "One Minute Clip",
          "tags": ["live", "clip"]
          },
            "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

    In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute zu einer bestimmten Epoch-Zeit (in diesem Fall am 22. Januar 2018 um 08:24:54 GMT).

    Der Anfragetext enthält die Dauer und stream_start_time sowie Details darüber, wohin der Clip gesendet werden soll. Hier ist ein Beispielanforderungskörper, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud Konto sendet:

      }

    In diesem Beispiel erstellen wir einen Clip mit einer Dauer von einer Minute, der 5 Minuten nach dem Start des Live-Streams beginnt.

    Beispiel 4: Dauer ohne Start- oder Endzeit

    Der Anfragetext enthält Start- und Endzeiten in Epoch-Zeit sowie Details darüber, wohin der Clip gesendet werden soll. Hier ist ein Beispielanforderungskörper, der einen Clip der dritten Minute eines Streams erstellt und an ein Video Cloud Konto sendet:

    Beispiel 5: Verwenden von stream_start_timecode und stream_end_timecode

    Der Anfragetext enthält Start- und Endzeiten/Frames in HH:MM:SS:FF-Timecodes und Details darüber, wohin der Clip gesendet werden soll. Beachten Sie, dass der Encoder Timecodes senden muss, um Timecodes verwenden zu können. Hier ist ein Beispielanforderungskörper, der einen Clip der 50 Minuten eines Streams erstellt und an ein Video Cloud Konto sendet:

      "stream_start_timecode": "01:10:18:15",
      "stream_end_timecode": "01:11:08:15",
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",

          "name": "Fifty Minute Clip",
          "tags": ["live", "clip"]
        },

        }
      }
    },
    "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
  ]
}

    Allgemeine Informationen zum Senden von Clips an Video Cloud

    Um zu sehen, welche Felder in der enthalten sein können video und ingest Objekte, siehe die Dynamic Ingest API Reference.

    Tun nicht umfassen die master Feld in der ingest Objekt, da diese Informationen von der Live-API bereitgestellt werden.

    Anfrage Body - S3

    Der Anfragetext enthält Start- und Endzeiten sowie Details darüber, wohin der Clip gesendet werden soll. Hier ist ein Beispielanforderungskörper, der einen Clip der dritten Minute eines Streams erstellt und an einen S3-Bucket sendet:

    {
  "live_job_id":"",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

    In diesem Beispiel erstellen wir einen Clip mit einer Dauer von 30 Sekunden und senden ihn an einen S3-Bucket. Wir stellen die Bucket-URL einschließlich des Dateinamens für Clip und eine Zeichenfolge bereit, die den Namen der gespeicherten S3-Bucket-Anmeldeinformationen enthält - die Anmeldeinformationen können vom Brightcove Support für Ihr Konto eingerichtet werden.

    API Antwort

    Die Antwort auf eine Clip-Anforderung zum Erstellen von Clips enthält eine ID für den Auftrag und das Label, das Sie im Anforderungstext festgelegt haben, sowie die Live-Job-ID:

    {
  "vod_jobs": [
    {
      "jvod_id": "9582606c50d84be5ad4bc104f2aa3360",
      "label": "last 60 secs of live job"
    }
  ],
  "live_job_id": "88ba5d87b61a4ef3a6dddabd0c38d319"
}
