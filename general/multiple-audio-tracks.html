Mehrere Audiospuren

{{ page.description }}

Einleitung

Brightcove-Player unterstützt Video mit mehreren alternativen Audiospuren. Wenn das Video mit mehreren alternativen Audiospuren vorbereitet ist, wird ein spezielles Kopfhörersymbol () automatically appears in the controlbar. Wenn Sie ein ordnungsgemäß codiertes Video haben, wird das Symbol angezeigt. Sie müssen nichts weiter tun. Sobald Sie auf das Symbol klicken, kann der Betrachter des Videos zwischen Audiospuren wählen.

Hinweise:
  • Mehrere Audiospuren werden nur für Konten unterstützt, die für Dynamic Delivery aktiviert sind, und das Konto muss speziell für mehrere Audiospuren aktiviert werden. Wenden Sie sich für weitere Informationen an Ihren Account Manager.
  • Mehrere Audiospuren werden nur für VoD-Inhalte unterstützt.
  • Nachdem ein Konto für mehrere Audiospuren aktiviert wurde, alle vorhandenen Videoquellen mit mehreren Audiospuren muss neu transkodiert werden.
  • Wenn Sie einem Video zusätzliche Audiospuren hinzufügen, müssen alle das gleiche Bitraten- / Wiedergabeprofil haben, um eine erfolgreiche Wiedergabe in HLS sicherzustellen (die DASH-Wiedergabe funktioniert weiterhin).
  • Mehrere Audiospuren werden derzeit für Smooth nicht unterstützt.
  • Wenn Sie eine Quellvideodatei mit mehreren Audiospuren hochladen, wird nach dem Transcodieren nur eine Audiospur generiert. Im Allgemeinen ist eine der Spuren als Standard markiert. Wenn dies jedoch nicht die Standardeinstellung ist, wählt das Ingest-System die "beste" Audiospur basierend auf der Stereoqualität und der höchsten Bitrate aus und markiert sie als en / main.

Beispiel

Sobald Sie auf Wiedergabe klicken, wird in der Steuerleiste das Kopfhörersymbol angezeigt, aus dem Sie zwischen zwei alternativen Audiospuren auswählen können.

Anwendungsfälle

Hier sind einige häufige Anwendungsfälle für mehrere alternative Audiospuren:

  • Unterstützung für mehrere Sprachen
  • Stereo versus Mehrkanal-Audio (Surround-Sound)
  • Kommentartrack
  • Beschreibendes Audio

Styling-Symbol und Menü

Wenn die Standardfarben nicht Ihren Anforderungen entsprechen, können Sie die Farbe des Kopfhörersymbols, der ausgewählten Option und anderer Optionen ändern. In der folgenden Tabelle und im folgenden Screenshot sind die Selektoren aufgeführt, die Sie verwenden müssen. (Note that the colors shown in the screenshot are NOT the defaults, but colors to easily identify.)

Selektoren für das Audiosymbol
Element Selektor
Kopfhörersymbol (rot im Screenshot) .video-js .vjs-control-bar .vjs-audio-button
Ausgewählte Option (gelb im Screenshot) .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected
Verfügbare Option (en) (grün im Screenshot) .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item

Das Folgende ist die tatsächliche CSS das beeinflusste die Änderungen im Screenshot:

.video-js .vjs-control-bar .vjs-audio-button {
  color: red;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected {
  background-color: yellow;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item {
  background-color: green;
}

Bekannte Probleme

  • Wenn Sie eine HLS (m3u8) -Fernwiedergabe über a hinzufügen <source> Im Player können die Zuschauer nicht mehrere Audiospuren in Chrome unter dem Android-Betriebssystem auswählen.
