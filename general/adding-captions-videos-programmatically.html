    Programmatisch Untertitel zu Videos hinzufügen

    In diesem Thema erfahren Sie, wie Sie mit dem Brightcove Player Untertitel zu Videos hinzufügen.

    Übersicht

    Fügen Sie Ihren Videos Untertitel hinzu, um dem Betrachter auf Wunsch zusätzliche Informationen oder eine Verständnishilfe zu bieten. Untertitel sind erforderlich, damit taube und hörbehinderte Betrachter den Audio-Teil von Videos verstehen. Um Untertitel für ein Video im Brightcove Player anzuzeigen, müssen Sie die folgenden Aufgaben ausführen:

    • Erstellen Sie eine Untertiteldatei für Ihr Video
    • Ändern Sie den Player-Veröffentlichungscode so, dass er ein <track> -Tag enthält
    • Gestalten Sie die Untertitel
    • Verwenden Sie Code, um Untertitel hinzuzufügen
    Untertitel können auch mit Video Cloud Studio konfiguriert werden. Weitere Informationen finden Sie unter Hinzufügen von Untertiteln zu Videos.

    Erstellen von Untertiteldateien

    Um einem Video Untertitel hinzuzufügen, müssen Sie zunächst eine Untertiteldatei erstellen. Für den Brightcove Player müssen Untertitel im Web Video Text Tracks Format (WebVTT) vorliegen. (Wenn Sie Untertitel in ein Dynamic Delivery-Konto aufnehmen, können Sie DFXP- oder SRT-Beschriftungen hinzufügen. Diese werden automatisch in WebVTT-Beschriftungen konvertiert.) Das WebVTT-Dateiformat ist ein reines Textformat und enthält Zeittextdaten gefolgt vom gewünschten Untertiteltext. Hier ein Auszug aus einer WebVTT-Datei:

    WEBVTT

00:00:00.000 --> 00:00:07.080 align:middle line:90%

00:00:07.080 --> 00:00:10.280 align:middle line:84%
Hi, I'm Bob Bailey, a Learning
Specialist with Brightcove.

00:00:10.280 --> 00:00:12.780 align:middle line:84%
In this video, you'll learn
about how Video Cloud Studio is

00:00:12.780 --> 00:00:14.940 align:middle line:84%
organized and get a
brief overview of what

00:00:14.940 --> 00:00:17.490 align:middle line:90%
you can accomplish using it.

    Weitere Informationen über das WebVTT-Format finden Sie hier. Sie sollten eine separate Untertiteldatei für jede Sprache haben, die Sie unterstützen müssen. Ihnen stehen neben Brightcove-Partnern, die Transkription, Untertitelung und Übersetzungen anbieten, auch viele Tools von Drittanbietern zum Erstellen von Untertiteldateien zur Verfügung. Eine vollständige Liste aller Brightcove-Partner finden Sie im Abschnitt Partners auf unserer Website.

    Wenn Sie Dynamic Delivery-Videos verwenden, können Sie auch SRT- oder DFXP-Untertitel aufnehmen, und Brightcove konvertiert sie automatisch in WebVTT. Siehe die WebVTT-Dateien aufnehmen (Textspuren / Bildunterschriften) Dokument für weitere Informationen.

    Fügen Sie Videos Untertitel hinzu

    In Video Cloud können Sie Ihre WebVTT-Untertitel einem Video hinzufügen, entweder als Remote-Asset, das Sie hosten, oder indem Sie es in Video Cloud aufnehmen. Sobald Sie dies getan haben, fügt der Brightcove-Player beim Laden des Videos automatisch die Untertitel hinzu und zeigt die an CC Taste in den Player-Steuerelementen. Weitere Informationen zum Hinzufügen von Untertiteln zu Videos finden Sie unter Überblick: Hinzufügen von Untertiteln zu einem Video.

    Hinweis: Wenn Sie Untertiteldateien auf Ihrem eigenen Server hosten, stellen Sie sicher, dass sich die Datei unter einer URL befindet, auf die der Brightcove Player zugreifen kann. Auf dem Server, der die Untertiteldatei hostet, sollte auch Cross-Origin Resource Sharing (CORS) aktiviert sein. Weitere Informationen über CORS finden Sie hier.

    Einschließlich eines <track> -Tags

    Um einem Brightcove-Player WebVTT-Untertiteldateien zuzuordnen, müssen Sie den In-Page-Einbettungscode so ändern, dass er einbezogen wird <track> Tag für jede Untertiteldatei, die Sie haben. Beispiel:

    <track kind="captions" src="<location of caption file>" srclang="en" label="English">

Beachten Sie, dass die Verwendung der Einstellung Auto (empfohlen) die Anzeige der Untertitel für das eigentliche Video nicht beeinflusst. Die Zuschauer sehen Videotitel für Anzeigen, aber sobald das Video abgespielt wird (Pre-Rolls) oder fortgesetzt wird (Mid-Rolls), werden die Untertitel wie erwartet angezeigt.

    Untertiteldesign

    In diesem Abschnitt erfahren Sie, wie Sie Untertitel programmgesteuert formatieren. Informationen zum Stylen von Untertiteln durch Ändern der WebVTT-Datei sowie der Benutzeroberfläche im Player finden Sie in der Bildunterschriften stylen Dokument.

    Zwei Methoden sind der Schlüssel zum programmgesteuerten Gestalten von Untertiteln:

    • player.textTrackSettings.setValues():: Legt dynamisch Stile für die Beschriftungen in einem Player fest
    • player.textTrackSettings.saveSettings():: Speichert die Einstellungen im Browser lokaler Speicher

    setValues (objectOfStyles)

    Einstellungen speichern()

    Eine andere Methode, die beim Stylen von Untertiteln hilfreich sein kann, ist player.textTrackSettings.saveSettings(). Diese Methode wird im Browser gespeichert lokaler Speicher die aktuellen Einstellungen des Untertitelstils. Der folgende Screenshot von Chrome Anwendung Die Registerkarte zeigt den lokalen Speicher für die Seite, die den Player bedient, und den Namen des Schlüssels. vjs-text-track-settings und die zugehörigen Werte.

    Diese Methode bietet Ihnen die Möglichkeit, die Beschriftungsstile über die Benutzeroberfläche des Players zu konfigurieren, die Einstellungen zu speichern und die zu verwendenden Werte zu kopieren player.textTrackSettings.setValues() , wie oben gezeigt.

    Implementieren mit Code

    Es ist möglich, programmgesteuert Textspuren für Beschriftungen hinzuzufügen. Die Schlüsselmethode für diese Funktionalität ist addRemoteTextTrack() , deren Syntax lautet:

    playerObject.addRemoteTextTrack({
  kind: string,
  language: string,
  label: string,
  src: string
});

    wo:

    • kind- Art der Textspur für diese Verwendung "Bildunterschriften"
    • language- Zum Beispiel ein Sprachcode mit zwei Buchstaben 'en'
    • label- Die Sprachbezeichnung, die beispielsweise in der Benutzeroberfläche für Beschriftungen angezeigt wird 'Englisch'
    • src- URL zur WebVTT-formatierten Untertiteldatei

    Der Funktionsaufruf gibt ein Objekt vom Typ zurück vjs.TextTrack.

    Anwendungsbeispiel:

    var frTrack = myPlayer.addRemoteTextTrack({
  kind: 'captions',
  language: 'fr',
  label: 'French',
  src: 'https://solutions.brightcove.com/bcls/captions/adding_captions_to_videos_french.vtt'
});

    Sie müssen auf die warten loadedmetadata Ereignis, das versendet werden soll, bevor Sie das verwenden addRemoteTextTrack() Methode. Das folgende Beispiel zeigt den Skriptblock zum Hinzufügen von englischen und französischen Untertiteln zu einem Video.

    <script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function () {
    var myPlayer = this;
    myPlayer.one("loadedmetadata", function () {
      var enTrack = myPlayer.addRemoteTextTrack({
        kind: 'captions',
        language: 'en',
        label: 'English',
        src: 'https://solutions.brightcove.com/bcls/captions/adding_captions_to_videos.vtt'
      });
      var frTrack = myPlayer.addRemoteTextTrack({
        kind: 'captions',
        language: 'fr',
        label: 'French',
        src: 'https://solutions.brightcove.com/bcls/captions/adding_captions_to_videos_french.vtt'
      });
    });
  });
</script>

    Die komplementäre Methode textTracks() Ermöglicht das Abrufen aller einem Player zugewiesenen Textspuren. Die Methode gibt ein Objekt vom Typ zurück vjs.TextTrackList.

    Live-Streaming mit Untertiteln

    Bis zu diesem Zeitpunkt wurden die Dokumentbeschriftungen als separate Datei geliefert und über das <track> -Element einem Video zugeordnet. Diese Art von Untertiteln wird manchmal als bezeichnet außerhalb der Bandbreite. Die alternative Methode beinhaltet das Einbetten der Untertiteldaten direkt in den Videoinhalt, häufig Live-Videostreams, und wird manchmal aufgerufen In-Band Bildunterschriften.

    Brightcove Live unterstützt derzeit 608 Untertitel im Band (innerhalb von h.264-Headern). Wenn sich Beschriftungen innerhalb des h.264-Eingangssignals befinden (im user_data-Paket korrekt signalisiert), werden diese an die h.264-Ausgänge weitergegeben.

    Wenn Sie einen Sendeelementar-Live-Encoder verwenden, können Sie Untertitel von SDI (EIA-608/CEA-608) oder anderen Quellen (SCTE-20, SCC, Teletext, DVB-Sub, Ancillary, ARIB, TTML, STL, SRT, SMI) erhalten und sie in den h.264-Stream einfügen, den Sie uns senden. Andere Encoder für Broadcasts können wahrscheinlich das Gleiche tun, aber wir haben sie nicht offiziell getestet.

    WebVTT-Untertitel werden für Live-Streams nicht unterstützt.

    Brightcove Player unterstützt In-Band-Untertitel und daher Live with Captions, die in HLS integriert sind. Daher werden keine Plugins benötigt. Die Implementierung von Brightcove Player funktioniert überall dort, wo natives HLS nicht verwendet wird. Wenn natives HLS verwendet wird, hängt die Unterstützung für In-Band-Untertitel vom Betriebssystem und vom Browser ab. Zum Beispiel bietet das native HLS von iOS volle Unterstützung für In-Band-Untertitel.

    Einschränkungen

    Untertitel werden auf folgenden Geräten und unter folgenden Betriebssystemen unterstützt.

    • iOS 9+
    • Android 4+
    • Internet Explorer 8+
    • Die neuesten Versionen der Browser Chrome, Safari und Firefox auf Desktops
    Da Safari die nativen Funktionen zum Styling von Untertiteln verwendet, ist die Menüoption Untertiteleinstellungen im CC-Menü in Safari-Browsern (sowohl iOS als auch macOS) nicht verfügbar.
