    Implementierung von serverseitigen Anzeigen im Live-Modul

    In diesem Thema erfahren Sie, wie Sie serverseitige Werbung im Live-Modul implementieren.

    Einführung

    Das Live-Modul unterstützt serverseitige Anzeigeneinfügung (SSAI), sodass serverseitige Anzeigen während eines Live-Streams angefordert und angezeigt werden können. SSAI bietet diese Hauptfunktionen während Live-Events:

    • Da Anzeigen serverseitig in den Live-Videostream eingefügt werden, sind sie nicht von Werbeblockern betroffen
    • Anzeigen werden an Cue-Points eingefügt, die von Ihrem Encoder gesendet werden, oder Sie können mit dem Live-Modul Control Room einen sofortigen Cue-Point erstellen
    • Sie können Füllschieferso konfigurieren, dass sie ungenutzte Anzeigenzeit ausfüllen
    Bevor Sie SSAI verwenden können, muss Ihr Video Cloud-Konto für Dynamic Delivery konfiguriert und für SSAI aktiviert sein. Ihr Kontomanager teilt Ihnen gerne weitere Details hierzu mit.

    Dieses Thema konzentriert sich auf die erforderlichen Einstellungen und den Prozess zum Anfordern einer serverseitigen Anzeige während eines Live-Events.

    Konfigurieren der serverseitigen Anzeigeneinstellungen

    Bevor serverseitige Anzeigen ausgeliefert werden können, müssen serverseitige Anzeigeneinstellungen konfiguriert werden. Die serverseitigen Anzeigeneinstellungen definieren verschiedene Aspekte der Anzeigenwiedergabe, einschließlich Anzeigenanrufe, Timing, Beacons und anderer Anzeigenkonfigurationsoptionen. Diese Einstellungen werden als Teil der serverseitigen Anzeigeneinstellungen im Admin-Modul konfiguriert. In diesem Beispiel wird ein einfaches Anzeigen-Tag definiert.

    Weitere Informationen finden Sie unter Konfigurieren von serverseitigen Anzeigeneinstellungen.

    Konfigurieren des Füllschiefers

    Füllschiefer sind Videos, die angezeigt werden, wenn Lücken in einer Live-Stream-Werbeunterbrechung bestehen, die nicht mit einer dynamisch bereitgestellten Anzeige gefüllt werden können. Wenn die angeforderte Anzeige kürzer ist als die Werbepause, wird die Füllungsliste angezeigt, sodass die Zuschauer keinen leeren Player sehen. Sie sollten mindestens einen Fill Slate konfigurieren, bevor Sie eine serverseitige Anzeige anfordern. Anzeigenschiefer werden als Teil der Live-Einstellungen im Admin-Modul konfiguriert. In diesem Beispiel wurde ein Füllfeld definiert, das das Brightcove-Logo anzeigt.

    Informationen zum Erstellen von Anzeigenschießen finden Sie unter Live-Einstellungen verwalten.

    Erstellen eines Live-Events, das serverseitige Anzeigen unterstützt

    Damit ein Live-Ereignis serverseitige Anzeigen anzeigen kann, muss bei der Erstellung des Ereignisses die Einstellung Serverseitige Anzeigeneinfügung (SSAI) aktivieren ausgewählt werden. Vollständige Informationen zum Erstellen eines Live-Events mit dem Live-Modul finden Sie unter Erstellen und Verwalten von Live-Events mit dem Live-Modul.

    Sie sollten die serverseitigen Anzeigeneinstellungen konfiguriert und mindestens eine Füllseite hinzugefügt haben, bevor Sie ein Ereignis erstellen.

    Gehen Sie folgendermaßen vor, um ein neues Live-Event zu erstellen, das serverseitige Anzeigen unterstützt.

    1. Öffnen Sie das Live-Modul.
    2. Klicken Sie auf der Seite „Ereignisse verwalten“ auf Veranstaltung erstellen.
    3. Geben Sie die Veranstaltungsdetails nach Bedarf ein.
    4. Klicken Sie auf ERWEITERTE OPTIONEN um die erweiterten Optionen zu erweitern.
    5. Aktivieren Sie Serverseitige Anzeigeneinfügung aktivieren (SSAI). Dies ermöglicht das serverseitige Einfügen von Anzeigen.
    6. Wählen Sie ein Fill Slate. Die Fill Slate wird angezeigt, wenn Lücken in einer Live-Stream-Werbeunterbrechung bestehen, die nicht mit einer dynamisch bereitgestellten Anzeige gefüllt werden kann.
    7. Wenn die Einstellung „ Ad Audio “ aktiviert ist, werden die Audiopegel in Anzeigen basierend auf der festgelegten Anpassung normalisiert. Dies kann verwendet werden, um sicherzustellen, dass Anzeigen nicht wesentlich lauter sind als der Inhalt. Brightcove empfiehlt, diese Option zu wählen.
    8. Klicken Sie auf „ Ereignis erstellen“
    9. Das Kontrollraumseite wird mit Details zum Live-Event einschließlich der geöffnet Streaming-Endpunkt (RTMP-URL) und Stream Name. Diese Werte werden vom Encoder verwendet.
    10. Konfigurieren Sie den Encoder und starten Sie den Live-Stream.
    11. Standardmäßig zeigt  der Control Room-Vorschau-Player den werbefreien Stream an. Anzeigen werden nicht im Control Room geschaltet, es sei denn, eine Live-Anzeigenkonfiguration ist ausgewählt. Verwenden Sie das Dropdown-Menü unten für den Player, um eine Anzeigenkonfiguration auszuwählen.
    12. ( Optional) Klicken Sie auf die Plus-Schaltfläche () , um eine zusätzliche Anzeigenkonfiguration auszuwählen. Dadurch wird auf der Control Room-Seite ein weiterer Vorschau-Player hinzugefügt.

    Klicken Sie auf die Lautstärketaste unter dem Vorschau-Player, um zu steuern, welcher Player-Audiostream aktiv ist.

    Wenn das Ereignis RTMP-Ausgaben verwendet, werden Anzeigen nicht in die RTMP-Ausgabe gestreamt. Der werbefreie Stream wird stattdessen gestreamt.
    Wenn das Ereignis von der Live to Facebook-Funktion im Social-Modul verwendet wird, werden keine Anzeigen an Facebook gestreamt. Der werbefreie Stream wird stattdessen gestreamt.

    Anfordern einer Anzeige

    Um eine Anzeige anzufordern, klicken Sie auf Gehe zu Ad. Wenn Sie darauf klicken, kann die Länge des Werbeblendes ausgewählt werden.

    Klicken Sie auf Einfügen , um einen Werbeblock in den Livestream einzufügen. Wenn die Anzeige nicht die gesamte Werbepause ausfüllt, wird die Füllungsliste (ausgewählt, als die Veranstaltung erstellt wurde) angezeigt. Anzeigen werden im Vorschauspieler des Control Room und in Spielern angezeigt, die mit dem Live-Modul veröffentlicht werden. Dieses Beispiel zeigt 2 Anzeigen, die im Control Roomgeschaltet werden.

    Veröffentlichen des Live-Events

    Live-Streams mit serverseitigen Anzeigen können nicht mit dem Medienmodul veröffentlicht werden, sie müssen mit dem Live-Modul veröffentlicht werden.

    Gehen Sie folgendermaßen vor, um den Einbettungscode für ein Live-Event mit serverseitigen Anzeigen zu generieren:

    1. Klicken Sie auf ein Ereignis, um die Control Room-Seite zu öffnen.
    2. Klicken Sie in der linken Navigationsleiste auf Veröffentlichen und Einbetten .
    3. Wähle einen verfügbaren Spieleraus.
      Wenn das Ereignis so konfiguriert ist, dass serverseitige Anzeigen (SSAI), DVR oder Multi-CDN aktiviert werden, muss Brightcove Player Version 6.18.3 und höher verwendet werden. Nur Spieler der Version 6.18.3 und höher werden in der Dropdown-Liste angezeigt.
    4. Wählen Sie die Anzeigenkonfiguration für den Player aus. Die Anzeigenkonfiguration bestimmt, welche Anzeigen im Player angezeigt werden.
    5. Klicken Sie auf die Player-URL , um eine Vorschau des Ereignisses in einem Browser anzuzeigen oder den entsprechenden zu verwendenden Einbettungscode zu kopieren.

    Veröffentlichen einer Remote-Live-Veranstaltung

    Wenn Sie ein Remote-Live-Event veröffentlichen möchten, führen Sie die folgenden Schritte aus:

    1. Nachdem Sie Ihr Live-Event erstellt haben, indem Sie die oben aufgeführten Schritte befolgen, kehren Sie zum Control Room des Live-Modulszurück.

    2. Wenn Sie den Wert Ihrer Anzeigenkonfig-ID kennen, können Sie sie in die Playlist-URL einfügen.

      Kopieren Sie im Control Room des Live-Moduls die Wiedergabe-URL.

      Geben Sie als Nächstes die Anzeigenkonfig-ID (045f90cfd55187ff6e60c8) ein, die Sie für die URL verwenden möchten:

      https://bcovlive-a.akamaihd.net/21e83ff8c229478e8c81416838eb6e1f/us-west-2/NA/045f90cfd55187ff6e60c8/playlist.m3u8

      Benennen Sie dann die Wiedergabeliste von playlist.m3u8 in um playlist_ssaiM.m3u8. Das Remote-Asset sollte ähnlich aussehen:

      https://bcovlive-a.akamaihd.net/21e83ff8c229478e8c81416838eb6e1f/us-west-2/NA/045f90cfd55187ff6e60c8/playlist_ssaiM.m3u8

      Gehe zu Schritt 6 und verwende diese Playlist-URL.

    3. Wenn Sie den Wert Ihrer Anzeigenkonfig-ID nicht kennen, können Sie die vollständige Playlist-URL wie folgt abrufen. Wählen Sie die Anzeigenkonfiguration aus, die Sie verwenden möchten.

    4. Öffnen Sie für Ihren Browser die Registerkarte Netzwerkkonsole. In diesem Fall verwenden wir den Chrome-Browser.

    5. Suchen Sie in der Netzwerkkonsole nach „PlayList_SSAIM“. Kopieren Sie die URL, die mit „playlist_SSAIM.m3u8" endet. Möglicherweise müssen Sie die Seite aktualisieren und Ihren Netzwerk-Tab erneut öffnen.

      Da die Anzeigenkonfig-ID in der URL enthalten ist, unterscheidet sich Ihre SSAI-URL je nach ausgewählter Anzeigenkonfiguration.

      In diesem Beispiel lautet die URL:

      https://bcovlive-a.akamaihd.net/21e83ff8c229478e8c81416838eb6e1f/us-west-2/NA/582ea6fafea84bf595f610b8ebbeeda8/playlist_ssaiM.m3u8

    6. Navigieren Sie in Studio zum Medienmodul und erstellen Sie ein neues Remote-Asset, das auf die Wiedergabelisten-URL aus dem vorherigen Schritt verweist. Wählen Sie im Menü auf der linken Seite die Option + Remote-Video hinzufügenaus.

    7. Geben Sie im Bereich „Remote-Video hinzufügen “ einen Videonamen und die Wiedergabelisten-URL ein. Wählen Sie die Schaltfläche „ Speichern “.

    8. Sie haben jetzt eine Video-ID, die auf Ihren Remote-Livestream verweist. Sie können dies mit jedem Spieler in Ihrem Konto verwenden, um Live-Inhalte mit serverseitigen Anzeigen zu streamen.

    Fehlerbehebung bei Problemen mit dem Live-Stream

    Falls Sie beim Einrichten Ihrer Live-Veranstaltung Hilfe benötigen, kontaktieren Sie uns. Um sicherzustellen, dass Ihr Problem schnellstmöglich gelöst wird, sollten Sie die folgenden Informationen für den Support bereitstellen:

    • Beschreibung der speziellen Symptome des Streams. Beispiel: Ist überhaupt keine Wiedergabe möglich, oder kommt es zu Stockungen oder zum Einfrieren?
    • Information, ob der Stream bisher korrekt verfügbar war
    • Im Codierer verwendete Einstiegspunkt-URL
    • Verwendete Codiersoftware und -hardware
    • URL für den Player, auf dem Sie die Live-Veranstaltung veröffentlicht haben
    • Video-ID Ihrer Live-Datei in Video Cloud Studio
    • Ergebnisse einer Ablaufverfolgungsroute von Ihrem Encoder zum Host des Veröffentlichungspunkts
